                                                                                                          ABSTRACT

                                                                                                          摘要

                                                                                                          Recent research on instructable agents has used memory-augmented Large Language Models (LLMs) as task planners, a technique that retrieves language-program examples relevant to the input instruction and uses them as in-context examples in the LLM prompt to improve the performance of the LLM in inferring the correct action and task plans. In this technical report, we extend the capabilities of HELPER, by expanding its memory with a wider array of examples and prompts, and by integrating additional APIs for asking questions. This simple expansion of HELPER into a shared memory enables the agent to work across the domains of executing plans from dialogue, natural language instruction following, active question asking, and commonsense room reorganization. We evaluate the agent on four diverse interactive visual-language embodied agent benchmarks: ALFRED, TEACh, DialFRED, and the Tidy Task. HELPER-X achieves few-shot, state-of-the-art performance across these benchmarks using a single agent, without requiring in-domain training, and remains competitive with agents that have undergone in-domain training. 近期关于可指令代理的研究采用带记忆增强的大型语言模型（LLMs）作为任务规划器：检索与输入指令相关的语言-程序示例，并将其作为上下文示例加入 LLM 提示词中，以提升 LLM 推断正确动作和任务计划的能力。在这份技术报告中，我们通过扩充记忆库中的示例与提示，并集成用于提问的额外 API，来拓展 HELPER 的能力。将 HELPER 这一简单扩展为共享记忆，使代理能够跨越来自对话的计划执行、自然语言指令跟随、主动提问以及常识性房间重组等领域工作。我们在四个多样的交互式视觉-语言具身代理基准上评估了该代理：ALFRED、TEACh、DialFRED 和 Tidy Task。HELPER-X 在这些基准上使用单一代理、无需领域内训练即可实现少样本、最先进的性能，并且与经过领域内训练的代理相比仍具有竞争力。

                                                                                                          1 INTRODUCTION

                                                                                                          1 引言

                                                                                                          A typical way to adapt LLMs to downstream applications is through prompting (Brown, 2020; Alayrac, 2022; Liu, 2022b; Hongjin, 2022; Mishra, 2022; Wei, 2021; Song, 2022b), exploiting their strong in-context and few-shot learning abilities. When the amount of in-context examples and task descriptions necessary to cover the task constraints increases, inference costs significantly rise due to additional attention operations. To handle computational challenges and LLM context length, a growing body of research explores the concept of "memory-augmented prompting" - a method that involves retrieving a set of pertinent in-context examples to append to the prompt, thereby broadening their applicability (Perez, 2021; Schick, 2021; Gao, 2020; Liu, 2022c; Song, 2023; Sarch, 2023; Lewis, 2020; Mao, 2021). HELPER Sarch (2023) retrieves a set of language-program examples based on the user's input instruction and adds them to the prompt to provide contextualized examples for GPT-4 task planning. 将 LLM 适配到下游应用的一种典型方式是通过提示（Brown, 2020; Alayrac, 2022; Liu, 2022b; Hongjin, 2022; Mishra, 2022; Wei, 2021; Song, 2022b），利用它们强大的上下文学习与少样本学习能力。当为覆盖任务约束所需的上下文示例与任务描述数量增加时，由于额外的注意力运算，推理成本会显著上升。为应对计算挑战与 LLM 上下文长度，越来越多的研究探讨“记忆增强式提示（memory-augmented prompting）”这一概念——检索一组相关的上下文示例并追加到提示中，从而拓展其适用范围（Perez, 2021; Schick, 2021; Gao, 2020; Liu, 2022c; Song, 2023; Sarch, 2023; Lewis, 2020; Mao, 2021）。HElPER Sarch（2023）根据用户输入指令检索一组语言-程序示例，并将其加入提示，为 GPT-4 的任务规划提供带语境的示例。

Applying HELPER, prompted with TEACh-specific examples and descriptions, to ALFRED - a related domain that shares the same action space and environments but differs in language inputs and task types - results in a notable 6.9% decrease in accuracy compared to a HELPER model with a specialized prompt and customized example memory specifically for ALFRED, and vice versa when doing the same on TEACh (3.2% decrease), as shown in Figure 1 (right). 尽管 GPT-4 具有强大的泛化能力，但为某一领域定制的记忆增强式提示并不能保证在相似却不同的领域中同样取得高性能。将 HELPER——使用 TEACh 特定示例与描述进行提示——应用到 ALFRED（一个相关领域：共享相同动作空间与环境，但语言输入与任务类型不同）会使准确率相较于为 ALFRED 量身定制提示与示例记忆的 HELPER 模型下降显著的 6.9%；反过来在 TEACh 上进行同样操作也同样下降（3.2%），如图 1（右）所示。

                                                                                                          We report that two simple extensions of HELPER that allows for strong performance across four domains, by expanding its memory with a wider array of examples and prompts, and by integrating a additional APIs for asking questions. Specifically,we introduce two HELPER-X variants: HELPER- XP , which retrieves domain-specific prompt templates and related in-context examples for the LLM,and HELPER- XS ,which retrieves in-context examples from a shared memory under a single prompt template. 我们报告：对 HELPER 的两个简单扩展能够在四个领域上实现强性能——通过用更广泛的示例与提示来扩展其记忆，并通过整合额外的用于提问的 API。具体而言，我们提出两种 HELPER-X 变体：HElPER- XP，它为 LLM 检索领域特定的提示模板及相关的上下文示例；以及 HELPER- XS，它在单一提示模板下从共享记忆中检索上下文示例。

                                                                                                          We evaluate HELPER-X across four domains that include dialogue-based task completion on TEACh (Padmakumar, 2021), following instructions from natural language on ALFRED (Shridhar, 2020), engaging in instruction following with active question asking on DialFRED (Gao, 2022), and organizing rooms using spatial commonsense reasoning in the Tidy Task (Sarch, 2022). HELPER-X demonstrates state-of-the-art performance in the few-shot example domain, that is, without in-domain training. Extending the language-program memory does not cause interference and does not hinder performance. In fact, HELPER-X matches, and sometimes even exceeds, the performance of agents prompted with a single-domain in mind. 我们在四个领域评估 HELPER-X：包括在 TEACh 上基于对话的任务完成（Padmakumar, 2021）；在 ALFRED 上根据自然语言指令执行（Shridhar, 2020）；在 DialFRED 上通过主动提问进行指令跟随（Gao, 2022）；以及在 Tidy Task（Sarch, 2022）中利用空间常识推理对房间进行组织。HElPER-X 在少样本示例领域（即不进行域内训练）展现了最先进的性能。扩展语言-程序记忆不会引发干扰，也不会阻碍性能。事实上，HElPER-X 的表现与仅面向单一领域进行提示的智能体相当，有时甚至更高。

                                                                                                          2 HELPER-X

                                                                                                          We extend HELPER (Sarch, 2023) to work across four domains. We propose two versions to extend the memory-augmented prompting of LLMs in HELPER: 1) HELPER- XP that retrieves from a memory of domain-tailored prompt templates and associated domain-specific examples (Section 2.2.1),and 2) HELPER- XS that expands the memory of HELPER into a shared memory of in-context examples across domains combined with a domain-agnostic prompt template (Section 2.2.2). 我们将 HELPER (Sarch, 2023) 扩展到四个领域。我们提出两种版本，以扩展 HELPER 中 LLM 的记忆增强提示：1) HELPER- XP，它从领域定制的提示模板及相关领域特定示例的记忆中检索（第 2.2.1 节），以及 2) HELPER- XS，它将 HELPER 的记忆扩展为跨领域共享的上下文示例记忆，并结合一个与领域无关的提示模板（第 2.2.2 节）。

                                                                                                          Additionally, we extend the capabilities of HELPER for question asking, by appending a question API with functions defining possible questions and their arguments to the LLM prompt (Section 2.2.3). We use HELPER (Sarch, 2023) for execution of the generated program using standard perception modules. 此外，我们通过在 LLM 提示中附加一个 question API 来扩展 HELPER 的提问能力，该 API 以函数形式定义可能的问题及其参数（第 2.2.3 节）。我们使用 HELPER（Sarch, 2023）结合标准感知模块来执行生成的程序。

                                                                                                          2.1 BACKGROUND

                                                                                                          2.1 背景

                                                                                                          Here, we give an account of HELPER to make the paper self-contained. 这里我们对 HELPER 作一说明，以使本文自洽。

                                                                                                          HELPER prompts an LLM, namely GPT-4 (gpt, 2023), to generate plans as Python programs. It assumes that the agent has access to a set of action skills S (e.g., go_to(X), pickup(X), etc.). HELPER adds these skills to the prompt in the form of a Python API. The LLM is instructed only to call these pre-defined skill functions in its generated programs. HELPER considers a key-value memory of language inputs and successful program pairs. It retrieves a set of in-context examples relevant to the current input language to add to the prompt to assist program generation. Each key is encoded into a language embedding. The top- k language-program pairs are retrieved based on their euclidean distance to the encoding of the language input I encoding.The HELPER prompt also contains a role description ("You are a helpful assistant with expertise in..."), a task description ("Your task is to ...") and guidelines to follow for program generation ("You should only use functions in the API..."), that are commonly tailored to the domain-of-interest. HELPER 会提示一个 LLM，即 GPT-4 (gpt, 2023)，将计划生成 Python 程序。它假设智能体可访问一组动作技能 S（如 go_to(X)、pickup(X) 等）。HELPER 以 Python API 的形式将这些技能加入提示中。LLM 只被要求在其生成的程序中调用这些预定义的技能函数。HELPER 维护一个由语言输入和成功程序对组成的键值记忆。它检索一组与当前输入语言相关的上下文示例，以加入提示中辅助程序生成。每个键都编码为语言嵌入。根据它们与语言输入 I 编码的欧氏距离，检索出前 k 个语言-程序对。HELPER 的提示还包含角色描述（“你是一个在……方面具有专长的有帮助的助手”）、任务描述（“你的任务是……”）以及程序生成需遵循的指南（“你应只使用 API 中的函数……”），这些通常会针对目标领域进行定制。

                                                                                                          HELPER (Sarch, 2023), using RGB input, estimates depth maps, object masks, and agent egomotion at each timestep. This facilitates the creation and upkeep of a 3D occupancy map and an object memory database, essential for obstacle navigation and object tracking. Object detection in each frame leads to instance aggregation based on 3D centroid proximity, with each instance characterized by dynamic state attributes (e.g., cooked, sliced, dirty). When an action fails, a Vision-Language Model (CLIP (Radford, 2021)) provides feedback, prompting the LLM to re-plan. For objects not present in the map, the LLM suggests areas for HELPER to search (e.g., "near the sink"). HELPER (Sarch, 2023) 使用 RGB 输入，在每个时间步估计深度图、物体掩码和智能体自运动。这有助于创建并维护 3D 占据图和物体记忆数据库，这对障碍导航和物体跟踪至关重要。每帧中的物体检测会基于 3D 质心邻近性进行实例聚合，每个实例由动态状态属性表征（如 cooked、sliced、dirty）。当动作失败时，视觉语言模型（CLIP (Radford, 2021)）会提供反馈，促使 LLM 重新规划。对于地图中不存在的物体，LLM 会建议 HELPER 去搜索的区域（例如“水槽附近”）。

                                                                                                          2.2 UNIFIED MEMORY-AUGMENTED PROMPTING

                                                                                                          2.2 统一的记忆增强式提示

                                                                                                          We explore two ways to expand HELPER to work across four domains, either through prompt retrieval (Section 2.2.1) or through a shared example memory (Section 2.2.2). 我们探索两种扩展 HELPER 以覆盖四个领域的方法：要么通过提示检索（第 2.2.1 节），要么通过共享的示例记忆（第 2.2.2 节）。

                                                                                                          2.2.1 PROMPT RETRIEVAL

                                                                                                          2.2.1 提示检索

                                                                                                          Given an input language instruction I ,the prompt retrieval agent HELPER- XP retrieves a specialized prompt template P and an associated set of specialized examples E . Each specialized prompt template contains role descriptions,task instructions and guidelines tailored to each domain, namely, dialogue-based task completion (based on TEACh (Padmakumar, 2021)), instruction following from natural language (based on ALFRED (Shridhar, 2020)), instruction following with active question asking (based on DialFRED (Gao, 2022)), or tidying up rooms (based on Tidy Task (Sarch, 2022)). For retrieval, a query is generated from the input instruction by encoding the instruction into an embedding vector using a pre-trained language model (Greene, 2022). The query retrieves the closest key from memory, where each key represents the language encodings of each prompt template and example text, as shown in Figure 2. The top- k in-context examples are further retrieved from the specialized set of examples associated with the retrieved prompt and added to the retrieved prompt template, and the resulting prompt is used for LLM's program generation. 给定一个输入语言指令 I，提示检索代理 HELPER-XP 检索一个专门的提示模板 P 及其对应的一组专门示例 E。每个专门提示模板都包含针对各个领域定制的角色描述、任务说明和指导方针，即：基于对话的任务完成（基于 TEACh (Padmakumar, 2021)）、从自然语言中遵循指令（基于 ALFRED (Shridhar, 2020)）、带主动提问的指令遵循（基于 DialFRED (Gao, 2022)），或整理房间（基于 Tidy Task (Sarch, 2022)）。在检索时，会通过使用预训练语言模型 (Greene, 2022) 将指令编码为嵌入向量，由输入指令生成查询。该查询从记忆中检索最接近的键，其中每个键代表每个提示模板和示例文本的语言编码，如图 2 所示。来自与检索到的提示相关联的专门示例集中的前 k 个上下文示例会进一步被检索出来，并添加到检索到的提示模板中，最终得到的提示用于 LLM 的程序生成。

                                                                                                           

                                                                                                          Figure 2: Illustration of the shared example memory (HELPER-X S ; top) and the prompt retrieval (HELPER-X P ; bottom). The memory is shared across domains in both versions, allowing language and task inputs from any of the domains. 图 2：共享示例记忆（HELPER-X S；上）与提示检索（HELPER-X P；下）的示意图。该记忆在两个版本中都跨领域共享，使来自任一领域的语言和任务输入都能被使用。

                                                                                                          2.2.2 SHARED EXAMPLE MEMORY

                                                                                                          2.2.2 共享示例记忆

                                                                                                          Given an input language instruction I ,the shared example memory agent HELPER- XS retrieves a set of in-context examples from a shared memory that includes in-context examples from all domains considered. These examples are added to a domain-agnostic prompt template that does not have a specialized role description, task instructions or guidelines for any single domain. A query is generated from the input instruction by encoding it into an embedding vector using a pre-trained language model (Greene, 2022). The keys represent encodings of each in-context example language in the shared memory. The query embedding retrieves the top- k nearest neighbors keys and their values. These are added to the prompt as relevant in-context examples for LLM program generation, as shown in Figure 2. 给定一条输入语言指令 I ，共享示例记忆智能体 HELPER- XS 会从一个共享记忆中检索一组上下文示例。该共享记忆包含了在所考虑的所有领域中的上下文示例。然后，这些示例会被加入到一个领域无关的提示模板中，该模板不包含任何专门的角色说明，也不针对单一领域提供任务指令或准则。通过使用预训练语言模型将输入指令编码成嵌入向量（Greene, 2022），生成一个查询。键表示共享记忆中每个上下文示例语言的编码。查询嵌入会检索 top- k 个最近邻键及其对应的值。随后，这些内容会作为与 LLM 程序生成相关的上下文示例加入提示中，如图 2 所示。

                                                                                                          2.2.3 QUESTION ASKING API

                                                                                                          2.2.3 提问 API

                                                                                                          A natural limitation of asking questions in a simulator is that only certain types of questions can be understood and answered. We constrained the set of possible questions asked by the agent by defining an API of available questions in the Dial-FRED (Gao, 2022) benchmark and their arguments that HELPER-X can call on to gather more information. These include questions in three categories—Location, Appearance, and Direction-pertaining to the agent's next interaction object. Importantly, this API can be continuously expanded by adding an additional function to the question-asking API. 在模拟器中提问的一个自然限制是，只有某些类型的问题能够被理解和回答。我们通过在 Dial-FRED（Gao，2022）基准中定义可用问题及其参数的 API，来约束代理可提出的问题集合，HELPER-X 可调用这些问题以获取更多信息。这些问题包括三类——位置、外观和方向——涉及代理下一次交互的对象。重要的是，只需向提问 API 中添加一个额外函数，该 API 就可以持续扩展。

                                                                                                          Implementation details. We follow the network implementation of HELPER (Sarch, 2023). We use GPT-4-0613 (gpt, 2023) for text generation and text-embedding-ada-002 (Greene, 2022) for text embeddings. We use the SOLQ object detector (Dong, 2021) and ZoeDepth network (Bhat, 2023) for depth estimation from RGB input. We use k=3 for example retrieval. 实现细节。我们沿用 HELPER（Sarch，2023）的网络实现。我们使用 GPT-4-0613（gpt，2023）进行文本生成，使用 text-embedding-ada-002（Greene，2022）进行文本嵌入。我们使用 SOLQ 目标检测器（Dong，2021）和 ZoeDepth 网络（Bhat，2023）从 RGB 输入中估计深度。我们使用k=3进行示例检索。

                                                                                                          3 实验

                                                                                                          3.1 INFERRING AND EXECUTING ACTION PLANS FROM DIALOGUE

                                                                                                          3.1 从对话推断并执行行动计划

                                                                                                          Inferring and executing task plans from dialogue involves understanding dialogue segments and executing related instructions using the provided information in the dialogue. This task evaluates the agent's ability in understanding noisy and free-form conversations between two humans discussing about a household task. 从对话中推断并执行任务计划，需要理解对话片段，并使用对话中提供的信息来执行相应指令。该任务评估智能体理解两个人之间关于家务任务的嘈杂且随意的自由对话的能力。

                                                                                                          Dataset We use the TEACh benchmark (Padmakumar, 2021), which consists of over 3,000 dialogues focused on household tasks in the AI2-THOR environment Kolve (2017). We use the Trajectory from Dialogue (TfD) variant, where an agent, given a dialogue segment, must infer action sequences to fulfill tasks like making a coffee or preparing a salad. The training dataset contains 1482 expert demonstrations with associated dialogues. The evaluation includes 181 'seen' and 612 'unseen' episodes, with 'seen' having different object placements and states than in training, and 'unseen' also having different object instances and room environments. The agent receives an egocentric image at each step and selects actions to execute, such as pickup(X), turn_left(), etc. 数据集 我们使用 TEACh 基准（Padmakumar，2021），其中包含 3,000 多段聚焦家务任务的对话，任务环境为 AI2-THOR（Kolve，2017）。我们采用“从对话得到轨迹”（Trajectory from Dialogue，TfD）变体：智能体在给定一个对话片段的情况下，必须推断行动序列以完成诸如冲咖啡或准备沙拉等任务。训练数据集包含 1482 条带有对应对话的专家演示。评估包含 181 个“见过”（seen）与 612 个“未见过”（unseen）情节，其中“seen”的物体摆放与状态与训练中不同；“unseen”则在物体实例与房间环境上也有所不同。智能体在每一步都会获得一个第一人称图像，并选择要执行的动作，例如 pickup(X)、turn_left() 等。

                                                                                                          Baselines We consider two kinds of baselines: 1. Methods that supervise low-level or high-level action prediction from language and visual input using expert demonstrations in the training set (1482 in number) (Pashevich, 2021; Zheng, 2022; Min, 2021; Zhang, 2022), and 2. Methods that use a small amount of expert demonstrations for prompting pretrained LLM (Sarch, 2023). Specifically, HELPER and HELPER-X use 11 domain-specific examples. We additionally include a comparison to HELPER-ALF, which uses specialized prompts and examples for ALFRED. 基线 我们考虑两类基线：1）使用训练集中专家演示（共 1482 条）来监督从语言和视觉输入预测低层或高层动作的方法（Pashevich，2021；Zheng，2022；Min，2021；Zhang，2022）；2）使用少量专家演示来提示预训练的 LLM 的方法（Sarch，2023）。具体而言，HELPER 和 HELPER-X 使用 11 个面向领域的示例。我们还额外加入与 HELPER-ALF 的对比：该方法为 ALFRED 使用了专门的提示与示例。

                                                                                                          Evaluation Metrics We follow the TEACh evaluation metrics. Task success rate (SR) is a binary metric of whether all subtasks were successfully completed. Goal condition success rate (GC) quantifies the proportion of achieved goal conditions across all sessions. Both SR and GC have path-length weighted variants weighted by (path length of the expert trajectory) / (path length taken by the agent). 评估指标 我们遵循 TEACh 的评估指标。任务成功率（SR）是一个二元指标，表示所有子任务是否都成功完成。目标条件成功率（GC）衡量在所有会话中实现的目标条件所占比例。SR 和 GC 也都提供了按路径长度加权的变体，权重为（专家轨迹的路径长度）/（智能体所采取的路径长度）。

                                                                                                          3.2 FOLLOWING NATURAL LANGUAGE INSTRUCTIONS 3.2 遵循自然语言指令

                                                                                                          Natural language instruction following evaluates the agent's ability to carry out high-level instructions ("Rinse off a mug and place it in the coffee maker") and low-level ones ("Walk to the coffee maker on the right") provided by a human user. Importantly, the language and tasks in this evaluation differ from the ones in the TEACh benchmark (Section 3.1). 自然语言指令遵循评估智能体执行人类用户提供的高层指令（“冲洗一个杯子并把它放进咖啡机”）和低层指令（“走到右边的咖啡机旁”）的能力。重要的是，这一评估中的语言和任务不同于 TEACh 基准（第 3.1 节）中的内容。

                                                                                                          Dataset Te ALFRED (Shridhar, 2020) is a vision-and-language navigation benchmark designed for embodied agents to execute tasks in domestic settings from RGB sensory input. It includes seven task types across 207 environments, that involve 115 object types in 4,703 task instances, varying from simple object relocation to placing a heated item in a receptacle. The dataset includes detailed human-authored instructions and high-level goals, based on 21,023 expert demonstrations. It also comprises 820 'seen' and 821 'unseen' validation episodes. Agents receive egocentric RGB images at each step and select actions from a predefined set to progress, such as pickup(X), turn_left(), etc. 数据集 Te ALFRED（Shridhar，2020）是一个面向具身智能体的视觉-语言导航基准，旨在让其从 RGB 感知输入中执行家庭环境中的任务。它包含跨 207 个环境的 7 类任务，涉及 115 种物体类型和 4,703 个任务实例，任务从简单的物体搬运到将加热后的物品放入容器不等。该数据集基于 21,023 次专家演示，包含详细的人工编写指令和高层目标。它还包括 820 个“已见”和 821 个“未见”验证回合。智能体在每一步接收第一人称 RGB 图像，并从预定义动作集中选择动作以推进，例如 pickup(X)、turn_left() 等。

                                                                                                          Baselines Again, we consider two sets of baselines: those that supervise low-level or high-level action prediction using the expert demonstrations in the training set (Pashevich, 2021; Zheng, 2022; Min, 2021; Zhang, 2022; 2021; Song, 2022a; Blukis, 2022; Bhambri, 2023; Liu, 2022a; Murray, 2022; Inoue, 2022), and those that use a small amount of demonstrations (<= 100) (few-shot) (Sarch, 2023; Song, 2023; Brohan, 2023; Liu, 2023). The SayCan (Brohan, 2023), FILM (Min, 2021) (FS), and HLSM (Blukis, 2022) (FS) few shot baselines are adapted for the few-shot ALFRED setting by the authors of Song (2023), where the planning modules in FILM and HLSM are re-trained using only 100 demonstrations. We adapt HELPER (Sarch, 2023) as a baseline with specialized prompts and examples for ALFRED, as well as a comparison to HELPER-TEACh, which uses specialized prompts and examples for TEACh. HELPER and our HELPER-X model each use 7 domain-specific examples in their memory. 基线 同样，我们考虑两类基线：一类使用训练集中的专家演示来监督低层或高层动作预测（Pashevich，2021；Zheng，2022；Min，2021；Zhang，2022；2021；Song，2022a；Blukis，2022；Bhambri，2023；Liu，2022a；Murray，2022；Inoue，2022），另一类使用少量演示（<= 100）（few-shot）（Sarch，2023；Song，2023；Brohan，2023；Liu，2023）。SayCan（Brohan，2023）、FILM（Min，2021）（FS）和 HLSM（Blukis，2022）（FS）这几个 few-shot 基线由 Song（2023）的作者适配到 few-shot ALFRED 设置中，其中 FILM 和 HLSM 的规划模块仅用 100 次演示重新训练。我们将 HELPER（Sarch，2023）适配为一个基线，使用针对 ALFRED 的专门提示和示例，并将其与 HELPER-TEACh 进行比较，后者使用针对 TEACh 的专门提示和示例。HELPER 和我们的 HELPER-X 模型各自在其记忆中使用 7 个领域特定示例。

                                                                                                          Evaluation Metrics We follow the ALFRED evaluation metrics: 1. Task success rate (SR) and 2. Goal condition success rate (GC). These are defined the same as in TEACh (Section 3.1). 评估指标 我们遵循 ALFRED 的评估指标：1. 任务成功率（SR）和 2. 目标条件成功率（GC）。它们与 TEACh（第 3.1 节）中的定义相同。

                                                                                                          3.3 INSTRUCTION FOLLOWING WITH ASKING QUESTIONS 3.3 带提问的指令跟随

                                                                                                          Question asking instruction following allows the agent to choose to ask questions to an oracle to gain additional information to help it complete a task defined by an initial natural language instruction. 带提问的指令跟随让智能体可以选择向某个“问答者”提问，以获取更多信息，从而帮助它完成由初始自然语言指令所定义的任务。

                                                                                                          Dataset The DialFRED benchmark (Gao, 2022) enables an agent to query users while executing language instructions, utilizing user responses for task improvement. It features a human-annotated dataset with 53K relevant questions and answers,plus an oracle for responding to agent queries. Agents can ask questions in three categories-Location, Appearance, and Direction-pertaining to their next interaction object. The dataset covers 25 task types across 207 environments, 115 object types, and includes 'seen' and 'unseen' episodes. The agent receives egocentric RGB images at each step and selects actions from a set, like pickup(X), turn_left(), etc. This benchmark's instructions and tasks are distinct from TEACh and partially overlap with ALFRED, with significant modifications and 18 new task types. 数据集 DialFRED 基准（Gao, 2022）使智能体在执行语言指令时能够向用户提问，并利用用户的回答来改进任务。该数据集包含 53K 个与之相关的人类标注问题与答案，并提供一个用于回应智能体提问的问答者。智能体可以在三个类别中提问——位置、外观和方向——用于指向其下一次交互对象。该数据集覆盖 25 种任务类型、207 个环境、115 种物体类型，并包含“seen”和“unseen”的场景。智能体在每一步接收以视角为中心的 RGB 图像，并从一组动作中选择，例如 pickup(X)、turn_left() 等。该基准的指令与任务不同于 TEACh，并与 ALFRED 部分重叠，但进行了显著修改并新增了 18 种任务类型。

                                                                                                          Questioning Implementation To ask questions in the DialFRED task, we add to the question asking API functions to query the oracle in DialFRED (see Section 2.2.3). HELPER-X asks questions when it does not know the location of an object required for the task at hand. Unlike ALFRED, success in DialFRED requires interacting with a specific instance of an object class. To account for this, HELPER-X also asks questions to help disambiguate when it has seen multiple instances of the same object. 提问实现 为了在 DialFRED 任务中进行提问，我们在提问 API 函数中加入了用于在 DialFRED 中查询问答者的接口（见第 2.2.3 节）。当 HELPER-X 不知道完成当前任务所需物体的位置时，它会发起提问。与 ALFRED 不同，DialFRED 的成功需要与同一物体类别的特定实例进行交互。为此，HElPER-X 还会提问以在其看到多个相同物体实例时进行消歧。

                                                                                                          Baselines We compare with the baselines in the DialFRED paper (Gao, 2022), which includes a sequence-to-sequence architecture for choosing to ask a question, trained with reinforcement learning, and the Episodic Transformer architecture (Pashevich, 2021), trained with behavioral cloning. We adapt HELPER (Sarch, 2023) as a baseline with specialized prompts and examples for DialFRED, as well as our question asking API. We consider a few shot setting with each few shot model receiving 7 domain-specific examples. 对比方法 我们与 DialFRED 论文中的对比方法（Gao, 2022）进行比较，其中包括用于选择“是否提问”的序列到序列架构（使用强化学习训练）以及 Episodic Transformer 架构（Pashevich, 2021，使用行为克隆训练）。我们将 HELPER（Sarch, 2023）改造成对比基线：为 DialFRED 设计了专门的提示与示例，并同样提供我们的提问 API。我们采用少样本设定，每个少样本模型接收 7 个面向领域的示例。

                                                                                                          Evaluation Metrics We follow the conventions of the DialFRED benchmark. We use the Task success rate (SR) metric. This is defined the same as TEACh (Section 3.1). 评估指标 我们遵循 DialFRED 基准的惯例。使用任务成功率（SR）作为指标，其定义与 TEACh（第 3.1 节）相同。

                                                                                                          3.4 TIDYING UP USING SPATIAL COMMONSENSE REASONING 使用空间常识推理来收拾杂乱

                                                                                                          Tidying up involves figuring out where to place items without explicit instructions, relying on spatial commonsense to infer a proper location for an object. This task tests the agent's ability to use commonsense reasoning regarding contextual, object-object, and object-room spatial relations. 收拾整理需要在没有明确指令的情况下弄清物品该放在哪里，依靠空间常识推断物体的合适位置。该任务检验智能体运用常识推理来处理与语境、物体—物体以及物体—房间空间关系相关的能力。

                                                                                                          Dataset We evaluate on the Tidy Task (Sarch, 2022) benchmark, where the agent is spawned in a disorganized room, and must reposition objects to bring them to an organized tidy state. The dataset consists of 8000 training, 200 validation, and 100 testing messy configurations in 120 distinct scenes of bedrooms, living rooms, kitchens and bathrooms. At each time step, the agent obtains an egocentric RGB and depth image and must choose an action from a specified set to transition to the next step, such as pickup(X), turn_left(), etc. In this setup, the models are prompted to tidy up the room, given a set of objects that are out of place obtained using the visual detector from TIDEE (Sarch, 2022). 我们在 Tidy Task（Sarch，2022）基准上评估。智能体会被放置到一间杂乱的房间中，必须调整物体位置，使其恢复到整洁有序的状态。该数据集包含 120 个不同场景里的 8000 个训练、200 个验证以及 100 个测试的杂乱配置，场景类型包括卧室、客厅、厨房和浴室。每个时间步，智能体获得一个以自身视角为中心的 RGB 与深度图像，并需要从指定动作集合中选择一个动作以进入下一步，例如 pickup(X)、turn_left() 等。在该设置下，模型被提示对房间进行整理，并给定一组借助 TIDEE 的视觉检测器从 TIDEE 中识别得到的摆放不当物体。

                                                                                                          Baselines We compare against TIDEE (Sarch, 2022), which includes a graph neural network encoding common object arrangements. This is supervised in the training set of the Tidy Task to predict where a target object should be re-positioned to in the current scene. We adapt HELPER (Sarch, 2023) as a baseline with specialized prompts and examples for the Tidy Task. We additionally include a random receptacle baseline which chooses random receptacle placement locations for the out of place objects. We consider a few shot setting with each few shot model receiving 3 domain-specific examples. 基线 我们与 TIDEE（Sarch，2022）进行比较。TIDEE 包含一个图神经网络，用于编码常见的物体摆放结构。该方法在 Tidy Task 的训练集上进行监督学习，以预测目标物体在当前场景中应被重新放置到哪里。我们为 Tidy Task 将 HELPER（Sarch，2023）改造成基线，使用针对该任务的专门提示与示例。我们还加入一个随机容器基线：为摆放不当的物体随机选择容器放置位置。我们考虑少样本设置，每个少样本模型接收 3 个面向领域的示例。

                                                                                                          Evaluation Metrics We use the following evaluation metrics for the Tidy Task: Correctly Moved (CM) Average number of correctly moved objects that are out of place in the scene, and moved by the agent. Higher is better. Incorrectly Moved (IM) Average number of incorrectly moved objects that are not out of place, but were moved by the agent. Lower is better. Energy The "cleaniness" energy, where lower energy represents a higher likelihood of the room object configuration aligning with the configurations in the organized AI2THOR rooms. Following ProcThor (Deitke, 2022), for each receptacle object, the probability that each object type appears on its surface is computed across the AI2THOR scenes. See the Appendix for more details. 评估指标 我们对 Tidy Task 使用以下评估指标：正确移动（CM）场景中摆放不当物体中被正确移动的平均数量（并且由智能体移动）。数值越高越好。错误移动（IM）场景中原本并不摆放不当的物体中，被智能体移动但属于错误移动的平均数量。数值越低越好。能量（Energy），“洁净度”能量；能量越低，表示房间中物体配置与 AI2THOR 已整理房间中的配置越一致。按照 ProcThor（Deitke，2022），对每个容器物体，会在 AI2THOR 场景中计算每种物体类型出现在其表面的概率。更多细节见附录。

                                                                                                          4 CONCLUSION

                                                                                                          4 结论

                                                                                                          We introduce HELPER-X, an embodied agent that executes tasks from dialogue or language instructions, ask questions, and tides up rooms. HELPER-X has two variants,HELPER- XP and HELPER- XS ,enhancing HELPER’s memory capabilities. HELPER- XP retrieves domain-specific templates and examples for large language models,while HELPER- XS retrieves only examples for a domain-agnostic prompt template through a shared memory. Evaluation of HELPER-X in four domains: TEACh, ALFRED, DialFRED, and the Tidy Task, yields state-of-the-art performance in the few-shot example setting. Memory and API expansions we considered maintained or improved performance for the LLM, highlighting the effectiveness of memory-enhanced LLMs in building versatile, instructable agents. 我们提出 HELPER-X，这是一种具身代理，能够根据对话或语言指令执行任务：提问、并整理房间。HELPER-X 有两个变体：HELPER- XP 与 HELPER- XS，以增强 HELPER 的记忆能力。HELPER- XP 为大语言模型检索领域专属的模板与示例，而 HELPER- XS 通过共享记忆，仅为领域无关的提示模板检索示例。我们在四个领域对 HELPER-X 进行评估：TEACh、ALFRED、DialFRED 和整洁任务，在少样本示例设置中取得了最先进的性能。我们所考虑的记忆与 API 扩展维持或提升了 LLM 的表现，凸显了基于记忆增强的大语言模型在构建多才、可指令代理方面的有效性。

                                                                                                          5.1 MEMORY-AUGMENTED PROMPTING OF LARGE LANGUAGE MODELS

                                                                                                          5.1 通过记忆增强的大语言模型提示

                                                                                                          Recently, external memories have been instrumental in scaling language models Borgeaud (2021); Khandelwal (2019), overcoming the constraints of limited context windows in parametric transformers Wu (2022). They also facilitate knowledge storage in various forms such as entity mentions de Jong (2021), knowledge graphs Das (2022), and question-answer pairs Chen (2022). Retrieval-augmented generation (RAG) (Lewis, 2020; Mao, 2021) has been shown to significantly improve response quality in large language models (LLMs) by integrating external knowledge sources with the model's internal representations. In agent-based domains, memory-augmented prompting has enhanced task planning in embodied instructional contexts (Song, 2023; Sarch, 2023) and open-world gaming (Wang, 2023b;a; Majumder, 2023). Our model, HELPER-X, employs memory-augmented prompting across four benchmarks, demonstrating that memory expansion across related domains can maintain performance. 近年来，外部记忆在扩展语言模型 Borgeaud (2021); Khandelwal (2019) 方面发挥了关键作用，从而克服了参数化Transformer中受限的上下文窗口约束 Wu (2022)。它们还能以多种形式支持知识存储，例如实体提及 de Jong (2021)、知识图谱 Das (2022) 以及问答对 Chen (2022)。检索增强生成（RAG）(Lewis, 2020; Mao, 2021) 已被证明，通过将外部知识源与模型的内部表征相结合，能够显著提升大语言模型（LLMs）的回复质量。在基于智能体的任务中，记忆增强提示已提升具身指令场景下的任务规划能力 (Song, 2023; Sarch, 2023)，并改进开放世界游戏 (Wang, 2023b;a; Majumder, 2023)。我们的模型 HELPER-X 在四个基准上采用记忆增强提示，表明在相关领域中扩展记忆可以维持性能。

                                                                                                          Numerous benchmarks assess embodied vision-and-language tasks, with significant advancements in learning-based embodied AI agents across tasks like scene rearrangement Gan (2022); Weihs (2021); Batra (2020); Sarch (2022); Trabucco (2022), object-goal navigation Anderson (2018); Yang (2019); Wortsman (2019); Chaplot (2020); Gupta (2017); Chang (2020); Gervet (2023); Chang (2023), point-goal navigation and exploration Anderson (2018); Savva (2019); Wijmans (2020); Ramakrishnan (2020); Gupta (2017); Chen (2019); Chaplot (2019); Kumar (2021), embodied question answering Gordon (2018); Das (2018); Zhu (2023); Datta (2022); Das (2020); Gao (2022), instructional and image navigation (Ku, 2020; Krantz, 2023), audio-visual navigation (Chen, 2020), interactive dialogue and natural language instruction following (Yenamandra, 2023; Shridhar, 2020; Padmakumar, 2021; Gao, 2023), and embodied commonsense reasoning (Kant, 2022; Sarch, 2022; Wu, 2023). Interactive instruction benchmarks (e.g., ALFRED (Shridhar, 2020) and TEACh (Padmakumar, 2021)) require agents to follow natural language directives and dialogue, identifying objects in scenes via interaction masks. Variants like Dial-FRED (Gao, 2022) allow agent inquiries about objects and locations. Benchmarks such as TIDEE (Sarch, 2022) and HouseKeep (Kant, 2022) test agents' ability to tidy rooms using commonsense, without explicit object placement directives. Unlike most methods confined to a single domain, our work focuses on creating a multi-domain agent adept in dialogue-based task planning, natural language instruction following, asking questions for disambiguation of instructions, and tidying up scenes. Our method shows competitive performance across the four domains with a few task-specific demonstrations and without domain-specific weights, beyond the single image object detector. 大量基准评估具身视觉-语言任务，学习型具身AI智能体在场景重排Gan (2022); Weihs (2021); Batra (2020); Sarch (2022); Trabucco (2022)、对象目标导航Anderson (2018); Yang (2019); Wortsman (2019); Chaplot (2020); Gupta (2017); Chang (2020); Gervet (2023); Chang (2023)、点目标导航与探索Anderson (2018); Savva (2019); Wijmans (2020); Ramakrishnan (2020); Gupta (2017); Chen (2019); Chaplot (2019); Kumar (2021)、具身问答Gordon (2018); Das (2018); Zhu (2023); Datta (2022); Das (2020); Gao (2022)、指令与图像导航(Ku, 2020; Krantz, 2023)、视听导航(Chen, 2020)、交互式对话与自然语言指令遵循(Yenamandra, 2023; Shridhar, 2020; Padmakumar, 2021; Gao, 2023)、以及具身常识推理(Kant, 2022; Sarch, 2022; Wu, 2023)等任务上取得了显著进展。交互式指令基准（如ALFRED (Shridhar, 2020)和TEACh (Padmakumar, 2021)）要求智能体遵循自然语言指令和对话，通过交互掩码识别场景中的物体。Dial-FRED (Gao, 2022)等变体允许智能体就物体和位置进行询问。TIDEE (Sarch, 2022)和HouseKeep (Kant, 2022)等基准测试智能体利用常识整理房间的能力，而不提供明确的物体摆放指令。不同于大多数局限于单一领域的方法，我们的工作聚焦于构建一个擅长基于对话的任务规划、自然语言指令遵循、通过提问消歧指令以及整理场景的多领域智能体。我们的方法仅借助少量任务特定示范，并且除单张图像物体检测器外无需领域特定权重，即可在四个领域上表现出有竞争力的性能。

                                                                                                          Interactive vision-language embodied agent methods train distinct agents for each language-defined task, using large datasets from expert demonstrations (Min, 2021; Inoue, 2022; Zhang, 2022; Kim, 2023; Pashevich, 2021). Some approaches use these demonstrations for end-to-end network training to directly predict actions from observations (Pashevich, 2021; Gao, 2022; Zhang, 2021). Others employ modular methods, training planners to generate subgoals handled by specialized perception, manipulation, and obstacle avoidance modules (Min, 2021; Inoue, 2022; Blukis, 2022; Zheng, 2022; Kim, 2023; Bhambri, 2023; Liu, 2022a; Murray, 2022; Liu, 2023). However, these methods often over-specialize to specific datasets and tasks, limited by the training domain's language and task structure. In contrast, our method performs competitively across multiple benchmarks with minimal task-specific demonstrations and without needing domain-specific networks. 交互式视觉-语言具身智能体方法为每项语言定义的任务训练不同的智能体，使用来自专家示范的大规模数据集(Min, 2021; Inoue, 2022; Zhang, 2022; Kim, 2023; Pashevich, 2021)。一些方法利用这些示范进行端到端网络训练，直接根据观测预测动作(Pashevich, 2021; Gao, 2022; Zhang, 2021)。另一些则采用模块化方法，训练规划器生成子目标，由专门的感知、操作和避障模块处理(Min, 2021; Inoue, 2022; Blukis, 2022; Zheng, 2022; Kim, 2023; Bhambri, 2023; Liu, 2022a; Murray, 2022; Liu, 2023)。然而，这些方法往往对特定数据集和任务过度专门化，受训练领域的语言和任务结构限制。相比之下，我们的方法仅需极少的任务特定示范，且无需领域特定网络，就能在多个基准上取得有竞争力的表现。

                                                                                                          S1 LIMITATIONS

                                                                                                          S1 局限性

                                                                                                          Our model has the following limitations: 我们的模型有以下局限性：

                                                                                                          1. Task planning from multimodal input: Currently, our LLM receives the environment's state only in case of a failure, through VLM feedback. Integrating the visual state of the environment in a more direct way may dramatically increase the accuracy of predicted plans. This direction aligns with recent work (Wang, 2023b; Mu, 2023; Yang, 2023) that uses visual features as input to language models.

                                                                                                          2. 从多模态输入进行任务规划：目前，我们的LLM仅在通过VLM反馈发生失败时，才接收环境状态。以更直接的方式将环境的视觉状态融入模型，可能会显著提升所预测计划的准确性。这一方向与近期工作一致（Wang, 2023b; Mu, 2023; Yang, 2023），这些工作将视觉特征作为语言模型的输入。

                                                                                                          1. Cost of GPT-4: While GPT-4 is the most accurate Large Language Model, its high cost necessitates exploring alternatives such as open-source models, hardware optimization, model compression or distillation of its knowledge to smaller models, to reduce expenses.

                                                                                                          2. GPT-4的成本：尽管GPT-4是准确度最高的大语言模型，但其高昂成本要求我们探索替代方案，例如开源模型、硬件优化、模型压缩，或将其知识蒸馏到更小的模型，以降低开销。

                                                                                                          1. Manual Addition of Domains: Our model supports four domains with shared examples and prompts, but manual intervention is needed for adding significantly different domains and tasks. Future developments should focus on automating the detection and integration of out-of-domain inputs.

                                                                                                          2. 手动添加领域：我们的模型支持四个领域，并共享示例与提示，但对于差异显著的领域与任务，需要进行手动介入。未来的发展应聚焦于自动检测并整合域外输入。

                                                                                                          S6 SIMULATION ENVIRONMENT

                                                                                                          S6 模拟环境

                                                                                                          The TEACh dataset builds on the Ai2thor simulation environment (Kolve, 2017). At each time step the agent may choose from the following actions: Forward(), Backward(), Turn Left(), Turn Right(), Look Up(), Look Down(), Strafe Left(), Strafe Right(), Pickup(X), Place(X), Open(X), Close(X), ToggleOn(X), ToggleOff(X), Slice(X), and Pour(X), where X refers an object specified via a relative coordinate (x,y) on the egocentric RGB frame. Navigation actions move the agent in discrete steps. We rotate in the yaw direction by 90 degrees, and rotate in the pitch direction by 30 degrees. The RGB and depth sensors are at a resolution of 480x480, a field of view of 90 degrees, and lie at a height of 0.9015 meters. The agent's coordinates are parameterized by a single (x,y,z) coordinate triplet with x and z corresponding to movement in the horizontal plane and y reserved for the vertical direction. The TEACh benchmark allows a maximum of 1000 steps and 30 API failures per episode.                                                                                                           S7.1 SEMANTIC MAPPING AND PLANNING

                                                                                                          S7.1 语义映射与规划

                                                                                                           

                                                                                                          Obstacle map HELPER-X maintains a 2D overhead occupancy map of its environment ∈RH×W that it updates at each time step from the input RGB-D stream. The map is used for exploration and navigation in the environment. HELPER-X 维护一张二维俯视占用地图 ∈RH×W ，该地图反映其环境，并在每个时间步从输入 RGB-D 流中更新。该地图用于环境中的探索与导航。

                                                                                                          At every time step t ,we unproject the input depth maps using intrinsic and extrinsic information of the camera to obtain a 3D occupancy map registered to the coordinate frame of the agent, similar to earlier navigation agents Chaplot (2019). The 2D overhead maps of obstacles and free space are computed by projecting the 3D occupancy along the height direction at multiple height levels and summing. For each input RGB image, we run a SOLQ object segmentor (Dong, 2021) (pretrained on COCO Lin (2014) then finetuned on TEACh rooms) to localize each of 116 semantic object categories. For failure detection, we use a simple matching approach from Min (2021) to compare RGB pixel values before and after taking an action. 在每个时间步 t ，我们利用相机的内外参信息对输入深度图进行反投影，得到与智能体坐标系配准的三维占用地图，形式类似于先前的导航智能体 Chaplot（2019）。通过在多个高度层级对三维占用进行沿高度方向投影并求和，计算障碍物与可通行区域的二维俯视图。对每一帧输入 RGB 图像，我们运行一个 SOLQ 目标分割器（Dong，2021）（先在 COCO Lin（2014）上预训练，再在 TEACh 房间上微调）以定位 116 个语义物体类别中的每一个。为检测失败，我们使用一种来自 Min（2021）的简单匹配方法，比对执行动作前后 RGB 像素值。

The object centroids are expressed with respect to the coordinate system of the agent, and, similar to the semantic maps, updated over time using egomotion. We track previously detected objects by their 3D centroid C∈R3 . We estimate the centroid by taking the 3D point corresponding to the median depth within the segmentation mask and bring it to a common coordinate frame. We do a simple form of non-maximum suppression on the object memory, by comparing the euclidean distance of centroids in the memory to new detected centroids of the same category, and keep the one with the highest score if they fall within a distance threshold. 物体位置与状态跟踪 我们维护物体记忆：一个由物体检测得到的三维质心及其预测语义标签组成的列表 {[(X,Y,Z)i,ℓi∈{1…N}],i=1..K} ，其中 K 表示迄今为止检测到的物体数量。物体质心以智能体坐标系表示，并且与语义地图类似，会随时间利用自运动估计进行更新。我们通过三维质心 C∈R3 来跟踪先前检测到的物体。质心估计方法是：取分割掩码内深度的中位数对应的三维点，并将其变换到统一的坐标系。我们在物体记忆上做一种简化的非极大值抑制：比较记忆中质心的欧氏距离与同一类别新检测质心之间的距离阈值，在落入阈值的情况下保留得分最高的那一个。

                                                                                                          Exploration and path planning HELPER- X explores the scene using a classical mapping method. We take the initial position of the agent to be the center coordinate in the map. We rotate the agent in-place and use the observations to instantiate an initial map. Second, the agent incrementally completes the maps by randomly sampling an unexplored, traversible location based on the 2D occupancy map built so far, and then navigates to the sampled location, accumulating the new information into the maps at each time step.                                                                                                           To navigate to a goal location, we compute the geodesic distance to the goal from all map locations using graph search ? given the top-down occupancy map and the goal location in the map.
