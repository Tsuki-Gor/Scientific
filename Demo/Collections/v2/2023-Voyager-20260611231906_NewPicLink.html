                                                                                                                                                                      Abstract

                                                                                                                                                                      摘要

                                                                                                                                                                      We introduce VOYAGER, the first LLM-powered embodied lifelong learning agent in Minecraft that continuously explores the world, acquires diverse skills, and makes novel discoveries without human intervention. VOYAGER consists of three key components: 1) an automatic curriculum that maximizes exploration, 2) an ever-growing skill library of executable code for storing and retrieving complex behaviors, and 3) a new iterative prompting mechanism that incorporates environment feedback, execution errors, and self-verification for program improvement. VOYAGER interacts with GPT-4 via blackbox queries, which bypasses the need for model parameter fine-tuning. The skills developed by VOYAGER are temporally extended, interpretable, and compositional, which compounds the agent's abilities rapidly and alleviates catastrophic forgetting. Empirically, VOYAGER shows strong in-context lifelong learning capability and exhibits exceptional proficiency in playing Minecraft. It obtains 3.3× more unique items,travels 2.3× longer distances, and unlocks key tech tree milestones up to 15.3 × faster than prior SOTA. VOYAGER is able to utilize the learned skill library in a new Minecraft world to solve novel tasks from scratch, while other techniques struggle to generalize. 我们提出 VOYAGER，这是首个在《我的世界》中由大语言模型驱动的具身终身学习智能体。它在无人干预的情况下持续探索世界，习得多样化技能，并做出新的发现。VOYAGER 由三个关键组件构成：1）自动课程体系，最大化探索；2）不断增长的技能库，可存储与检索用于实现复杂行为的可执行代码；3）一种新的迭代式提示机制，将环境反馈、执行错误以及自我验证纳入以改进程序。VOYAGER 通过黑盒查询与 GPT-4 交互，从而无需对模型参数进行微调。VOYAGER 所发展出的技能具有时间上的延展性、可解释性与可组合性，这会迅速叠加其能力，并缓解灾难性遗忘。基于实证结果，VOYAGER 展现出强大的情境中终身学习能力，并在《我的世界》游戏表现上尤为出色。它获得 3.3× 更多独特物品，行进 2.3× 更长距离，并将解锁关键科技树里程碑的速度提升至比先前最先进方案快 15.3 ×。在一个全新的《我的世界》世界中，VOYAGER 能够直接调用所学技能库，从零开始解决新任务；而其他方法难以实现泛化。

                                                                                                                                                                      1 Introduction

                                                                                                                                                                      1 引言

                                                                                                                                                                      Building generally capable embodied agents that continuously explore, plan, and develop new skills in open-ended worlds is a grand challenge for the AI community [1-5]. Classical approaches employ reinforcement learning (RL) [6, 7] and imitation learning [8-10] that operate on primitive actions, which could be challenging for systematic exploration [11-15], interpretability [16-18], and generalization [19-21]. Recent advances in large language model (LLM) based agents harness the world knowledge encapsulated in pre-trained LLMs to generate consistent action plans or executable policies [16, 22, 19]. They are applied to embodied tasks like games and robotics [23, 27], as well as NLP tasks without embodiment [28-30]. However, these agents are not lifelong learners that can progressively acquire, update, accumulate, and transfer knowledge over extended time spans [31, 32]. 构建能够在开放式世界中持续探索、规划并发展新技能的通用具身智能体，是 AI 社区面临的一项重大挑战[1-5]。传统方法采用强化学习（RL）[6, 7]和模仿学习[8-10]，它们基于原始动作，因而在系统性探索[11-15]、可解释性[16-18]和泛化能力[19-21]方面都颇具挑战。近年来，基于大语言模型（LLM）的智能体利用预训练 LLM 中蕴含的世界知识，生成一致的行动计划或可执行策略[16, 22, 19]。它们已被应用于游戏和机器人等具身任务[23, 27]，以及不涉及具身的 NLP 任务[28-30]。然而，这些智能体并非能够在长时间跨度内逐步获取、更新、积累并迁移知识的终身学习者[31, 32]。

                                                                                                                                                                      Let us consider Minecraft as an example. Unlike most other games studied in AI [33, 34, 10], Minecraft does not impose a predefined end goal or a fixed storyline but rather provides a unique playground with endless possibilities [23]. Minecraft requires players to explore vast, procedurally generated 3D terrains and unlock a tech tree using gathered resources. Human players typically start by learning the basics, such as mining wood and cooking food, before advancing to more complex tasks like combating monsters and crafting diamond tools. We argue that an effective lifelong learning agent should have similar capabilities as human players: (1) propose suitable tasks based on its current skill level and world state, e.g., learn to harvest sand and cactus before iron if it finds itself in a desert rather than a forest; (2) refine skills based on environmental feedback and commit mastered skills to memory for future reuse in similar situations (e.g. fighting zombies is similar to fighting spiders); (3) continually explore the world and seek out new tasks in a self-driven manner. 以 Minecraft 为例。与 AI 中研究的大多数其他游戏不同[33, 34, 10]，Minecraft 不设预定义终点或固定剧情，而是提供了一个充满无限可能的独特游乐场[23]。Minecraft 要求玩家探索广阔的程序生成三维地形，并利用收集到的资源解锁科技树。人类玩家通常先学习基础操作，如伐木和烹饪食物，再逐步进阶到更复杂的任务，如对抗怪物和制作钻石工具。我们认为，一个有效的终身学习智能体应具备与人类玩家相似的能力：（1）根据当前技能水平和世界状态提出合适任务，例如当身处沙漠而非森林时，先学习采集沙子和仙人掌，再学铁；（2）根据环境反馈改进技能，并将已掌握的技能存入记忆，以便在未来相似情境中复用（例如，打僵尸与打蜘蛛相似）；（3）持续探索世界，并以自驱方式寻找新任务。

                                                                                                                                                                      Towards these goals, we introduce VOYAGER, the first LLM-powered embodied lifelong learning agent to drive exploration, master a wide range of skills, and make new discoveries continually without human intervention in Minecraft. VOYAGER is made possible through three key modules (Fig. 2): 1) an automatic curriculum that maximizes exploration; 2) a skill library for storing and retrieving complex behaviors; and 3) a new iterative prompting mechanism that generates executable code for embodied control. We opt to use code as the action space instead of low-level motor commands because programs can naturally represent temporally extended and compositional actions [16, 22], which are essential for many long-horizon tasks in Minecraft. VOYAGER interacts with a blackbox LLM (GPT-4 [35]) through prompting and in-context learning [36,38]. Our approach bypasses the need for model parameter access and explicit gradient-based training or finetuning. 为实现这些目标，我们提出 VOYAGER，这是首个由 LLM 驱动的具身终身学习智能体，能够在 Minecraft 中推动探索、掌握广泛技能，并在无人干预下持续发现新事物。VOYAGER 之所以可行，依赖于三个关键模块（图 2）：1）最大化探索的自动课程；2）用于存储和检索复杂行为的技能库；3）一种新的迭代提示机制，可生成用于具身控制的可执行代码。我们选择以代码而非低层运动指令作为动作空间，因为程序能够自然表示时间延展和组合性的动作[16, 22]，这对 Minecraft 中许多长时程任务至关重要。VOYAGER 通过提示和上下文学习[36,38]与一个黑盒 LLM（GPT-4 [35]）交互。我们的方法无需访问模型参数，也无需显式的基于梯度的训练或微调。

                                                                                                                                                                      More specifically, VOYAGER attempts to solve progressively harder tasks proposed by the automatic curriculum, which takes into account the exploration progress and the agent's state. The curriculum is generated by GPT-4 based on the overarching goal of "discovering as many diverse things as possible". This approach can be perceived as an in-context form of novelty search [39, 40]. Voyager incrementally builds a skill library by storing the action programs that help solve a task successfully. 更具体地说，VOYAGER 会尝试解决自动课程提出的、难度逐步增加的任务；该课程会考虑探索进展和智能体状态。课程由 GPT-4 基于“尽可能发现更多样化事物”这一总目标生成。这种方法可视为一种上下文中的新颖性搜索[39, 40]。Voyager 通过存储成功解决任务的行动程序，逐步构建技能库。

                                                                                                                                                                      Each program is indexed by the embedding of its description, which can be retrieved in similar situations in the future. Complex skills can be synthesized by composing simpler programs, which compounds VOYAGER's capabilities rapidly over time and alleviates catastrophic forgetting in other continual learning methods [31, 32]. 每个程序都由其描述的嵌入向量进行索引，可在未来相似情境中检索。复杂技能可以通过组合更简单的程序合成，这会随着时间推移迅速增强 VOYAGER 的能力，并缓解其他持续学习方法中的灾难性遗忘[31, 32]。

                                                                                                                                                                      However, LLMs struggle to produce the correct action code consistently in one shot [41]. To address this challenge, we propose an iterative prompting mechanism that: (1) executes the generated program to obtain observations from the Minecraft simulation (such as inventory listing and nearby creatures) and error trace from the code interpreter (if any); (2) incorporates the feedback into GPT-4's prompt for another round of code refinement; and (3) repeats the process until a self-verification module confirms the task completion, at which point we commit the program to the skill library (e.g., craftStoneShovel() and combatZombieWithSword()) and query the automatic curriculum for the next milestone (Fig. 2). 然而，LLM 很难一次性稳定地产生正确的动作代码[41]。为此，我们提出一种迭代提示机制：（1）执行生成的程序，从 Minecraft 模拟中获取观察信息（如背包列表和附近生物），以及代码解释器给出的错误追踪（如有）；（2）将反馈纳入 GPT-4 的提示中，进行新一轮代码改进；（3）重复该过程，直到自我验证模块确认任务完成，此时我们将程序提交至技能库（例如 craftStoneShovel() 和 combatZombieWithSword()），并向自动课程查询下一个里程碑（图 2）。

                                                                                                                                                                      Empirically, VOYAGER demonstrates strong in-context lifelong learning capabilities. It can construct an ever-growing skill library of action programs that are reusable, interpretable, and generalizable to novel tasks. We evaluate VOYAGER systematically against other LLM-based agent techniques (e.g., ReAct [29], Reflexion [30], AutoGPT [28]) in MineDojo [23], an open-source Minecraft AI framework. VOYAGER outperforms prior SOTA by obtaining 3.3× more unique items,unlocking key tech tree milestones up to 15.3× faster,and traversing 2.3× longer distances. We further demonstrate that VOYAGER is able to utilize the learned skill library in a new Minecraft world to solve novel tasks from scratch, while other methods struggle to generalize. 经验上，VOYAGER 展示出强大的情境内终身学习能力。它能够构建一个不断增长的技能库，由可复用、可解释且可泛化到新任务的行动程序组成。我们在 MineDojo [23]（一个开源的《我的世界》AI 框架）中，系统地将 VOYAGER 与其他基于 LLM 的智能体技术进行评估（例如 ReAct [29]、Reflexion [30]、AutoGPT [28]）。VOYAGER 通过获得 3.3× 更多的独特物品，15.3× 内解锁关键科技树里程碑更快，并且能穿越 2.3× 更远的距离，从而超过以往的最先进水平。我们进一步表明，VOYAGER 能够在一个新的《我的世界》世界中使用所学的技能库，从零开始解决新的任务，而其他方法难以实现泛化。

                                                                                                                                                                      2.1 Automatic Curriculum

                                                                                                                                                                      2.1 自动课程

                                                                                                                                                                      Embodied agents encounter a variety of objectives with different complexity levels in open-ended environments. An automatic curriculum offers numerous benefits for open-ended exploration, ensuring a challenging but manageable learning process, fostering curiosity-driven intrinsic motivation for agents to learn and explore, and encouraging the development of general and flexible problem-solving strategies [42-44]. Our automatic curriculum capitalizes on the internet-scale knowledge contained within GPT-4 by prompting it to provide a steady stream of new tasks or challenges. The curriculum unfolds in a bottom-up fashion, allowing for considerable adaptability and responsiveness to the exploration progress and the agent's current state (Fig. 3). As VOYAGER progresses to harder self-driven goals, it naturally learns a variety of skills, such as "mining a diamond". 具身智能体在开放式环境中会遇到复杂度不同的一系列目标。自动课程为开放式探索带来许多益处：确保学习过程既具挑战性又可管理；培养以好奇心驱动的内在动机，使智能体能够学习与探索；并促使形成通用且灵活的问题解决策略[42-44]。我们的自动课程利用GPT-4所包含的互联网规模知识，通过提示其持续生成新的任务或挑战。课程以自底向上的方式展开，使其能够对探索进度和智能体当前状态保持很强的适应性与响应性（图3）。随着VOYAGER逐步迈向更困难的自主目标，它自然会学会多种技能，比如“开采一颗钻石”。

                                                                                                                                                                      The input prompt to GPT-4 consists of several components: 送入GPT-4的输入提示由若干部分组成：

                                                                                                                                                                      (1) Directives encouraging diverse behaviors and imposing constraints, such as "My ultimate goal is to discover as many diverse things as possible ... The next task should not be too hard since I may not have the necessary resources or have learned enough skills to complete it yet."; (1) 鼓励多样化行为并施加约束的指令，例如“我的最终目标是尽可能发现多种多样的事物……下一个任务不应太难，因为我可能尚未具备必要资源，或还学得不够技能来完成它。”；

                                                                                                                                                                      (2) The agent's current state, including inventory, equipment, nearby blocks and entities, biome, time, health and hunger bars, and position; (2) 智能体的当前状态，包括物品栏、装备、附近的方块与实体、生物群系、时间、生命值与饥饿条，以及位置；

                                                                                                                                                                      (4) Additional context: We also leverage GPT-3.5 to self-ask questions based on the agent's current state and exploration progress and self-answer questions. We opt to use GPT-3.5 instead of GPT-4 for standard NLP tasks due to budgetary considerations. (4) 额外上下文：我们还会利用GPT-3.5基于智能体的当前状态与探索进度进行自问自答。出于预算考虑，我们选择使用GPT-3.5而不是GPT-4来执行标准NLP任务。

                                                                                                                                                                      2.2 Skill Library

                                                                                                                                                                      2.2 技能库

                                                                                                                                                                      With the automatic curriculum consistently proposing increasingly complex tasks, it is essential to have a skill library that serves as a basis for learning and evolution. Inspired by the generality, interpretability, and universality of programs [45], we represent each skill with executable code that scaffolds temporally extended actions for completing a specific task proposed by the automatic curriculum. 随着自动课程持续提出越来越复杂的任务，拥有一个可作为学习与演进基础的技能库至关重要。受程序的通用性、可解释性与普遍性的启发 [45]，我们用可执行代码来表征每项技能，用于搭建为完成自动课程提出的特定任务而进行的、具有时间延展性的行动脚手架。

                                                                                                                                                                      The input prompt to GPT-4 consists of the following components: 给 GPT-4 的输入提示包含以下组成部分：

Therefore, you should make it generic and reusable."; (1) 代码生成指南，例如“你的函数将被复用以构建更复杂的函数。因此，你应当使其具有通用性且可复用”；

                                                                                                                                                                      2.3 Iterative Prompting Mechanism

                                                                                                                                                                      2.3 迭代提示机制

                                                                                                                                                                      We introduce an iterative prompting mechanism for self-improvement through three types of feedback: 我们提出一种通过三类反馈实现自我改进的迭代提示机制：

                                                                                                                                                                      (1) Environment feedback, which illustrates the intermediate progress of program execution (Fig. 5, left). For example, "I cannot make an iron chestplate because I need: 7 more iron ingots" highlights the cause of failure in crafting an iron chestplate. We use bot.chat() inside control primitive APIs to generate environment feedback and prompt GPT-4 to use this function as well during code generation; (1) 环境反馈，展示程序执行的中间进展（图5左）。例如，“我无法制作铁胸甲，因为我还需要：7个铁锭”指出了制作铁胸甲失败的原因。我们在控制原语API中使用 bot.chat() 生成环境反馈，并提示 GPT-4 在代码生成时也使用该函数；

                                                                                                                                                                      (2) Execution errors from the program interpreter that reveal any invalid operations or syntax errors in programs, which are valuable for bug fixing (Fig. 5, right); (2) 来自程序解释器的执行错误，揭示程序中的无效操作或语法错误，这对修复bug很有价值（图5右）；

                                                                                                                                                                      (3) Self-verification for checking task success. Instead of manually coding success checkers for each new task proposed by the automatic curriculum, we instantiate another GPT-4 agent for self-verification. By providing VOYAGER's current state and the task to GPT-4, we ask it to act as a critic [47-49] and inform us whether the program achieves the task. In addition, if the task fails, it provides a critique by suggesting how to complete the task (Fig. 6). Hence, our self-verification is more comprehensive than self-reflection [30] by both checking success and reflecting on mistakes. (3) 用于检查任务是否成功的自我验证。我们不再为自动课程提出的每个新任务手动编写成功检查器，而是实例化另一个 GPT-4 代理进行自我验证。通过向 GPT-4 提供 VOYAGER 的当前状态和任务，我们让它充当批判者[47-49]，并告知我们程序是否完成了任务。此外，如果任务失败，它还会提出批评，建议如何完成任务（图6）。因此，我们的自我验证比 self-reflection [30] 更全面，因为它既检查成功，也反思错误。

                                                                                                                                                                      During each round of code generation, we execute the generated program to obtain environment feedback and execution errors from the code interpreter, which are incorporated into GPT-4's prompt for the next round of code refinement. This iterative process repeats until self-verification validates the task's completion, at which point we add this new skill to the skill library and ask the automatic curriculum for a new objective (Fig. 2). If the agent gets stuck after 4 rounds of code generation, then we query the curriculum for another task. This iterative prompting approach significantly improves program synthesis for embodied control, enabling VOYAGER to continuously acquire diverse skills without human intervention. 在每一轮代码生成中，我们都会执行生成的程序，以从代码解释器获得环境反馈和执行错误，并将其纳入 GPT-4 下一轮代码细化的提示中。这个迭代过程会持续进行，直到自我验证确认任务完成，此时我们把这项新技能加入技能库，并让自动课程提出一个新目标（图2）。如果智能体在4轮代码生成后仍然受阻，就会向课程请求另一个任务。这种迭代提示方法显著提升了具身控制中的程序合成能力，使 VOYAGER 能够在无需人工干预的情况下持续获得多样技能。

                                                                                                                                                                      We ablate 6 design choices (automatic curriculum, skill library, environment feedback, execution errors, self-verification, and GPT-4 for code generation) in VOYAGER and study their impact on exploration performance (see Appendix, Sec. B.3 for details of each ablated variant). Results are shown in Fig. 9. We highlight the key findings below: 我们在 VOYAGER 中移除 6 个设计选择（自动课程、技能库、环境反馈、执行错误、自我验证，以及用于代码生成的 GPT-4），并研究它们对探索表现的影响（详见附录，Sec. B.3，了解每种消融变体）。结果如图 9 所示。我们在下方强调关键发现：

                                                                                                                                                                      • Automatic curriculum is crucial for the agent's consistent progress. The discovered item count drops by 93% if the curriculum is replaced with a random one,because certain tasks may be too challenging if attempted out of order. On the other hand, a manually designed curriculum requires significant Minecraft-specific expertise, and does not take into account the agent's live situation. It falls short in the experimental results compared to our automatic curriculum.

                                                                                                                                                                      • 自动课程对智能体的稳定进展至关重要。若将课程替换为随机课程，发现物品数量将下降 93%，因为某些任务在未按顺序尝试时可能过于困难。另一方面，手工设计的课程需要大量特定于 Minecraft 的专业知识，且无法考虑智能体的实时情境。与我们的自动课程相比，该方法在实验结果上有所不足。

                                                                                                                                                                      • Voyager w/o skill library exhibits a tendency to plateau in the later stages. This underscores the pivotal role that the skill library plays in VOYAGER. It helps create more complex actions and steadily pushes the agent's boundaries by encouraging new skills to be built upon older ones.

                                                                                                                                                                      • 无技能库（Voyager w/o skill library）的模型在后期阶段呈现趋于停滞的倾向。这凸显了技能库在 VOYAGER 中的关键作用。它通过鼓励在更早的技能基础上构建新技能，帮助智能体形成更复杂的行动，并持续拓展其边界。

                                                                                                                                                                      3.5 Multimodal Feedback from Humans

                                                                                                                                                                      3.5 来自人类的多模态反馈

                                                                                                                                                                      VOYAGER does not currently support visual perception, because the available version of GPT-4 API is text-only at the time of this writing. However, VOYAGER has the potential to be augmented by multimodal perception models [58, 59] to achieve more impressive tasks. We demonstrate that given human feedback, VOYAGER is able to construct complex 3D structures in Minecraft, such as a Nether Portal and a house (Fig. 10). There are two ways to integrate human feedback: VOYAGER 目前不支持视觉感知，因为本文撰写时可用的 GPT-4 API 版本仅支持文本。不过，VOYAGER 有潜力通过多模态感知模型 [58, 59] 进行增强，以完成更令人印象深刻的任务。我们展示了在有人类反馈的情况下，VOYAGER 能够在 Minecraft 中构建复杂的 3D 结构，例如下界传送门和房屋（图 10）。整合人类反馈有两种方式：

                                                                                                                                                                      (1) Human as a critic (equivalent to VOYAGER's self-verification module): humans provide visual critique to VOYAGER, allowing it to modify the code from the previous round. This feedback is essential for correcting certain errors in the spatial details of a 3D structure that VOYAGER cannot perceive directly. (1) 人类作为批评者（相当于 VOYAGER 的自我验证模块）：人类向 VOYAGER 提供视觉批评，使其能够修改上一轮的代码。这种反馈对于纠正 VOYAGER 无法直接感知的 3D 结构空间细节中的某些错误至关重要。

                                                                                                                                                                      (2) Human as a curriculum (equivalent to VOYAGER's automatic curriculum module): humans break down a complex building task into smaller steps, guiding VOYAGER to complete them incrementally. This approach improves VOYAGER's ability to handle more sophisticated 3D construction tasks. (2) 人类作为课程设计者（相当于 VOYAGER 的自动课程模块）：人类将复杂的建造任务拆解为更小的步骤，引导 VOYAGER 逐步完成。这种方式提升了 VOYAGER 处理更复杂 3D 建造任务的能力。

                                                                                                                                                                      4 Limitations and Future Work

                                                                                                                                                                      4 局限与未来工作

                                                                                                                                                                      Cost. The GPT-4 API incurs significant costs. It is 15× more expensive than GPT-3.5. Nevertheless, VOYAGER requires the quantum leap in code generation quality from GPT-4 (Fig. 9), which GPT-3.5 and open-source LLMs cannot provide [60]. 成本。GPT-4 API 会产生显著费用。它比 GPT-3.5 15× 更昂贵。尽管如此，VOYAGER 需要来自 GPT-4 的代码生成质量“跨越式”提升（图 9），而 GPT-3.5 和开源 LLM 无法提供这种能力[60]。

                                                                                                                                                                      Inaccuracies. Despite the iterative prompting mechanism, there are still cases where the agent gets stuck and fails to generate the correct skill. The automatic curriculum has the flexibility to reattempt this task at a later time. Occasionally, self-verification module may also fail, such as not recognizing spider string as a success signal of beating a spider. 不准确性。尽管采用了迭代式提示机制，仍有一些情况下，智能体会卡住并未能生成正确技能。自动课程具备在之后再次尝试该任务的灵活性。有时，自我验证模块也可能失败，例如未能将蜘蛛字符串识别为击败蜘蛛的成功信号。

                                                                                                                                                                      Hallucinations. The automatic curriculum occasionally proposes unachievable tasks. For example, it may ask the agent to craft a "copper sword" or "copper chestplate", which are items that do not exist within the game. Hallucinations also occur during the code generation process. For instance, GPT-4 tends to use cobblestone as a fuel input, despite being an invalid fuel source in the game. Additionally, it may call functions absent in the provided control primitive APIs, leading to code execution errors. 幻觉。自动课程偶尔会提出无法完成的任务。例如，它可能要求智能体制作“铜剑”或“铜护胸甲”，而这些物品在游戏中并不存在。代码生成过程中也会出现幻觉。例如，GPT-4 往往会把圆石当作燃料输入，尽管在游戏中它不是有效燃料。此外，它可能调用所给控制原语 API 中不存在的函数，从而导致代码执行错误。

                                                                                                                                                                      We are confident that improvements in the GPT API models as well as novel techniques for finetuning open-source LLMs will overcome these limitations in the future. 我们相信，未来随着 GPT API 模型的改进，以及为开源 LLM 进行微调的全新技术，这些局限将会被克服。

                                                                                                                                                                      5 Related work

                                                                                                                                                                      5 相关工作

                                                                                                                                                                      Decision-making Agents in Minecraft. Minecraft is an open-ended 3D world with incredibly flexible game mechanics supporting a broad spectrum of activities. Built upon notable Minecraft benchmarks [23, 61-65], Minecraft learning algorithms can be divided into two categories: 1) Low-level controller: Many prior efforts leverage hierarchical reinforcement learning to learn from human demonstrations [66-68]. Kanitscheider et al. [14] design a curriculum based on success rates, but its objectives are limited to curated items. MineDojo [23] and VPT [8] utilize YouTube videos for large-scale pre-training. DreamerV3 [69], on the other hand, learns a world model to explore the environment and collect diamonds. 2) High-level planner: Volum et al. [70] leverage few-shot prompting with Codex [41] to generate executable policies, but they require additional human interaction. Recent works leverage LLMs as a high-level planner in Minecraft by decomposing a high-level task into several subgoals following Minecraft recipes [55, 53, 71], thus lacking full exploration flexibility. Like these latter works, VOYAGER also uses LLMs as a high-level planner by prompting GPT-4 and utilizes Mineflayer [52] as a low-level controller following Volum et al. [70]. Unlike prior works, VOYAGER employs an automatic curriculum that unfolds in a bottom-up manner, driven by curiosity, and therefore enables open-ended exploration. Minecraft 中的决策智能体。Minecraft 是一个开放式的 3D 世界，拥有极其灵活的游戏机制，能够支持广泛的活动类型。基于著名的 Minecraft 基准测试 [23, 61-65]，Minecraft 的学习算法可分为两类：1）低级控制器：许多既有工作采用分层强化学习，从人类示范中学习 [66-68]。Kanitscheider 等人 [14] 设计了基于成功率的课程，但其目标仅限于预先策划的物品。MineDojo [23] 和 VPT [8] 使用 YouTube 视频进行大规模预训练。DreamerV3 [69] 则学习一个世界模型，用于探索环境并收集钻石。2）高级规划器：Volum 等人 [70] 借助结合 Codex [41] 的 few-shot 提示生成可执行策略，但这需要额外的人类交互。近期工作通过遵循 Minecraft 食谱，将一个高层任务分解为多个子目标，从而在 Minecraft 中将 LLM 用作高级规划器，因此缺乏完整的探索灵活性。与这些工作类似，VOYAGER 也将 LLM 作为高级规划器，通过提示 GPT-4，并使用 Mineflayer [52] 作为低级控制器，且遵循 Volum 等人 [70]。不同于以往工作，VOYAGER 采用自动化课程，自下而上展开，由好奇心驱动，因此能够实现开放式探索。

                                                                                                                                                                      Large Language Models for Agent Planning. Inspired by the strong emergent capabilities of LLMs, such as zero-shot prompting and complex reasoning [72, 37, 38, 36, 73, 74], embodied agent research [75-78] has witnessed a significant increase in the utilization of LLMs for planning purposes. Recent efforts can be roughly classified into two groups. 1) Large language models for robot learning: Many prior works apply LLMs to generate subgoals for robot planning [27, 27, 25, 79, 80]. Inner Monologue [26] incorporates environment feedback for robot planning with LLMs. Code as Policies [16] and ProgPrompt [22] directly leverage LLMs to generate executable robot policies. VIMA [19] and PaLM-E [59] fine-tune pre-trained LLMs to support multimodal prompts. 2) Large language models for text agents: ReAct [29] leverages chain-of-thought prompting [46] and generates both reasoning traces and task-specific actions with LLMs. Reflexion [30] is built upon ReAct [29] with self-reflection to enhance reasoning. AutoGPT [28] is a popular tool that automates NLP tasks by crafting a curriculum of multiple subgoals for completing a high-level goal while incorporating ReAct [29]'s reasoning and acting loops. DERA [81] frames a task as a dialogue between two GPT-4 [35] agents. Generative Agents [82] leverages ChatGPT [50] to simulate human behaviors by storing agents' experiences as memories and retrieving those for planning, but its agent actions are not executable. SPRING [83] is a concurrent work that uses GPT-4 to extract game mechanics from game manuals, based on which it answers questions arranged in a directed acyclic graph and predicts the next action. All these works lack a skill library for developing more complex behaviors, which are crucial components for the success of VOYAGER in lifelong learning. 用于智能体规划的“大语言模型”。受益于 LLM 的强涌现能力，例如零样本提示与复杂推理 [72, 37, 38, 36, 73, 74]，具身智能体研究 [75-78] 显著增加了将 LLM 用于规划的用法。近期工作大致可分为两类。1）用于机器人学习的大语言模型：许多既有工作将 LLM 用于为机器人规划生成子目标 [27, 27, 25, 79, 80]。Inner Monologue [26] 将环境反馈融入基于 LLM 的机器人规划。Code as Policies [16] 与 ProgPrompt [22] 直接利用 LLM 生成可执行的机器人策略。VIMA [19] 和 PaLM-E [59] 对预训练 LLM 进行微调，以支持多模态提示。2）用于文本智能体的大语言模型：ReAct [29] 利用链式思维提示 [46]，并让 LLM 同时生成推理过程痕迹与面向任务的操作。Reflexion [30] 在 ReAct [29] 的基础上加入自我反思，以增强推理。AutoGPT [28] 是一个流行工具，它通过为完成高层目标构建包含多个子目标的课程，并结合 ReAct [29] 的推理与行动循环来自动化 NLP 任务。DERA [81] 将任务表述为两个 GPT-4 [35] 智能体之间的对话。Generative Agents [82] 借助 ChatGPT [50] 通过将智能体的经验存为记忆并进行检索来模拟人类行为，但其智能体动作不可执行。SPRING [83] 是一项并行工作，它使用 GPT-4 从游戏手册中提取游戏机制；基于这些信息，它以有向无环图排列问题，回答并预测下一步行动。所有这些工作都缺乏技能库来开发更复杂的行为，而这正是 VOYAGER 在终身学习中取得成功的关键组成部分。

                                                                                                                                                                      Code Generation with Execution. Code generation has been a longstanding challenge in NLP [41, 84, 85, 73, 37], with various works leveraging execution results to improve program synthesis. Execution-guided approaches leverage intermediate execution outcomes to guide program search [86-88]. Another line of research utilizes majority voting to choose candidates based on their execution performance [89, 90]. Additionally, LEVER [91] trains a verifier to distinguish and reject incorrect programs based on execution results. CLAIRIFY [92], on the other hand, generates code for planning chemistry experiments and makes use of a rule-based verifier to iteratively provide error feedback to LLMs. VOYAGER distinguishes itself from these works by integrating environment feedback, execution errors, and self-verification (to assess task success) into an iterative prompting mechanism for embodied control. 带执行的代码生成。代码生成长期以来都是 NLP 中的一项挑战 [41, 84, 85, 73, 37]，已有多种工作利用执行结果来改进程序合成。执行引导方法会利用中间执行结果来指导程序搜索 [86-88]。另一条研究路线则使用多数投票，根据候选程序的执行表现进行选择 [89, 90]。此外，LEVER [91] 训练一个验证器，用于根据执行结果区分并拒绝错误程序。CLAIRIFY [92] 则另一方面，为规划化学实验生成代码，并借助基于规则的验证器迭代向 LLM 提供错误反馈。VOYAGER 的不同之处在于，它将环境反馈、执行错误以及自我验证（用于评估任务是否成功）整合到一个用于具身控制的迭代式提示机制中，从而区别于这些工作。

                                                                                                                                                                      6 Conclusion

                                                                                                                                                                      6 结论

                                                                                                                                                                      In this work, we introduce VOYAGER, the first LLM-powered embodied lifelong learning agent, which leverages GPT-4 to explore the world continuously, develop increasingly sophisticated skills, and make new discoveries consistently without human intervention. VOYAGER exhibits superior performance in discovering novel items, unlocking the Minecraft tech tree, traversing diverse terrains, and applying its learned skill library to unseen tasks in a newly instantiated world. VOYAGER serves as a starting point to develop powerful generalist agents without tuning the model parameters. 在本研究中，我们提出 VOYAGER，这是首个由大语言模型驱动的具身终身学习智能体。它借助 GPT-4 持续探索世界，逐步发展更为精妙的技能，并在无需人工介入的情况下持续做出新发现。VOYAGER 在发现新颖物品、解锁 Minecraft 技术树、穿越多样地形，以及将其已习得的技能库应用到新生成世界中的未见任务方面表现出更优性能。VOYAGER 可作为构建强大通用型智能体的起点，而无需调整模型参数。

                                                                                                                                                                      7 Broader Impacts

                                                                                                                                                                      7 更广泛的影响

                                                                                                                                                                      Our research is conducted within Minecraft, a safe and harmless 3D video game environment. While VOYAGER is designed to be generally applicable to other domains, such as robotics, its application to physical robots would require additional attention and the implementation of safety constraints by humans to ensure responsible and secure deployment. 我们的研究在 Minecraft 这一安全且无害的 3D 电子游戏环境中进行。尽管 VOYAGER 的设计旨在可普遍适用于其他领域（例如机器人技术），但若将其应用于实体机器人，还需要额外关注，并由人类实施安全约束，以确保负责任且安全的部署。

                                                                                                                                                                      A.2 Prompting

                                                                                                                                                                      A.2 提示

                                                                                                                                                                      GPT-4 and GPT-3.5 offer users the ability to designate the role of each prompt message among three options: GPT-4 和 GPT-3.5 让用户能够在三个选项中为每条提示消息指定其角色：

                                                                                                                                                                      • System: A high-level instruction that guides the model behavior throughout the conversation. It sets the overall tone and objective for the interaction.

                                                                                                                                                                      • 系统：一种高层级指令，用于在对话过程中指导模型的行为。它为交互设定整体语气和目标。

                                                                                                                                                                      • User: A detailed instruction that guides the assistant for the next immediate response.

                                                                                                                                                                      • 用户：一条详细指令，用于指导助手对下一次即时回复。

                                                                                                                                                                      • Assistant: A response message generated the model.

                                                                                                                                                                      • 助手：由模型生成的回复消息。

                                                                                                                                                                      See https://platform.openai.com/docs/guides/chat/introduction for more details. 更多详情见 https://platform.openai.com/docs/guides/chat/introduction 。

                                                                                                                                                                      To save token usage, instead of engaging in multi-round conversations, we concatenate a system prompt and a user prompt to obtain each assistant's response. 为节省 token 用量，我们不进行多轮对话，而是将系统提示词与用户提示词拼接起来，以获得每个助手的回复。

                                                                                                                                                                      A.3 Automatic Curriculum

                                                                                                                                                                      A.3 自动课程

                                                                                                                                                                      A.3.1 Components in the Prompt

                                                                                                                                                                      A.3.1 提示词中的组成部分

                                                                                                                                                                      The input prompt to GPT-4 consists of several components: 提供给 GPT-4 的输入提示词由若干部分构成：

                                                                                                                                                                      (1) Directives encouraging diverse behaviors and imposing constraints (so that the proposed task is achievable and verifiable): See Sec. A.3.4 for the full prompt; (1) 促进多样化行为并施加约束的指令（以便所提出的任务可实现且可验证）：完整提示词见附录 A.3.4；

                                                                                                                                                                      (2) The agent's current state: (2) 智能体的当前状态：

                                                                                                                                                                      • Inventory: A dictionary of items with counts, for example, { 'cobblestone': 4, 'furnace': 1, 'stone_pickaxe': 1, 'oak_planks': 7, 'dirt': 6, 'wooden_pickaxe': 1, 'crafting_table': 1, 'raw_iron': 4, 'coal': 1};

                                                                                                                                                                      • 物品栏：物品及其数量的字典，例如 { 'cobblestone': 4, 'furnace': 1, 'stone_pickaxe': 1, 'oak_planks': 7, 'dirt': 6, 'wooden_pickaxe': 1, 'crafting_table': 1, 'raw_iron': 4, 'coal': 1}；

                                                                                                                                                                      • Equipment: Armors or weapons equipped by the agents;

                                                                                                                                                                      • 装备：智能体所穿戴的护甲或所携带的武器；

                                                                                                                                                                      • Nearby blocks: A set of block names within a 32-block distance to the agent, for example, 'dirt', 'water', 'spruce_planks', 'grass_block', 'dirt_path', 'sugar_cane', 'fern';

                                                                                                                                                                      • 附近方块：距离智能体 32 格范围内的一组方块名称，例如 'dirt'、'water'、'spruce_planks'、'grass_block'、'dirt_path'、'sugar_cane'、'fern'；

                                                                                                                                                                      • Other blocks that are recently seen: Blocks that are not nearby or in the inventory;

                                                                                                                                                                      • 最近看到的其他方块：不在附近或不在物品栏中的方块；

                                                                                                                                                                      • Nearby entities: A set of entity names within a 32-block distance to the agent, for example, 'pig', 'cat', 'villager', 'zombie';

                                                                                                                                                                      • 附近实体：距离智能体 32 格范围内的一组实体名称，例如 'pig'、'cat'、'villager'、'zombie'；

                                                                                                                                                                      • A list of chests that are seen by the agent: Chests are external containers where the agent can deposit items. If a chest is not opened before, its content is "Unknown". Otherwise, the items inside each chest are shown to the agent.

                                                                                                                                                                      B.2 Baselines

                                                                                                                                                                      B.2 基线

                                                                                                                                                                      ReAct [29] uses chain-of-thought prompting [46] by generating both reasoning traces and action plans with LLMs. We provide it with our environment feedback and the agent states as observations. ReAct undergoes one round of code generation from scratch, followed by three rounds of code refinement. This process is then repeated until the maximum prompting iteration is reached. ReAct [29] 通过链式思维提示 [46]，由 LLM 同时生成推理轨迹和行动计划。我们将环境反馈和智能体状态作为观测提供给它。ReAct 先进行一轮从零开始的代码生成，然后进行三轮代码精炼。之后会重复该过程，直到达到最大提示迭代次数。

                                                                                                                                                                      Reflexion [30] is built on top of ReAct [29] with self-reflection to infer more intuitive future actions. We provide it with environment feedback, the agent states, execution errors, and our self-verification module. Similar to ReAct, Reflexion undergoes one round of code generation from scratch, followed by three rounds of code refinement. This process is then repeated until the maximum prompting iteration is reached. Reflexion [30] 构建在 ReAct [29] 之上，并加入自我反思，以推断更直观的未来行动。我们为其提供环境反馈、智能体状态、执行错误以及我们的自我验证模块。与 ReAct 类似，Reflexion 先进行一轮从零开始的代码生成，然后进行三轮代码精炼。之后会重复该过程，直到达到最大提示迭代次数。

                                                                                                                                                                      AutoGPT [28] is a popular software tool that automates NLP tasks by decomposing a high-level goal into multiple subgoals and executing them in a ReAct-style loop. We re-implement AutoGPT by using GPT-4 to do task decomposition and provide it with the agent states, environment feedback, and execution errors as observations for subgoal execution. Compared with VOYAGER, AutoGPT lacks the skill library for accumulating knowledge, self-verification for assessing task success, and automatic curriculum for open-ended exploration. During each subgoal execution, if no execution error occurs, we consider the subgoal completed and proceed to the next one. Otherwise, we refine the program until three rounds of code refinement (equivalent to four rounds of code generation) are completed, and then move on to the next subgoal. If three consecutive subgoals do not result in acquiring a new item, we replan by rerunning the task decomposition. AutoGPT [28] 是一种流行的软件工具，它通过将高层目标分解为多个子目标，并以类似 ReAct 的循环执行来自动化 NLP 任务。我们使用 GPT-4 重新实现 AutoGPT：让它完成任务分解，并将智能体状态、环境反馈与执行错误作为观测提供给子目标执行。与 VOYAGER 相比，AutoGPT 缺少用于积累知识的技能库、用于评估任务成功的自我验证，以及用于开放式探索的自动课程。在每次子目标执行过程中，如果未发生执行错误，我们就认为该子目标已完成并进入下一个。否则，我们会不断精炼程序，直到完成三轮代码精炼（等价于四轮代码生成），然后再切换到下一个子目标。若连续三个子目标都未能获得新物品，我们将通过重新运行任务分解来进行重规划。

                                                                                                                                                                      B.3 Ablations
