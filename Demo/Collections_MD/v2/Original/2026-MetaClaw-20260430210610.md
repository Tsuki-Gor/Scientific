Large language model (LLM) agents have rapidly emerged as powerful assistants for complex, multi-step tasks, yet agents deployed in the wild remain largely static, trained once and served unchanged regardless of how user needs evolve. This creates a fundamental tension: they must serve users continuously without interruption, yet their capabilities grow stale as the task distribution drifts with real-world usage. On platforms such as OpenClaw, where a single agent connects to 20+ messaging channels and handles diverse, evolving workloads, existing approaches either store raw trajectories without distilling transferable behavioral knowledge, maintain static skill libraries disconnected from weight optimization, or incur service downtime during retraining. We present MetaClaw, a continual meta-learning framework that jointly maintains a base LLM policy and an evolving skill library of reusable behavioral instructions, improving both through two complementary mechanisms. Skill-driven fast adaptation analyzes failure trajectories and synthesizes new skills via an LLM evolver, taking effect immediately with zero service downtime. Opportunistic policy optimization performs gradient-based weight updates via cloud LoRA fine-tuning using RL with a process reward model, triggered only during user-inactive windows by the Opportunistic Meta-Learning Scheduler (OMLS), which monitors configurable sleep hours, system keyboard inactivity, and Google Calendar occupancy. The two mechanisms are mutually reinforcing: a better policy produces more informative failures for skill synthesis, and richer skills yield higher-reward trajectories for policy optimization. To prevent stale reward contamination, a skill generation versioning mechanism strictly separates support data (failure trajectories consumed by skill evolution) from query data (post-adaptation trajectories used for RL updates). Built on a proxy-based architecture, MetaClaw scales to production-size LLMs without a local GPU. Experiments on MetaClaw-Bench (934 questions, 44 simulated workdays) and AutoResearchClaw (23-stage autonomous research pipeline) demonstrate consistent improvements: skill-driven adaptation improves accuracy by up to 32% relative; the full pipeline advances Kimi-K2.5 from 21.4% to 40.6% accuracy (vs. GPT-5.2 baseline 41.1%) with an ${8.25} \times$ gain in end-to-end task completion; and skill injection alone improves AutoResearchClaw composite robustness by 18.3%.

## 1 Introduction
## 1 引言


Large language model (LLM) agents have demonstrated remarkable capabilities across complex tasks (Yao et al., 2022; Shinn et al., 2023), yet agents deployed in the wild remain largely static, trained once and served unchanged regardless of how the user's needs evolve (Zhang et al., 2025b; Naihin et al., 2023; Song et al.). Consider OpenClaw (OpenClaw, 2026), an open-source CLI agent platform connecting to 20+ messaging channels, where a single user's workload may shift from multi-step file system operations one week to multi-agent messaging workflows the next. As the task distribution drifts, a frozen model becomes increasingly misaligned with actual usage patterns, repeatedly failing on task types underrepresented during pretraining.
大语言模型（LLM）智能体在处理复杂任务时展现出了卓越的能力（Yao et al., 2022; Shinn et al., 2023），然而在野外环境中部署的智能体大多保持静态，无论用户需求如何演变，它们在训练一次后便不再改变（Zhang et al., 2025b; Naihin et al., 2023; Song et al.）。以 OpenClaw（OpenClaw, 2026）为例，这是一个连接 20 多个消息渠道的开源 CLI 智能体平台，单个用户的工作负载可能从一周的多步文件系统操作转变为下一周的多智能体消息工作流。随着任务分布的偏移，冻结的模型会逐渐与实际使用模式不匹配，导致在预训练期间未充分覆盖的任务类型上反复失败。

Our key observation is that two fundamentally different timescales of adaptation are in fact naturally complementary. Behavioral heuristics (e.g., "always verify a file path before reading," "confirm before destructive commands") can be distilled within seconds from a single failed conversation and injected immediately as skill instructions. Improving the model's underlying policy across diverse task types requires gradient-based optimization over many trajectories, on a timescale of minutes to hours. The two mechanisms are also mutually reinforcing: a better policy produces more informative failures for skill synthesis, and richer skills yield higher-reward trajectories for policy optimization. No existing system unifies these two forms of adaptation into a coherent framework that exploits this virtuous cycle.
我们的核心观察是，两种本质上不同的适应时间尺度实际上是天然互补的。行为启发式规则（例如“读取前务必验证文件路径”、“执行破坏性命令前需确认”）可以在单次失败对话后的几秒内提炼出来，并立即作为技能指令注入。而跨多种任务类型改进模型的基础策略，则需要对大量轨迹进行基于梯度的优化，时间尺度在几分钟到几小时不等。这两种机制也是相互促进的：更好的策略能为技能合成产生更具信息量的失败案例，而更丰富的技能则能为策略优化提供更高奖励的轨迹。目前尚无现有系统能将这两种适应形式统一为一个利用此良性循环的连贯框架。


We present MetaClaw, a continual meta-learning (Finn et al., 2017; Yao et al., 2021) framework that jointly maintains a base LLM policy and an evolving skill library of reusable behavioral instructions. The skill library serves a dual role: as a meta-parameter that accumulates behavioral knowledge across the task stream, and as an adaptation basis from which task-specific skills are retrieved at inference time. MetaClaw improves both components through two mechanisms. Skill-driven fast adaptation performs gradient-free skill evolution: an LLM analyzes failure trajectories and synthesizes new behavioral instructions (Xia et al., 2026) that take effect immediately with zero service downtime. Opportunistic policy optimization uses RL with a process reward model (PRM) (Zhang et al., 2025c) to update model weights via cloud (TML, 2026) LoRA fine-tuning (Hu et al., 2021), optimizing post-adaptation performance. Two design principles govern their coordination. First, when to run policy optimization: our Opportunistic Meta-Learning Scheduler (OMLS) monitors three idle signals, i.e., configurable sleep hours, system keyboard inactivity, and Google Calendar event occupancy, and triggers weight updates only during user-inactive windows, eliminating downtime. Second, which data to use: we distinguish support data (failure trajectories consumed by skill evolution) from query data (trajectories collected after new skills take effect). Only query data, reflecting the agent's post-adaptation behavior, is valid for RL; support data carries rewards conditioned on the old skill context and is excluded. Our skill generation versioning mechanism enforces this separation by stamping each trajectory with its skill generation index and flushing stale samples from the training buffer whenever skills evolve.
我们提出了 MetaClaw，这是一个持续元学习（Finn et al., 2017; Yao et al., 2021）框架，旨在共同维护基础 LLM 策略和一个可重用行为指令的演进技能库。技能库发挥双重作用：作为跨任务流积累行为知识的元参数，以及作为推理时检索任务特定技能的适应基础。MetaClaw 通过两种机制改进这两个组件。基于技能的快速适应执行无梯度技能演进：LLM 分析失败轨迹并合成新的行为指令（Xia et al., 2026），实现零服务停机时间的即时生效。机会主义策略优化利用带有过程奖励模型（PRM）（Zhang et al., 2025c）的强化学习，通过云端（TML, 2026）LoRA 微调（Hu et al., 2021）更新模型权重，从而优化适应后的性能。它们的协调遵循两条设计原则。首先，何时运行策略优化：我们的机会主义元学习调度器（OMLS）监控三种空闲信号，即用户可配置的睡眠时间、系统键盘不活跃状态以及 Google 日历事件占用情况，仅在用户不活跃窗口触发权重更新，从而消除停机时间。其次，使用哪些数据：我们将支持数据（被技能演进消耗的失败轨迹）与查询数据（新技能生效后收集的轨迹）区分开来。只有反映智能体适应后行为的查询数据才对强化学习有效；支持数据携带的是基于旧技能背景的奖励，因此被排除在外。我们的技能生成版本控制机制通过为每条轨迹打上技能生成索引戳，并在技能演进时清除训练缓冲区中的陈旧样本，从而强制执行这种分离。


In summary, our primary contribution is MetaClaw, a continual meta-learning framework that unifies skill-driven fast adaptation with opportunistic policy optimization, enabling deployed LLM agents to evolve continuously through a proxy-based architecture without requiring a local GPU. We evaluate on MetaClaw-Bench, a new benchmark of 934 questions over 44 simulated workdays, where each day forms a sequential, feedback-driven multi-round session of real CLI tasks (file editing, JSON structuring, shell scripting). Experiments with GPT-5.2 and Kimi-K2.5 show that skill-driven fast adaptation alone improves overall accuracy by up to 32.2% in relative terms; MetaClaw (Full) further advances Kimi-K2.5 from 21.4% to 40.6%, improves end-to-end task completion by ${8.25} \times$ on Part I and file-check completion by ${185}\%$ on Part II,and nearly closes the gap with GPT-5.2’s baseline. We further validate on AutoResearchClaw, a 23-stage autonomous research pipeline, where skill injection alone improves the composite robustness score by 18.3%, demonstrating cross-domain generalization of MetaClaw's adaptation mechanisms.
总之，我们的主要贡献是 MetaClaw，这是一个持续元学习框架，它将基于技能的快速适应与机会主义策略优化相结合，使部署的 LLM 智能体能够通过基于代理（proxy-based）的架构持续进化，而无需本地 GPU。我们在 MetaClaw-Bench 上进行了评估，这是一个包含 44 个模拟工作日、共 934 个问题的新基准测试，每一天都构成了一个序列化的、反馈驱动的真实 CLI 任务多轮会话（文件编辑、JSON 结构化、Shell 脚本编写）。使用 GPT-5.2 和 Kimi-K2.5 的实验表明，仅基于技能的快速适应就能使整体准确率相对提升高达 32.2%；MetaClaw（完整版）进一步将 Kimi-K2.5 的准确率从 21.4% 提升至 40.6%，将第一部分的端到端任务完成率提升了 ${8.25} \times$，将第二部分的文件检查完成率提升了 ${185}\%$，并几乎缩小了与 GPT-5.2 基准表现的差距。我们进一步在 AutoResearchClaw（一个 23 阶段的自主研究流水线）上进行了验证，仅技能注入就使综合鲁棒性得分提高了 18.3%，证明了 MetaClaw 适应机制的跨领域泛化能力。

## 2 问题设置


We consider a deployed CLI agent that serves a user over a stream of tasks ${\tau }_{1},{\tau }_{2},\ldots$ drawn from a non-stationary distribution ${p}_{t}\left( \tau \right)$ . Each task ${\tau }_{i}$ consists of a user instruction and environmental context (file system state, shell history,etc.),and the agent must produce a sequence of actions ${a}_{1 : T}$ to accomplish the task. The agent’s behavior at any point in time is fully determined by a meta-model:
我们考虑一个部署在 CLI 环境中的智能体，它通过任务流 ${\tau }_{1},{\tau }_{2},\ldots$ 为用户提供服务，这些任务服从非平稳分布 ${p}_{t}\left( \tau \right)$。每个任务 ${\tau }_{i}$ 由用户指令和环境上下文（文件系统状态、Shell 历史记录等）组成，智能体必须生成一系列动作 ${a}_{1 : T}$ 来完成任务。智能体在任何时间点的行为完全由元模型决定：


$$
\mathcal{M} = \left( {\theta ,\mathcal{S}}\right) , \tag{1}
$$



where $\theta$ denotes the parameters of the base LLM policy and $\mathcal{S} = \left\{  {{s}_{1},{s}_{2},\ldots ,{s}_{K}}\right\}$ is a library of skill instructions, i.e., concise, reusable behavioral directives injected into the agent's system prompt at inference time. Given a task $\tau$ ,the agent generates actions according to:
其中 $\theta$ 表示基础 LLM 策略的参数，$\mathcal{S} = \left\{  {{s}_{1},{s}_{2},\ldots ,{s}_{K}}\right\}$ 是技能指令库，即在推理时注入智能体系统提示词中的简洁、可重用的行为指令。给定任务 $\tau$，智能体根据以下方式生成动作：


$$
a \sim  {\pi }_{\theta }\left( {\cdot  \mid  \tau ,\operatorname{RETRIEVE}\left( {\mathcal{S},\tau }\right) }\right) , \tag{2}
$$



where RETRIEVE $\left( {\mathcal{S},\tau }\right)  \subseteq  \mathcal{S}$ selects the most relevant skills for the current task via embedding-based retrieval.
其中 RETRIEVE $\left( {\mathcal{S},\tau }\right)  \subseteq  \mathcal{S}$ 通过基于嵌入的检索为当前任务选择最相关的技能。


The meta-model $\mathcal{M}$ evolves over the task stream as the agent accumulates experience. We distinguish two types of trajectory data based on their role in this evolution. Support data ${\mathcal{D}}^{\text{ sup }}$ consists of trajectories whose failures drive adaptation of the skill library $\mathcal{S}$ ; these trajectories are consumed by the adaptation process and reflect pre-adaptation behavior. Query data ${\mathcal{D}}^{\text{ qry }}$ consists of trajectories collected after adaptation has taken effect; these reflect the agent's post-adaptation behavior and are used to optimize the policy parameters $\theta$ . Maintaining a strict separation between support and query data is essential: mixing them would cause $\theta$ to be optimized against stale reward signals that no longer reflect the agent's current capabilities.
元模型 $\mathcal{M}$ 随着智能体积累经验在任务流中不断进化。我们根据轨迹数据在进化中的作用将其分为两类。支持数据 ${\mathcal{D}}^{\text{ sup }}$ 由其失败案例驱动技能库 $\mathcal{S}$ 适应的轨迹组成；这些轨迹被适应过程消耗，反映了适应前的行为。查询数据 ${\mathcal{D}}^{\text{ qry }}$ 由适应生效后收集的轨迹组成；它们反映了智能体适应后的行为，并用于优化策略参数 $\theta$。保持支持数据和查询数据的严格分离至关重要：将它们混合会导致 $\theta$ 在过时的奖励信号上进行优化，而这些信号已不再反映智能体当前的能力。


The goal of MetaClaw is to continuously improve $\mathcal{M}$ over the task stream,not merely to solve each task in isolation, but to become progressively better at adapting to new tasks as they arrive. This positions MetaClaw as a continual meta-learning system: the agent learns from a non-stationary task stream while simultaneously improving its own adaptation capability.
MetaClaw 的目标是在任务流中持续改进 $\mathcal{M}$，不仅是为了孤立地解决每个任务，而是为了在面对新任务时能够逐步提高适应能力。这使 MetaClaw 成为一个持续元学习系统：智能体在从非平稳任务流中学习的同时，也在不断提升自身的适应能力。

### 3.1 Overview
### 3.1 概述


MetaClaw improves the meta-model $\mathcal{M} = \left( {\theta ,\mathcal{S}}\right)$ through two complementary mechanisms operating at different timescales (Figure 1). Skill-driven fast adaptation analyzes failure trajectories and synthesizes new skill instructions that are immediately injected into the agent’s prompt,evolving $\mathcal{S}$ without touching model weights. Opportunistic policy optimization uses post-adaptation trajectories to update $\theta$ via reinforcement learning, deferred to user-inactive windows by the Opportunistic Meta-Learning Scheduler (OMLS). A skill generation versioning mechanism ensures that policy optimization always trains on query data collected under the current skill library, preventing stale reward contamination from support data. The two mechanisms are mutually reinforcing: a better $\theta$ produces more informative failures for skill synthesis,and richer skills produce higher-reward trajectories for policy optimization. This virtuous cycle enables the system to learn to become better at adapting. The complete procedure is summarized in Algorithm 1.
MetaClaw 通过在不同时间尺度上运行的两种互补机制来改进元模型 $\mathcal{M} = \left( {\theta ,\mathcal{S}}\right)$（图 1）。基于技能的快速适应分析失败轨迹并合成新的技能指令，这些指令会立即注入到智能体的提示词中，从而在不触动模型权重的情况下进化 $\mathcal{S}$。机会主义策略优化利用适应后的轨迹，通过强化学习更新 $\theta$，并由机会主义元学习调度器（OMLS）推迟到用户不活跃的时间窗口执行。技能生成版本控制机制确保策略优化始终在当前技能库下收集的查询数据上进行训练，防止过时的奖励污染支持数据。这两种机制相互促进：更好的 $\theta$ 会产生更具信息量的失败案例以用于技能合成，而更丰富的技能会为策略优化产生更高奖励的轨迹。这种良性循环使系统能够学会如何更好地进行适应。完整流程总结在算法 1 中。


### 3.2 Skill-Driven Fast Adaptation
### 3.2 基于技能的快速适应


Given the current meta-model $\left( {\theta ,{\mathcal{S}}_{g}}\right)$ ,the agent executes tasks and collects trajectories. Trajectories that reveal failure modes form the support set ${\mathcal{D}}_{g}^{\text{ sup }}$ . Skill-driven adaptation evolves the skill library via a gradient-free experience distillation process:
给定当前元模型 $\left( {\theta ,{\mathcal{S}}_{g}}\right)$，智能体执行任务并收集轨迹。揭示失败模式的轨迹构成支持集 ${\mathcal{D}}_{g}^{\text{ sup }}$。基于技能的适应通过无梯度经验蒸馏过程演化技能库：


$$
{\mathcal{S}}_{g + 1} = {\mathcal{S}}_{g} \cup  \mathcal{E}\left( {{\mathcal{S}}_{g},{\mathcal{D}}_{g}^{\text{ sup }}}\right) , \tag{3}
$$



where $\mathcal{E}$ is a skill evolver,an LLM that analyzes failure trajectories and synthesizes new behavioral instructions. The index $g$ denotes the skill generation,incremented each time the library changes. This step modifies only $\mathcal{S}$ , leaving $\theta$ fixed,and takes effect immediately for all subsequent tasks. Because skill injection operates through the prompt rather than model parameters, fast adaptation incurs zero service downtime.
其中 $\mathcal{E}$ 是技能演化器，即一个分析失败轨迹并合成新行为指令的 LLM。索引 $g$ 表示技能生成版本，每次库发生变化时都会递增。此步骤仅修改 $\mathcal{S}$，保持 $\theta$ 不变，并对所有后续任务立即生效。由于技能注入是通过提示词而非模型参数进行的，因此快速适应不会产生服务停机时间。


This mechanism is gradient-free by design,not by approximation. The skill library $\mathcal{S}$ lives in a discrete natural-language space where gradient descent is ill-defined; LLM-based failure analysis is the natural adaptation mechanism for this space.
该机制在设计上就是无梯度的，而非近似处理。技能库 $\mathcal{S}$ 存在于离散的自然语言空间中，梯度下降在此处定义不明确；基于 LLM 的故障分析是该空间中自然的适应机制。


The skill library $\mathcal{S}$ plays a dual role in the learning structure. As a meta-parameter, $\mathcal{S}$ accumulates behavioral knowledge across the entire task stream,with each skill generation ${\mathcal{S}}_{g + 1} \supseteq  {\mathcal{S}}_{g}$ representing the system’s growing operational knowledge. As an adaptation basis, RETRIEVE $\left( {\mathcal{S},\tau }\right)$ extracts a task-specific subset at inference time, providing instant specialization without any parameter update. This dual character arises because natural-language instructions are inherently cross-task transferable: a skill distilled from one failure (e.g., "verify file path before reading") generalizes to all tasks involving file operations. Unlike systems where task-specific adaptations are ephemeral and discarded after each task, each adaptation episode in MetaClaw contributes lasting knowledge to the meta-model, making knowledge accumulation a feature rather than a side effect.
技能库 $\mathcal{S}$ 在学习结构中扮演双重角色。作为元参数，$\mathcal{S}$ 在整个任务流中积累行为知识，每个技能生成版本 ${\mathcal{S}}_{g + 1} \supseteq  {\mathcal{S}}_{g}$ 都代表了系统不断增长的操作知识。作为适应基础，RETRIEVE $\left( {\mathcal{S},\tau }\right)$ 在推理时提取特定任务的子集，无需任何参数更新即可提供即时专业化。这种双重特性源于自然语言指令天生具有跨任务迁移性：从一次失败中提炼出的技能（例如“读取前验证文件路径”）可推广至所有涉及文件操作的任务。与那些任务特定适应是短暂的、在任务结束后即被丢弃的系统不同，MetaClaw 中的每次适应片段都会为元模型贡献持久的知识，使知识积累成为一种特性而非副作用。


### 3.3 Opportunistic Policy Optimization
### 3.3 机会主义策略优化


After each skill-driven adaptation step, the agent continues serving tasks under the latest skill library. Because policy optimization is deferred to idle windows, the skill library may have advanced through several generations by the time training begins. Let ${g}^{ * }$ denote the current skill generation when a training window opens. The RL buffer $\mathcal{B}$ accumulates query trajectories across all post-adaptation generations,and policy optimization updates $\theta$ over this buffer:
在每次基于技能的适应步骤后，智能体继续在最新的技能库下处理任务。由于策略优化被推迟到空闲窗口，当训练开始时，技能库可能已经演进了多个版本。设 ${g}^{ * }$ 为训练窗口开启时的当前技能生成版本。RL 缓冲区 $\mathcal{B}$ 积累了所有适应后版本的查询轨迹，策略优化在此缓冲区上更新 $\theta$：


$$
{\theta }_{t + 1} = {\theta }_{t} + \alpha {\nabla }_{\theta }{\mathbb{E}}_{\left( {\tau ,\xi ,{g}^{\prime }}\right)  \sim  \mathcal{B}}\left\lbrack  {R\left( {{\pi }_{\theta }\left( {\cdot  \mid  \tau ,{\mathcal{S}}_{{g}^{\prime }}}\right) }\right) }\right\rbrack  , \tag{4}
$$



where ${g}^{\prime } \leq  {g}^{ * }$ is the skill generation under which each trajectory was collected,and $R$ is a process reward model (PRM) score. The versioning mechanism (Section 3.4) guarantees that $\mathcal{B}$ contains only query data,i.e.,every sample reflects post-adaptation behavior under its respective skill generation. Crucially, policy optimization does not optimize $\theta$ for raw task performance,but for how well the agent performs after skill adaptation. A better $\theta$ yields a meta-model from which skill-driven adaptation produces stronger post-adaptation behavior, resulting in an improved meta-model ${\mathcal{M}}^{\prime } = \left( {{\theta }_{t + 1},{\mathcal{S}}_{{g}^{ * }}}\right)$ .
其中 ${g}^{\prime } \leq  {g}^{ * }$ 是收集每条轨迹时所处的技能生成版本，$R$ 是过程奖励模型 (PRM) 分数。版本控制机制（第 3.4 节）确保 $\mathcal{B}$ 仅包含查询数据，即每个样本都反映了其各自技能生成版本下的适应后行为。至关重要的是，策略优化并非为了原始任务性能而优化 $\theta$，而是为了优化智能体在技能适应后的表现。更好的 $\theta$ 会产生一个元模型，基于该模型，基于技能的适应能产生更强的适应后行为，从而得到改进的元模型 ${\mathcal{M}}^{\prime } = \left( {{\theta }_{t + 1},{\mathcal{S}}_{{g}^{ * }}}\right)$。


In practice, policy optimization is realized via cloud LoRA fine-tuning using GRPO, deferred to idle windows by the Opportunistic Meta-Learning Scheduler (Section 3.5). Importantly, training is initiated only after the query buffer $\mathcal{B}$ has accumulated a sufficient number of trajectories; launching RL with too few samples leads to high-variance gradient estimates and unstable policy updates. This means policy optimization naturally lags behind skill-driven adaptation by days or longer, reinforcing the asymmetry between the two timescales: skills evolve continuously, while the policy improves in discrete, data-gated steps.
在实践中，策略优化通过使用 GRPO 的云端 LoRA 微调实现，并由机会主义元学习调度器（第 3.5 节）推迟到空闲窗口执行。重要的是，只有在查询缓冲区 $\mathcal{B}$ 积累了足够数量的轨迹后才会启动训练；使用过少的样本启动 RL 会导致高方差的梯度估计和不稳定的策略更新。这意味着策略优化在时间上自然滞后于基于技能的适应数天或更久，从而强化了两者之间的时间尺度不对称性：技能持续演化，而策略则在离散的、数据门控的步骤中改进。


### 3.4 Skill Generation Versioning
### 3.4 技能生成版本控制


The support-query separation defined in Section 2 must be enforced in MetaClaw's online setting, where tasks arrive sequentially and skill evolution is triggered asynchronously. Without a dedicated mechanism, support data can leak into the policy optimization buffer.
第 2 节中定义的支持-查询分离必须在 MetaClaw 的在线设置中强制执行，在该设置中任务按顺序到达，且技能演化是异步触发的。如果没有专门的机制，支持数据可能会泄露到策略优化缓冲区中。


The problem is concrete: a trajectory $\left( {{\tau }_{i},{\xi }_{i}}\right)$ that triggers skill evolution from ${\mathcal{S}}_{g}$ to ${\mathcal{S}}_{g + 1}$ carries a reward ${r}_{i}$ reflecting performance under ${\mathcal{S}}_{g}$ ,before the new skill existed. If this trajectory enters the RL buffer,policy optimization receives a gradient that penalizes $\theta$ for a failure that skill-driven adaptation has already corrected, optimizing for pre-adaptation rather than post-adaptation performance and violating the meta-learning objective in Eq. 4.
问题很具体：触发从 ${\mathcal{S}}_{g}$ 到 ${\mathcal{S}}_{g + 1}$ 技能演化的轨迹 $\left( {{\tau }_{i},{\xi }_{i}}\right)$，其携带的奖励 ${r}_{i}$ 反映的是新技能出现前在 ${\mathcal{S}}_{g}$ 下的表现。如果将该轨迹存入强化学习缓冲区，策略优化过程接收到的梯度会惩罚 $\theta$，而这本是技能驱动的自适应已经纠正过的失败。这导致优化目标偏向于自适应前的表现，而非自适应后的表现，从而违背了公式 4 中的元学习目标。


We enforce separation via a skill generation version ${g}_{i}$ stamped on each collected sample:
我们通过在每个采集样本上标记技能生成版本 ${g}_{i}$ 来强制实现这种分离：


- Support set ${\mathcal{D}}_{g}^{\text{ sup }}$ : trajectories collected under ${\mathcal{S}}_{g}$ whose failures trigger skill evolution ${\mathcal{S}}_{g} \rightarrow  {\mathcal{S}}_{g + 1}$ . These are consumed by the skill evolver and discarded from the RL buffer.
- 支持集 ${\mathcal{D}}_{g}^{\text{ sup }}$：在 ${\mathcal{S}}_{g}$ 下采集的轨迹，其失败触发了技能演化 ${\mathcal{S}}_{g} \rightarrow  {\mathcal{S}}_{g + 1}$。这些轨迹由技能演化器消耗，并从强化学习缓冲区中丢弃。


- Query set ${\mathcal{D}}_{g + 1}^{\text{ qry }}$ : trajectories collected after ${\mathcal{S}}_{g + 1}$ takes effect. Only these,reflecting the agent’s post-adaptation behavior, are eligible for policy optimization gradient updates.
- 查询集 ${\mathcal{D}}_{g + 1}^{\text{ qry }}$：在 ${\mathcal{S}}_{g + 1}$ 生效后采集的轨迹。只有这些反映智能体自适应后行为的轨迹，才有资格用于策略优化的梯度更新。


When the skill generation counter advances from $g$ to $g + 1$ ,the trainer flushes all samples with version $\leq  g$ from its buffer. This ensures policy optimization always updates $\theta$ with respect to the agent’s adapted behavior, preserving the integrity of the meta-learning structure.
当技能生成计数器从 $g$ 推进到 $g + 1$ 时，训练器会从缓冲区中清除所有版本为 $\leq  g$ 的样本。这确保了策略优化始终针对智能体的自适应行为更新 $\theta$，从而保持了元学习结构的完整性。


### 3.5 Opportunistic Meta-Learning Scheduler
### 3.5 机会主义元学习调度器


Policy optimization requires a model weight hot-swap upon completion, which briefly interrupts inference. In a deployed interactive system, this creates a tension: policy optimization must run periodically to improve $\theta$ ,but it must not degrade the user's experience.
策略优化需要在完成后进行模型权重热切换，这会短暂中断推理。在部署的交互式系统中，这产生了一种矛盾：策略优化必须定期运行以改进 $\theta$，但绝不能降低用户体验。


We introduce the Opportunistic Meta-Learning Scheduler (OMLS), a background daemon that defers policy optimization to periods when the user is not actively interacting with the agent. OMLS monitors three complementary idle signals:
我们引入了机会主义元学习调度器（OMLS），这是一个后台守护进程，将策略优化推迟到用户未与智能体主动交互的时段。OMLS 监控三个互补的空闲信号：


(1) Sleep window. The user configures a sleep schedule (e.g., 23:00-07:00). During this window, the system is guaranteed to be idle, providing the largest contiguous training block.
(1) 休眠窗口。用户配置休眠时间表（例如 23:00-07:00）。在此窗口期间，系统保证处于空闲状态，可提供最长的连续训练块。


(2) System inactivity. OMLS polls the operating system's input device idle timer (e.g., ioreg HIDIdleTime on macOS). If no keyboard or mouse activity is detected for $\delta$ minutes (default: 30),a training window opens. Upon renewed input, the trainer pauses gracefully via mid-batch checkpointing.
(2) 系统不活跃状态。OMLS 轮询操作系统的输入设备空闲计时器（例如 macOS 上的 ioreg HIDIdleTime）。如果 $\delta$ 分钟（默认：30）内未检测到键盘或鼠标活动，则开启训练窗口。一旦检测到新的输入，训练器通过批次间检查点优雅地暂停。


(3) Calendar-aware scheduling. OMLS queries the user's Google Calendar API. When the current time falls within a scheduled meeting, the user is presumed unavailable, opening an opportunistic training window. This is the most anticipatory of the three signals: it leverages the user's own schedule to predict idle periods proactively.
(3) 日历感知调度。OMLS 查询用户的 Google 日历 API。当当前时间处于预定会议期间时，假定用户不可用，从而开启机会主义训练窗口。这是三个信号中最具预见性的：它利用用户自己的日程安排来主动预测空闲时段。


A training window opens when any signal indicates user absence and closes when any signal indicates the user has returned. The RL trainer supports pause/resume across fragmented idle windows, accumulating gradient steps opportunistically without requiring a single long contiguous block.
当任何信号表明用户不在时，训练窗口开启；当任何信号表明用户返回时，训练窗口关闭。强化学习训练器支持在碎片化的空闲窗口间暂停/恢复，从而在无需单个长连续块的情况下，机会主义地累积梯度步数。

Per-day accuracy trends. Figure 2 visualizes per-day accuracy (3-day rolling average) for all five conditions. ## 5 相关工作


Skill-based and memory-augmented agents. A line of work augments LLM agents with external memory or reusable skill libraries to improve performance without modifying model weights (Shinn et al., 2023; Zhao et al., 2024; Fang et al., 2025; Tang et al., 2025; Ouyang et al., 2025; Chhikara et al., 2025; Liu et al., 2026a; Wang et al., 2024b). Reflexion (Shinn et al., 2023) stores verbal self-reflections in an episodic buffer, allowing the agent to avoid repeating past mistakes. Mem0 (Chhikara et al., 2025) and SimpleMem (Liu et al., 2026a) maintain longer-horizon memory through hierarchical retrieval. On the skill side, Voyager (Wang et al.) incrementally builds a library of executable code skills from successful episodes, while ExpeL (Zhao et al., 2024) and Agent-KB (Tang et al., 2025) distills cross-task experience into natural-language rules. A key limitation shared by these methods is that the skill library (or memory) is treated as a static artifact (Xia et al., 2026): it is never coordinated with weight-level optimization, and successful trajectories are reused indiscriminately without regard for whether the agent's behavior has changed since they were collected. MetaClaw addresses both gaps by coupling skill evolution with policy optimization through explicit support-query separation.
基于技能和记忆增强的智能体。一系列研究通过外部记忆或可重用技能库增强LLM智能体，以在不修改模型权重的情况下提升性能（Shinn et al., 2023; Zhao et al., 2024; Fang et al., 2025; Tang et al., 2025; Ouyang et al., 2025; Chhikara et al., 2025; Liu et al., 2026a; Wang et al., 2024b）。Reflexion（Shinn et al., 2023）将口头自我反思存储在情景缓冲区中，使智能体能避免重蹈覆辙。Mem0（Chhikara et al., 2025）和SimpleMem（Liu et al., 2026a）通过分层检索维持长程记忆。在技能方面，Voyager（Wang et al.）从成功片段中增量构建可执行代码技能库，而ExpeL（Zhao et al., 2024）和Agent-KB（Tang et al., 2025）将跨任务经验提炼为自然语言规则。这些方法共同的一个关键局限是，技能库（或记忆）被视为静态工件（Xia et al., 2026）：它从未与权重级优化协同，且成功轨迹被不加区分地重用，而不考虑智能体行为自收集以来是否已发生变化。MetaClaw通过明确的支持-查询分离，将技能演化与策略优化相结合，弥补了这两项空白。

Continual and meta-learning. Meta-learning (Finn et al., 2017; Nichol et al., 2018; Hospedales et al., 2021) frames learning as optimizing for fast adaptation to new tasks, typically in an offline episode-based setting. Meta-reinforcement learning extends this idea to sequential decision-making: ${\mathrm{{RL}}}^{2}$ (Duan et al.,2016) trains a recurrent policy whose hidden state implicitly encodes task context, PEARL (Rakelly et al., 2019) infers a probabilistic context variable for off-policy adaptation, and ProMP (Rothfuss et al., 2019) applies trust-region constraints at the meta-level. These methods demonstrate effective fast adaptation in robotic control and navigation, but operate on simple network architectures with low-dimensional action spaces and assume fixed offline task distributions. Continual learning (Kirkpatrick et al., 2017; Lopez-Paz and Ranzato, 2017; Chaudhry et al., 2019; Zenke et al., 2017; Wang et al., 2024a, 2022) studies sequential task adaptation without forgetting through regularization, replay, or architectural strategies, yet does not incorporate fast adaptation mechanisms at inference time. Online meta-learning approaches (Finn et al., 2019; Nagabandi et al., 2018; Harrison et al., 2020; Yao et al., 2020) relax the offline assumption and even handle task heterogeneity, but remain grounded in representation learning over simple networks. MetaClaw extends the meta-learning objective to a non-stationary stream of LLM agent tasks where fast adaptation is gradient-free (skill synthesis in discrete natural-language space) and slow adaptation is gradient-based (policy optimization via RL), with a versioning protocol that preserves the support-query structure in an online, asynchronous setting.
持续学习与元学习。元学习（Finn 等人，2017；Nichol 等人，2018；Hospedales 等人，2021）将学习过程构建为针对新任务的快速适应优化，通常在离线的情景化设置中进行。元强化学习将这一理念扩展至序列决策：${\mathrm{{RL}}}^{2}$（Duan 等人，2016）训练一种循环策略，其隐藏状态隐式编码了任务上下文；PEARL（Rakelly 等人，2019）推断出用于离线策略适应的概率上下文变量；ProMP（Rothfuss 等人，2019）在元层面应用了信赖域约束。这些方法在机器人控制和导航中展现了有效的快速适应能力，但均基于低维动作空间的简单网络架构，且假设任务分布在离线状态下是固定的。持续学习（Kirkpatrick 等人，2017；Lopez-Paz 和 Ranzato，2017；Chaudhry 等人，2019；Zenke 等人，2017；Wang 等人，2024a, 2022）研究如何在不遗忘的前提下通过正则化、重放或架构策略实现序列任务适应，但并未在推理阶段引入快速适应机制。在线元学习方法（Finn 等人，2019；Nagabandi 等人，2018；Harrison 等人，2020；Yao 等人，2020）放宽了离线假设，甚至能处理任务异构性，但仍局限于简单网络上的表征学习。MetaClaw 将元学习目标扩展至非平稳的 LLM 智能体任务流，其中快速适应是无梯度的（在离散自然语言空间中进行技能合成），而慢速适应是基于梯度的（通过强化学习进行策略优化），并采用版本控制协议，在在线异步环境中保留了支持集-查询集（support-query）结构。


## 6 Conclusion
## 6 结论


We presented MetaClaw, a continual meta-learning framework that enables deployed LLM agents to improve autonomously through normal usage. MetaClaw combines two complementary adaptation mechanisms operating at different timescales: fast, inference-time skill injection that distills reusable behavioral knowledge from failures, and slow, gradient-based policy optimization that refines the model during idle windows. Built on a lightweight proxy architecture, the system requires no local GPUs and integrates transparently with existing personal agents and LLM providers. Experiments on MetaClaw-Bench demonstrate consistent improvements across models and adaptation modes, with the full pipeline yielding the largest gains on both partial execution quality and end-to-end task completion. Evaluation on AutoResearchClaw further shows that skill injection generalizes to open-ended research pipelines without any gradient updates. A current limitation is that idle-window detection depends on user configuration, which may not generalize to all deployment environments. We believe MetaClaw establishes a principled foundation for agents that genuinely learn and evolve in the wild, simply by being used.
我们提出了 MetaClaw，这是一个持续元学习框架，使已部署的 LLM 智能体能够通过日常使用实现自主进化。MetaClaw 结合了两种在不同时间尺度上运行的互补适应机制：一是推理阶段的快速技能注入，从失败中提炼可复用的行为知识；二是基于梯度的慢速策略优化，在空闲窗口期对模型进行精炼。该系统基于轻量级代理架构，无需本地 GPU，并能与现有的个人智能体及 LLM 提供商无缝集成。在 MetaClaw-Bench 上的实验表明，该框架在不同模型和适应模式下均能实现持续改进，且完整流水线在部分执行质量和端到端任务完成度上均取得了最大增益。在 AutoResearchClaw 上的评估进一步证明，技能注入无需任何梯度更新即可泛化至开放式研究流水线。目前的局限在于空闲窗口检测依赖于用户配置，这可能无法推广至所有部署环境。我们相信，MetaClaw 为智能体在现实世界中仅通过使用即可实现真正的学习与进化奠定了原则性基础。
