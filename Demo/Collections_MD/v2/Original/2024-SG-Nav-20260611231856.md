## Abstract
## 摘要


In this paper, we propose a new framework for zero-shot object navigation. Existing zero-shot object navigation methods prompt LLM with the text of spatially closed objects, which lacks enough scene context for in-depth reasoning. To better preserve the information of environment and fully exploit the reasoning ability of LLM, we propose to represent the observed scene with 3D scene graph. The scene graph encodes the relationships between objects, groups and rooms with a LLM-friendly structure, for which we design a hierarchical chain-of-thought prompt to help LLM reason the goal location according to scene context by traversing the nodes and edges. Moreover, benefit from the scene graph representation, we further design a re-perception mechanism to empower the object navigation framework with the ability to correct perception error. We conduct extensive experiments on MP3D, HM3D and RoboTHOR environments, where SG-Nav surpasses previous state-of-the-art zero-shot methods by more than $\mathbf{{10}\% }$ SR on all benchmarks,while the decision process is explainable. To the best of our knowledge, SG-Nav is the first zero-shot method that achieves even higher performance than supervised object navigation methods on the challenging MP3D benchmark. Project page.
在本文中，我们提出一种用于零样本目标导航的新框架。现有的零样本目标导航方法通过提供空间上接近物体的文本来提示大语言模型，但缺乏足够的场景上下文，难以进行深入推理。为更好地保留环境信息，并充分发挥大语言模型的推理能力，我们提出用三维场景图来表征所观察的场景。场景图以适合大语言模型的结构编码物体、群组与房间之间的关系。为此，我们设计了分层的思维链提示，引导大语言模型通过遍历节点与边，结合场景上下文推理目标位置。此外，借助场景图表征，我们进一步设计了再感知机制，使目标导航框架具备纠正感知误差的能力。我们在 MP3D、HM3D 和 RoboTHOR 等环境中开展了大量实验。在所有基准上，SG-Nav 相比以往的最先进零样本方法提升超过 $\mathbf{{10}\% }$ SR，且决策过程可解释。据我们所知，SG-Nav 是首个在具有挑战性的 MP3D 基准上取得性能，甚至优于监督式目标导航方法的零样本方法。项目页面。


## 1 Introduction
## 1 引言


Object navigation, which requires an agent to navigate to an object specified by its category in unknown environment [3], is a fundamental problem in a wide range of robotic tasks. Thanks to the advancements in deep learning and reinforcement learning (RL), modular-based object navigation approaches [6, 32, 48] have achieved impressive performance in recent years, which caches the visual perception results of the RGB-D observation into semantic map and learns a policy to predict actions according to this map. However, conventional methods rely on time-consuming training in simulation environment, which only works well on specific datasets and can only handle limited categories of goal object.
物体导航要求智能体在未知环境中导航到由其类别指定的目标物体[3]，是一类广泛机器人任务中的基础问题。得益于深度学习与强化学习（RL）的进展，基于模块的物体导航方法[6, 32, 48]近年来取得了令人印象深刻的性能：它将RGB-D观测的视觉感知结果缓存到语义地图中，并学习一个策略依据该地图预测动作。然而，传统方法依赖耗时的仿真环境训练，这只对特定数据集效果良好，并且只能处理有限数量的目标类别。


In order to solve above limitations and enhance the generalization ability of object navigation, zero-shot object navigation has received increasing attention. In the zero-shot setting, the system does not require any training or finetuning before applied to real-world scenarios, and the goal category can be freely specified by text in an open-vocabulary manner. Since training data is unavailable, recent zero-shot object navigation methods [46, 52] leverage large language models (LLM) [36, 1] instead of RL agent to perform decision-making processes, where the extensive knowledge in LLM helps to address various common-sense reasoning problems in zero-shot. In these works, the LLM is required to select a frontier for the agent to explore the unobserved area. As LLM cannot perceive the environment visually, they select the categories of objects near each frontier to prompt LLM and rate the possibility of each frontier. While these approaches achieve zero-shot object navigation, there are still some drawbacks in their design: (1) they prompt LLM with only the text of nearby object categories, which is less informative and lacks scene context such as spatial relationship; (2) they let LLM directly output the possibility for each frontier, which is an abstract task and does not fully utilizes the reasoning ability of the LLM. As a result, their reasoning process is not explainable and the performance is far from satisfactory.
为了解决上述局限并提升物体导航的泛化能力，零样本物体导航受到越来越多的关注。在零样本设置下，系统在应用到现实场景前无需任何训练或微调，目标类别也可以以开放词汇的方式通过文本自由指定。由于没有训练数据，近期的零样本物体导航方法[46, 52]用大语言模型（LLM）[36, 1]来执行决策过程，而不是使用RL智能体，其中LLM中丰富的知识有助于解决零样本中各种常识推理问题。在这些工作中，LLM需要为智能体选择一个前沿以探索未观测区域。由于LLM无法进行视觉感知，它们会为每个前沿选择附近物体的类别来提示LLM，并对每个前沿的可能性进行评分。尽管这些方法实现了零样本物体导航，但其设计仍存在一些缺点：（1）它们只用附近物体类别的文本去提示LLM，信息量不足，且缺乏空间关系等场景上下文；（2）它们让LLM直接输出每个前沿的可能性，这是一种抽象任务，未充分利用LLM的推理能力。因此，其推理过程不可解释，性能也远未达到令人满意的水平。

In this paper, we propose a new object navigation framework namely SG-Nav, which fully exploits the reasoning ability of LLM for zero-shot navigation planning with high accuracy and fast speed. As shown in Figure 1, different from previous LLM-based object navigation framework that prompts LLM with only text of object categories and directly outputs probability, SG-Nav represents the complicated 3D environment in an online updated hierarchical 3D scene graph and utilizes it to hierarchically prompt LLM for reliable and explainable decision making. Benefit from the scene graph representation, we further design a graph-based re-perception mechanism to empower the object navigation framework with the ability to correct perception error. Specifically, we first construct an online 3D hierarchical scene graph along with the exploration of the agent. Since there are many levels and nodes in the scene graph, it is challenging to build the scene graph in real time. To solve this problem, we propose to densely connect the newly detected nodes to previous nodes in an incremental manner. We design a new form of prompt to control the computational complexity of this dense connecting process linear to the number of new nodes, which is followed by a pruning process to discard less informative edges. With the 3D scene graph, we divide it into several subgraphs and propose a hierarchical chain-of-thought to let LLM perceive the structural information in each subgraph for explainable probability prediction. Then we interpolate the probability of each subgraph to the frontiers for decision making, which can be further explained by summarizing the reasoning process for the most related subgraphs. Moreover, we also notice that previous object navigation framework fails when detecting out a false positive goal object. So we propose a graph-based re-perception mechanism to judge the credibility of the observed goal object by accumulating relevant probability of subgraphs. The agent will give up goal object with low credibility for robust navigation. We conduct extensive experiments on MP3D, HM3D and RoboTHOR environments. Our SG-Nav achieves state-of-the-art zero-shot performance on all benchmarks, surpassing previous top-performance zero-shot method by more than 10% in terms of SR. SG-Nav can also explain the reason for its decision at each step, making it more practical in human-agent interaction scenarios.
在本文中，我们提出一种新的物体导航框架SG-Nav，它以高精度和快速速度为零样本导航规划充分利用LLM的推理能力。如图1所示，区别于以往仅用物体类别文本提示LLM并直接输出概率的LLM物体导航框架，SG-Nav以在线更新的分层3D场景图来表示复杂的3D环境，并利用它对LLM进行分层提示，从而实现可靠且可解释的决策。得益于场景图表示，我们进一步设计了一种基于图的再感知机制，使物体导航框架具备纠正感知错误的能力。具体而言，我们首先在智能体探索过程中构建在线的3D分层场景图。由于场景图中存在大量层级与节点，要在真实时间中构建场景图具有挑战。为解决这一问题，我们以增量方式将新检测到的节点密集连接到先前节点。我们设计了一种新的提示形式，用以控制这种密集连接过程的计算复杂度，使其与新节点数量线性相关，随后再通过剪枝过程丢弃信息较少的边。有了3D场景图，我们将其划分为多个子图，并提出分层的链式思维（chain-of-thought），让LLM在每个子图中感知结构信息，以进行可解释的概率预测。接着，我们将各子图的概率插值到前沿上用于决策，这还可以通过总结与最相关子图的推理过程来进一步解释。此外，我们也注意到：当检测到一个错误的正样本目标物体时，先前的物体导航框架会失效。因此，我们提出一种基于图的再感知机制，通过累积相关子图的概率来判断所观测目标物体的可信度。对于可信度较低的目标物体，智能体会放弃它，以实现稳健导航。我们在MP3D、HM3D和RoboTHOR环境中进行了大量实验。我们的SG-Nav在所有基准上实现了最先进的零样本性能，相比以往表现最好的零样本方法在SR指标上提升超过10%。SG-Nav还能解释每一步决策的原因，使其在人机/人-智能体交互场景中更具可用性。

### 3.1 Zero-shot Object Navigation
### 3.1 零样本目标导航


Problem Definition. A mobile agent is tasked with navigating to an object $o$ specified by its category $c$ in an unknown environment. The agent receives streaming RGB-D video as input along with its exploration and decides where to go until it finds the goal object. Formally,at each time instant $t$ , the input to the agent is a posed RGB-D image and $o$ and the agent should execute an action $a \sim  \mathcal{A}$ , where $\mathcal{A}$ consists of move_forward,turn_left,turn_right and stop. The task is successful if the agent stops within ${d}_{s}$ meters of the goal object in less than $T$ steps. Otherwise it fails.
问题定义。一个移动智能体需要在未知环境中导航到一个由其类别 $c$ 指定的目标对象 $o$。智能体接收流式 RGB-D 视频作为输入，并结合其探索过程，决定下一步去哪里，直到找到目标对象。形式化地，在每个时刻 $t$，智能体的输入是一张已定位的 RGB-D 图像，同时 $o$ 指定目标对象，智能体应执行一个动作 $a \sim  \mathcal{A}$，其中 $\mathcal{A}$ 包括 move_forward、turn_left、turn_right 和 stop。若智能体在不超过 $T$ 步内，于目标对象 ${d}_{s}$ 米之内停止，则任务成功。否则失败。


Different from conventional object navigation which requires training in simulation environment with fixed vocabulary of categories $c$ ,we study zero-shot object navigation: the goal category $c$ can be freely specified with text in an open-vocabulary manner, and the navigation system does not require any training or finetuning, which is of great generalization ability.
不同于传统目标导航需要在模拟环境中训练，且类别词表 $c$ 固定，我们研究零样本目标导航：目标类别 $c$ 可用开放词汇文本自由指定，导航系统无需任何训练或微调，具有很强的泛化能力。


Overview. Since performing open-vocabulary object navigation without training is very challenging, we resort to large language model (LLM) [36, 1] to solve this problem. LLM demonstrates extensive knowledge and surprising generalization ability, which is a best choice to handle this zero-shot task. To make LLM aware of the visual observation of the environment, we propose to construct an online 3D scene graph, which contains rich context information about the environment and is suitable for interacting with LLM. At each step, we prompt the LLM with our 3D scene graph with hierarchical chain-of-thought and graph-based re-perception for decision making. The pipeline is shown in Figure 2
概述。由于在不训练的情况下进行开放词汇目标导航极具挑战，我们借助大语言模型（LLM）[36, 1] 来解决该问题。LLM 展示出广泛的知识与令人惊讶的泛化能力，是处理这一零样本任务的最佳选择。为使 LLM 感知环境的视觉观测，我们提出构建一个在线 3D 场景图，其中包含关于环境的丰富上下文信息，并适用于与 LLM 交互。每一步中，我们以分层链式推理与基于图的再感知方式，将 3D 场景图提供给 LLM 以用于决策。该流程如图 2 所示。


### 3.2 Online 3D Scene Graph Construction
### 3.2 在线3D场景图构建


In this subsection, we first provide formal definition of our 3D scene graph. Then we detail how to construct the scene graph in real time and online.
在本小节中，我们首先给出3D场景图的形式化定义。随后详细说明如何实时在线构建该场景图。


Hierarchical 3D Scene Graph. A 3D scene graph describe the 3D scene with nodes and edges between these nodes. We define three kinds of nodes: object, group and room, to represent the scenes in different graininess. Here group indicates a set of related objects, such as a dining table and its surrounding chairs, TV and TV cabinet. Edge can exist between any pair of nodes. If two nodes are not in the same level (e,g, chair and living room), the edge between them represents affiliation (i.e. this chair belongs to this living room). If two nodes are in the same level, the edge represents the relationship between them, which can be spatial relationship such as on the top of between close objects, or functional relationship between far away objects like a TV and its opposite sofa.
层次化3D场景图。3D场景图用节点及这些节点之间的边来描述3D场景。我们定义三类节点：对象、组和房间，以不同粒度表示场景。其中，组表示一组相关对象，例如餐桌及其周围的椅子、电视和电视柜。任意一对节点之间都可以存在边。若两个节点不在同一层级（例如椅子和客厅），则它们之间的边表示从属关系（即这把椅子属于这个客厅）。若两个节点处于同一层级，则边表示它们之间的关系，这种关系可以是空间关系，如相近物体之间的上方关系，也可以是功能关系，如电视与其对面的沙发这类相距较远物体之间的关系。

Incremental Updating and Pruning. Since there are many objects in the environment and the relationship between them are complicated, building the scene graph for the whole scene is time-consuming. To enable real-time online construction of the 3D scene graph, we propose to incrementally update it across frames. That is, at each time $t$ ,we register new nodes ${\mathcal{N}}_{t}$ from the RGB-D observation,and connect these nodes with the previous scene graph ${\mathcal{G}}_{t - 1}$ to acquire ${\mathcal{G}}_{t}$ .
增量更新与剪枝。由于环境中有许多对象，且它们之间的关系很复杂，为整个场景构建场景图耗时较长。为实现3D场景图的实时在线构建，我们提出跨帧增量更新。也就是说，在每个时刻$t$，我们从RGB-D观测中注册新节点${\mathcal{N}}_{t}$，并将这些节点与之前的场景图${\mathcal{G}}_{t - 1}$连接，以获得${\mathcal{G}}_{t}$。


We adopt different strategy to acquire different level of nodes. For object nodes, we leverage online open-vocabulary 3D instance segmentation method [12] to acquire the 3D instance. Formally, at each time $t$ ,the method detects and segments objects from the RGB-D observation and synchronously reconstruct the 3D point clouds for the environment. These newly detected objects will be matched with object nodes detected from the previous $t - 1$ frames and merged into them. Objects that do not match with any previous one will be registered as a new object node. Each object node has a semantic category and its confidence score. Then group nodes are computed based on all object nodes up to time $t$ . If two object nodes are connected by an edge and their categories are related,we register a group for them. This process will iterate until no new node can be assign to this group. We pre-define a dictionary to tell which pair of categories are related by LLM, which is detailed in Section 6.2 For room nodes, we follow the same paradigm for acquiring object nodes, where room is regarded as instance and thus we can prompt the open-vocabulary 3D instance segmentation model with the room type.
我们采用不同策略获取不同层级的节点。对于对象节点，我们利用在线开放词汇3D实例分割方法[12]获取3D实例。形式上，在每个时刻$t$，该方法从RGB-D观测中检测并分割对象，并同步重建环境的3D点云。这些新检测到的对象将与前$t - 1$帧中检测到的对象节点进行匹配并合并到其中。无法与任何先前对象匹配的对象将被注册为新的对象节点。每个对象节点都有一个语义类别及其置信度。随后，基于截至时刻$t$的所有对象节点计算组节点。如果两个对象节点由一条边连接且它们的类别相关，我们就为它们注册一个组。该过程将迭代进行，直到没有新节点可被分配到该组。我们预定义了一个字典，用于说明哪些类别对是由LLM判定为相关的，详见第6.2节。对于房间节点，我们采用与获取对象节点相同的范式，其中房间被视为实例，因此可以用房间类型提示开放词汇3D实例分割模型。


We also design different strategies to build edges between the nodes of different levels. We first connect high-level nodes with their affiliated low-level nodes. For room-object node pair, we think the object belongs to the room if the instance mask of this object on the point clouds is contained in the instance mask of this room. For room-group node pair, we connect them if all objects in this group belongs to the room. We do not additionally connect group and object nodes since the former is registered based on the latter.
我们还设计了不同策略，在不同层级节点之间构建边。我们首先将高层节点与其所属的低层节点相连。对于房间-对象节点对，如果该对象在点云上的实例掩码被包含在该房间的实例掩码中，我们就认为该对象属于该房间。对于房间-组节点对，如果该组中的所有对象都属于该房间，我们就将它们连接起来。我们不额外连接组节点和对象节点，因为前者是基于后者注册的。


Then we connect related object nodes to build intra-level edges, as shown in Figure 3 Note there is no intra-level edges between group and room nodes. For newly registered object nodes at time $t$ ,we first densely connect them with all object nodes,and then perform pruning to remove less informative edges. We combine $m$ newly registered nodes and $n$ previous nodes to form $m\left( {m + n}\right)$ edges. For each edge, we can prompt LLM with the categories of connected nodes and ask LLM to infer their relationship. However,traversing all $m\left( {m + n}\right)$ edges and feed prompts to LLM requires $\mathcal{O}\left( {m\left( {m + n}\right) }\right)$ computational complexity. To speed up the dense connecting process for real-time processing, we propose a new format of prompt: Predict the most likely relationships between these pairs of objects: $\left\lbrack  \left\{  {\left( {a,b}\right)  \mid  a \in  {S}_{n},b \in  {S}_{n} \cup  {S}_{p}}\right\}  \right\rbrack$ ,where ${\mathcal{S}}_{n}$ and ${\mathcal{S}}_{p}$ are the set of $m$ newly registered nodes and $n$ previous nodes. This enables LLM to generate relationships between all pairs of nodes in one-shot with much less computational cost, for which we provide theoretical derivation below.
然后我们连接相关对象节点构建层内边，如图3所示。注意，组节点与房间节点之间不存在层内边。对于时间$t$新注册的对象节点，我们首先将其与所有对象节点进行密集连接，然后进行剪枝以删除信息量较少的边。我们将$m$个新注册节点与$n$个先前节点结合，形成$m\left( {m + n}\right)$条边。对于每条边，我们可以向LLM输入连接节点的类别，并让LLM推断它们之间的关系。然而，遍历所有$m\left( {m + n}\right)$条边并向LLM输入提示会带来$\mathcal{O}\left( {m\left( {m + n}\right) }\right)$的计算复杂度。为加快实时处理中的密集连接过程，我们提出一种新的提示格式：预测这些对象对之间最可能的关系：$\left\lbrack  \left\{  {\left( {a,b}\right)  \mid  a \in  {S}_{n},b \in  {S}_{n} \cup  {S}_{p}}\right\}  \right\rbrack$，其中${\mathcal{S}}_{n}$和${\mathcal{S}}_{p}$分别是$m$个新注册节点和$n$个先前节点的集合。这使LLM能够以更低的计算成本一次性生成所有节点对之间的关系，下面我们给出其理论推导。

where $c$ is a small constant. Consequently,our method reduces the complexity from $\mathcal{O}\left( {m\left( {m + n}\right) }\right)$ to $\mathcal{O}\left( m\right)$ . In this way,the LLM can generate new edge proposals in fast speed.

We further prune the dense connected edges to make the 3D scene graph more precise. We divide newly generated relationships into two categories: long-range and short-range. The basis for this division is whether there is a RGB-D image containing the two connected objects. If true, this edge is short-range, for which we can feed the RGB image to a vision language model (VLM) like LLaVA [23] and ask whether this relationship exists. Nonexistent ones will be pruned. Otherwise the edge is a long-range one. We design two standards to validate it: (1) the line connecting two objects is unobstructed and parallel to the wall of room; (2) these two object nodes belong to the same room node. Only edges meet with both standards can be preserved.
我们进一步剪枝密集连接的边，使3D场景图更加精确。我们将新生成的关系分为两类：长程和短程。划分依据是是否存在包含这两个连接对象的RGB-D图像。如果存在，则该边为短程边，我们可以将RGB图像输入给LLaVA [23]等视觉语言模型（VLM），并询问该关系是否存在。不存在的边将被剪除。否则，该边为长程边。我们设计了两个标准来验证它：(1) 连接两个对象的线段无遮挡且与房间墙面平行；(2) 这两个对象节点属于同一个房间节点。只有同时满足这两个标准的边才能保留。


### 3.3 Prompting LLM with 3D Scene Graph
### 3.3 使用三维场景图来提示LLM


In this subsection, we describe how to prompt LLM with the 3D scene graph at each step for decision making. Since directly predicting $a$ is hard for LLM,we adopt Frontier-based Exploration (FBE) strategy to divide the decision into two part: (1) synchronous with the 3D scene graph, we build an online occupancy map to indicate explored, unexplored and occupied area in BEV, based on which we compute the frontiers of explored area. In order to further explore the environment, the agent must go through one of the frontier. So in high level, the LLM is only required to select one of the frontier which has the highest probability to find $o$ ; (2) once the frontier to be explore is chosen,we simply apply local policy like Fast Marching Method [34] for path planning and output $a$ .
在本小节中，我们介绍如何在每一步决策时使用三维场景图来提示LLM。由于直接预测 $a$ 对LLM来说很困难，我们采用基于前沿的探索（Frontier-based Exploration, FBE）策略，将决策分成两部分：（1）与三维场景图同步，我们构建在线占据地图，用于在BEV中标记已探索、未探索和已占据区域，并据此计算已探索区域的前沿。为进一步探索环境，智能体必须经过某个前沿。因此，从高层来看，LLM只需选择概率最高、最可能找到 $o$ 的前沿；（2）一旦选定要探索的前沿，我们就直接使用局部策略（如快速行进方法 Fast Marching Method [34]）进行路径规划，并输出 $a$ 。

Hierarchical Chain-of-thought Prompting. Different from previous methods [46, 52] that prompt LLM with only categories of objects near each frontier, we compute the probability ${P}^{fro}$ of each frontier by prompting LLM with our 3D scene graph. At time $t$ ,we divide the scene graph ${\mathcal{G}}_{t}$ into subgraphs, each of which is determined by an object node with all its parent nodes and other directly connected object nodes. For each subgraph,we predict ${P}^{sub}$ ,the possibility of goal appearing near this subgraph. Then the probability ${P}_{i}^{fro}$ of the $i$ -th frontier can be averaged by:
分层链式思维提示。不同于以往方法 [46, 52] 只在每个前沿附近提示目标类别，我们通过用我们的三维场景图来提示LLM，从而计算每个前沿的概率 ${P}^{fro}$ 。在时刻 $t$ ，我们将场景图 ${\mathcal{G}}_{t}$ 划分为若干子图，每个子图由一个目标节点及其所有父节点和其他所有直接相连的目标节点共同确定。对于每个子图，我们预测 ${P}^{sub}$ ——目标出现在该子图附近的可能性。随后，$i$ - 个前沿的概率 ${P}_{i}^{fro}$ 可以通过下式进行平均：


$$
{P}_{i}^{fro} = \mathop{\sum }\limits_{{j = 1}}^{M}\frac{{P}_{j}^{sub}}{{D}_{ij}}\left( {i \in  \{ 1,2,\ldots ,N\} }\right) \tag{2}
$$



where $M$ is the total number of subgraphs and ${D}_{ij}$ is the distance between the center of the $i$ -th frontier and the central object node of the $j$ -th subgraph.
其中，$M$ 是子图总数，${D}_{ij}$ 是第 $i$ 个前沿中心与第 $j$ 个子图中中心目标节点之间的距离。


In order to make the prediction of ${P}^{sub}$ accurate and explainable,we propose a hierarchical chain-of-thought method to prompt LLM, which fully exploits the structure information contained in the subgraphs. Specifically, for each subgraph, LLM is first prompted with Predict the most likely distance between [object] and [goal], where [object] means the category of the central object node. Then we prompt LLM to ask question related to the navigation goal, with the prompt: Ask questions about the [object] and [goal] for predicting their distance. After obtaining the questions about goal, we prompt LLM to answer those questions according to the information of subgraph, with the prompt: Given [nodes] and [edges] of subgraph, answer above [questions]. Therefore, from those answers we get all the necessary information required for inferring the position of goal. Finally, we prompt LLM to summarize the conversation and predict $P$ ,with the prompt Based on the above conversation, determine the most likely distance between this subgraph and [goal]. And ${P}^{\text{ sub }}$ can be simply computed by taking the inverse of the outputted distance. The LLM also summarizes the analysis process of the nearest 3 subgraphs to the selected frontier to explain the reason for decision.
为使对 ${P}^{sub}$ 的预测既准确又可解释，我们提出一种分层链式思维方法来提示LLM，它充分利用子图中包含的结构信息。具体而言，对于每个子图，首先提示LLM：预测 [object] 与 [goal] 之间最可能的距离，其中 [object] 表示中心目标节点的类别。随后，我们提示LLM提出与导航目标相关的问题，提示为：围绕 [object] 和 [goal] 提出问题，用于预测它们的距离。得到关于目标的这些问题后，我们再提示LLM根据子图信息回答这些问题，提示为：给定子图的 [nodes] 和 [edges]，回答以上 [questions]。因此，从这些回答中，我们获得推断目标位置所需的全部必要信息。最后，我们提示LLM总结对话并预测 $P$ ，提示为：基于以上对话，确定该子图与 [goal] 之间最可能的距离。而 ${P}^{\text{ sub }}$ 可通过将输出的距离取倒数来简单计算。LLM还会总结与所选前沿最近的3个子图的分析过程，以解释决策理由。

$$



where ${C}_{k}$ is the confidence score of this goal object, ${D}_{j}$ is the distance between the central object node of the $j$ -th subgraph and this object. The success condition for our re-perception mechanism is:
其中，${C}_{k}$ 为该目标对象的置信度得分，${D}_{j}$ 为第 $j$ 个子图中中心目标节点与该对象之间的距离。我们的再感知机制的成功条件是：

{N}_{\text{ stop }} < {N}_{\text{ max }}\text{ ,where }{N}_{\text{ stop }} = N\text{ ,s.t. }\mathop{\sum }\limits_{{i = 1}}^{{N - 1}}{S}_{i} < {S}_{\text{ thres }} \leq  \mathop{\sum }\limits_{{i = 1}}^{N}{S}_{i} \tag{4}
$$



where ${N}_{\max }$ is a pre-defined maximum number of steps. If succeed,the agent will navigate to this goal object without any further exploration or perception. Otherwise, it will give up this object and continue the exploration.

Evaluation Metrics: We report three metrics including success rate (SR), success rate weighted by path length (SPL) and SoftSPL. SR is the core metric of object-goal navigation task, representing the success rate of successful navigation episode. SPL measures the ability of agent to find the optimal path. If success, SPL = optimal path length, otherwise SPL = 0. SoftSPL [8] measures the navigation progress and navigation efficiency of the agent. Higher is better for all the three metrics.
评估指标：我们报告三个指标，包括成功率（SR）、按路径长度加权的成功率（SPL）和SoftSPL。SR是目标物体导航任务的核心指标，表示成功导航回合的成功率。SPL衡量智能体找到最优路径的能力。若成功，则SPL = 最优路径长度；否则SPL = 0。SoftSPL [8]衡量智能体的导航进展和导航效率。对于这三个指标，数值越高越好。

## 5 Conclusion
## 5 结论


In this paper, we have proposed SG-Nav, a zero-shot object-goal navigation framework that prompts LLM with scene graph to inference the position of goal object. We first construct a hierarchical 3D scene graph online to preserve rich context information of the environment. To speed up the building of scene graph, we propose an incremental updating and pruning strategy, which first densely connects newly registered nodes to all nodes by efficiently prompting the LLM. Then we divide the edges into long-range and short-range to prune them with different strategies. At each step, we divide the scene graph into subgraphs and prompt LLM with hierarchical chain-of-thought to output the probability score of each subgraph. Then the score for each frontier can be acquired by simple interpolation, which is also explainable as we can summarize the decision process of LLM on each subgraph. With the graph representation, we further design a re-perception mechanism to empower SG-Nav with the ability to give up false positive goal objects and thus correct perception error. Extensive experiments on three datasets validate the great performance of SG-Nav and the effectiveness of our design.
本文提出了 SG-Nav，这是一种零样本目标物体导航框架，通过场景图提示 LLM 推理目标物体的位置。我们首先在线构建层次化三维场景图，以保留环境的丰富上下文信息。为加速场景图构建，我们提出了一种增量更新与剪枝策略，先通过高效提示 LLM，将新注册节点与所有节点密集连接。随后，我们将边分为长距离和短距离，并采用不同策略进行剪枝。在每一步，我们将场景图划分为子图，并通过层次化思维链提示 LLM 输出每个子图的概率分数。然后可通过简单插值得到每个前沿点的分数，这一过程也具备可解释性，因为我们可以总结 LLM 在各个子图上的决策过程。借助图表示，我们进一步设计了再感知机制，使 SG-Nav 能够放弃误检的目标物体，从而纠正感知错误。在三个数据集上的大量实验验证了 SG-Nav 的优异性能以及我们设计的有效性。


Potential Limitations. Despite of the great performance, SG-Nav still has several limitations: (1) SG-Nav relies on online 3D instance segmentation method for scene graph construction. Currently we adopt 2D vision-language model and vision foundation model to segment each frame and merge the masks across different frames, which is not end-to-end and 3D-aware. A stronger online 3D instance segmentation model will further boost the performance of SG-Nav. (2) SG-Nav can only handle object-goal navigation yet. However, due to the strong zero-shot generalization ability of LLM and the rich context contained in 3D scene graph, we believe SG-Nav can be further extended to more tasks such as image-goal navigation and vision-and-language navigation.
潜在局限性。尽管性能出色，SG-Nav 仍有若干局限：(1) SG-Nav 依赖在线三维实例分割方法来构建场景图。目前我们采用二维视觉语言模型和视觉基础模型对每一帧进行分割，并跨不同帧合并掩码，这并非端到端，也不具备三维感知能力。更强的在线三维实例分割模型将进一步提升 SG-Nav 的性能。(2) SG-Nav 目前只能处理目标物体导航。然而，凭借 LLM 强大的零样本泛化能力以及三维场景图中包含的丰富上下文，我们相信 SG-Nav 还可进一步扩展到更多任务，如图像目标导航和视觉-语言导航。

### 6.3 Time Complexity for Edge Updating
### 6.3 边更新的时间复杂度


Here we provide detailed proof for Eq (1). As shown in Figure 7, the computational complexity of processing $m\left( {m + n}\right)$ pairs of nodes for the naive method is $\mathcal{O}\left( {m\left( {m + n}\right) }\right)$ and the time is ${T}_{\text{ naive }} = m\left( {m + n}\right)  \cdot  \mathcal{O}\left( {\left( {L}_{\text{ pro }} + {L}_{\text{ res }}\right) }^{r}\right)$ . Complexity of our method is ${T}_{\text{ our }} = \; \mathcal{O}\left( {\left( {L}_{pro} + m\left( m + n\right)  \cdot  {L}_{res} + m\left( m + n\right)  \cdot  2\right) }^{r}\right)$ ,where ${L}_{pro} \gg  2$ and ${L}_{pro} \gg  {L}_{res}$ .
这里我们给出式(1)的详细证明。如图7所示，朴素方法处理$m\left( {m + n}\right)$对节点的计算复杂度为$\mathcal{O}\left( {m\left( {m + n}\right) }\right)$，耗时为${T}_{\text{ naive }} = m\left( {m + n}\right)  \cdot  \mathcal{O}\left( {\left( {L}_{\text{ pro }} + {L}_{\text{ res }}\right) }^{r}\right)$。我们方法的复杂度为${T}_{\text{ our }} = \; \mathcal{O}\left( {\left( {L}_{pro} + m\left( m + n\right)  \cdot  {L}_{res} + m\left( m + n\right)  \cdot  2\right) }^{r}\right)$，其中${L}_{pro} \gg  2$和${L}_{pro} \gg  {L}_{res}$。


So we can approximate that $\frac{2}{{L}_{pro}} = 0$ and $\frac{{L}_{res}}{{L}_{pro}} = \alpha \left( {\alpha  \leq  {0.01}}\right)$ . According to the actual scene graph,usually $1 \leq  m \leq  5,1 \leq  n \leq  {100}$ . Then we compute the ratio as:
因此可近似认为$\frac{2}{{L}_{pro}} = 0$和$\frac{{L}_{res}}{{L}_{pro}} = \alpha \left( {\alpha  \leq  {0.01}}\right)$。根据实际场景图，通常$1 \leq  m \leq  5,1 \leq  n \leq  {100}$。然后我们计算比值为：


$$
\frac{{T}_{\text{ our }}}{{T}_{\text{ naive }}} = \frac{{\left( {L}_{pro} + m\left( m + n\right)  \cdot  {L}_{res} + m\left( m + n\right)  \cdot  2\right) }^{r}}{m\left( {m + n}\right)  \cdot  {\left( {L}_{pro} + {L}_{res}\right) }^{r}} = \frac{{\left( 1 + \frac{m\left( {m + n}\right)  \cdot  {L}_{res}}{{L}_{pro}} + \frac{m\left( {m + n}\right)  \cdot  2}{{L}_{pro}}\right) }^{r}}{m\left( {m + n}\right)  \cdot  {\left( 1 + \frac{{L}_{res}}{{L}_{pro}}\right) }^{r}} \tag{5}
$$



Since $\frac{2}{{L}_{pro}} = 0$ ,we have:
由于$\frac{2}{{L}_{pro}} = 0$，我们有：


$$
\frac{{T}_{\text{ our }}}{{T}_{\text{ naive }}} = \frac{1}{m\left( {m + n}\right) } \cdot  {\left( \frac{1 + \left( {{m}^{2} + {mn}}\right)  \cdot  \alpha }{1 + \alpha }\right) }^{r} \tag{6}
$$



Since $\alpha$ meets $0 < \alpha  \ll  1$ ,we can use the small value approximation principle ${\left( \frac{1 + {p\alpha }}{1 + {q\alpha }}\right) }^{r} \approx \; {\left( 1 + \left( p - q\right) \alpha \right) }^{r}$ to simplify the equation:
由于$\alpha$满足$0 < \alpha  \ll  1$，我们可以使用小量近似原理${\left( \frac{1 + {p\alpha }}{1 + {q\alpha }}\right) }^{r} \approx \; {\left( 1 + \left( p - q\right) \alpha \right) }^{r}$来简化方程：


$$
\frac{{T}_{\text{ our }}}{{T}_{\text{ naive }}} < \frac{1}{m + n} \cdot  \frac{{\left( 1 + \left( {m}^{2} + mn - 1\right)  \cdot  \alpha \right) }^{r}}{m} \tag{7}
$$



Then we use the prior $1 \leq  m \leq  5$ and $1 \leq  n \leq  {100}$ to scale the coefficient into a constant $c$ . The coefficient monotonically increases with $r,n$ and $\alpha$ :
然后我们利用先验$1 \leq  m \leq  5$和$1 \leq  n \leq  {100}$将系数缩放为常数$c$。该系数随$r,n$和$\alpha$单调增加：


$$
\frac{{\left( 1 + \left( {m}^{2} + mn - 1\right)  \cdot  \alpha \right) }^{r}}{m}\overset{r = 2,n = {100},\alpha  = {0.01}}{ \leq  }\frac{{\left( {0.01}{m}^{2} + m - {0.01}\right) }^{2}}{m} \tag{8}
$$



The coefficient decreases first and then increases with $m$ . It takes the maximum value when $m$ is at its maximum:
该系数随$m$先减后增。当$m$取最大值时，它达到最大：


$$
\frac{{\left( {0.01}{m}^{2} + m - {0.01}\right) }^{2}}{m}\overset{m = 5}{ \leq  }{5.49} \tag{9}
$$



Consequently, we obtain the ratio:
因此，我们得到比值：


$$
\frac{{T}_{\text{ our }}}{{T}_{\text{ naive }}} < \frac{c}{m + n} \tag{10}
$$



where $c = {5.49}$ . Our method reduces the complexity by $\left( {m + n}\right)$ times,from $\mathcal{O}\left( {m\left( {m + n}\right) }\right)$ to $\mathcal{O}\left( m\right)$ . So LLM generates relationship with $\mathcal{O}\left( m\right)$ complexity,which is much faster than the naive baseline.
其中$c = {5.49}$。我们的方法将复杂度降低了$\left( {m + n}\right)$倍，从$\mathcal{O}\left( {m\left( {m + n}\right) }\right)$降至$\mathcal{O}\left( m\right)$。因此，LLM生成关系的复杂度为$\mathcal{O}\left( m\right)$，这比朴素基线快得多。

- The paper should point out any strong assumptions and how robust the results are to violations of these assumptions (e.g., independence assumptions, noiseless settings, model well-specification, asymptotic approximations only holding locally). The authors should reflect on how these assumptions might be violated in practice and what the implications would be.

- The authors should reflect on the scope of the claims made, e.g., if the approach was only tested on a few datasets or with a few runs. In general, empirical results often depend on implicit assumptions, which should be articulated.
- 作者应反思所作主张的适用范围，例如该方法是否只在少量数据集或少量运行上测试过。一般而言，经验结果往往依赖隐含假设，这些假设应被明确说明。


- The authors should reflect on the factors that influence the performance of the approach. For example, a facial recognition algorithm may perform poorly when image resolution is low or images are taken in low lighting. Or a speech-to-text system might not be used reliably to provide closed captions for online lectures because it fails to handle technical jargon.
- 作者应反思影响该方法性能的因素。例如，面部识别算法在图像分辨率低或图像处于弱光条件下时可能表现不佳。又或者，语音转文字系统可能无法可靠地为在线讲座提供字幕，因为它无法处理专业术语。


- The authors should discuss the computational efficiency of the proposed algorithms and how they scale with dataset size.
- 作者应讨论所提算法的计算效率，以及它们如何随数据集规模扩展。


- If applicable, the authors should discuss possible limitations of their approach to address problems of privacy and fairness.
- 如适用，作者应讨论其方法在处理隐私和公平性问题方面的潜在局限。


- While the authors might fear that complete honesty about limitations might be used by reviewers as grounds for rejection, a worse outcome might be that reviewers discover limitations that aren't acknowledged in the paper. The authors should use their best judgment and recognize that individual actions in favor of transparency play an important role in developing norms that preserve the integrity of the community. Reviewers will be specifically instructed to not penalize honesty concerning limitations.
- 尽管作者可能担心如实说明局限性会被审稿人视为拒稿理由，但更糟的结果可能是审稿人发现了论文中未被承认的局限性。作者应尽其所能作出判断，并认识到有助于透明度的个人行为在形成维护社区完整性的规范方面起着重要作用。审稿人将被特别要求不要因作者坦诚局限性而予以扣分。
