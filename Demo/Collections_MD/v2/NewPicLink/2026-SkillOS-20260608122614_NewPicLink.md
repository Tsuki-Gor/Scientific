LLM-based agents are increasingly deployed to handle streaming tasks, yet they often remain one-off problem solvers that fail to learn from past interactions. Reusable skills distilled from experience provide a natural substrate for self-evolution, where high-quality skill curation serves as the key bottleneck. Existing approaches either rely on manual skill curation, prescribe heuristic skill operations, or train for short-horizon skill adaptation, but still struggle to learn complex long-term curation policies from indirect and delayed feedback. We propose SkillOS, an experience-driven RL training recipe for learning skill curation in self-evolving agents. SKILLOS pairs a frozen agent executor that retrieves and applies skills with a trainable skill curator that updates an external SKILLREPO from accumulated experience. To provide learning signals for curation, we train on grouped task streams based on skill-relevant task dependencies, where earlier trajectories update the SKILLREPO, and later related tasks evaluate these updates. We further design composite rewards to better attribute downstream executor feedback to curation decisions. Across multi-turn agentic tasks and single-turn reasoning tasks, SKILLOS consistently outperforms memory-free and strong memory-based baselines in both effectiveness and efficiency, with the learned skill curator generalizing across different executor backbones and task domains. Further analyses show that the learned curator produces more targeted skill use, while the evolving SKILL REPO develops richer internal structure and higher-level meta-skills over time.
基于 LLM 的智能体正日益被部署来处理流式任务，但它们往往仍只是一次性问题解决者，无法从过往交互中学习。由经验提炼而来的可复用技能为自我进化提供了天然基础，而高质量的技能编排是关键瓶颈。现有方法要么依赖人工技能编排，要么规定启发式技能操作，要么只针对短期技能适应进行训练，但仍难以从间接且延迟的反馈中学到复杂的长期编排策略。我们提出 SkillOS，这是一种面向经验驱动的强化学习训练方案，用于学习自我进化智能体中的技能编排。SKILLOS 将一个冻结的智能体执行器与一个可训练的技能编排器配对；前者检索并应用技能，后者根据积累的经验更新外部 SKILLREPO。为向编排过程提供学习信号，我们基于与技能相关的任务依赖对任务流进行分组训练：较早的轨迹更新 SKILLREPO，后续相关任务则评估这些更新。我们进一步设计了复合奖励，以更好地将下游执行器反馈归因于编排决策。在多轮智能体任务和单轮推理任务上，SKILLOS 在有效性和效率上始终优于无记忆基线和强记忆基线，且所学技能编排器能在不同执行器骨干和任务领域间泛化。进一步分析表明，所学编排器能产生更有针对性的技能使用，而不断演化的 SKILL REPO 也会随着时间发展出更丰富的内部结构和更高层次的元技能。


## 1. Introduction
## 1. 引言


LLM-based agents (Wang et al., 2024) are increasingly deployed in real-world scenarios, where they must move beyond instantaneous problem-solving toward long-term proficiency (He et al., 2026b). However, the prevailing paradigm of "one-off" task execution limits their utility in streaming settings, where tasks unfold sequentially over time. This makes self-evolution (Fang et al., 2025a; Gao et al., 2025) essential: capable agents should not repeatedly start from scratch, but instead continually accumulate, refine, and reuse experience for future tasks.
基于大语言模型的智能体（Wang 等，2024）正越来越多地部署到真实场景中，此时它们必须从“即时解决问题”进一步走向长期熟练度（He 等，2026b）。然而，“一次性”任务执行范式限制了它们在流式场景中的效用：任务会在一段时间内按顺序展开。这使得自我演化（Fang 等，2025a；Gao 等，2025）变得必不可少：具备能力的智能体不应反复从零开始，而应持续积累、完善并复用经验，以用于未来任务。


A key substrate for self-evolution is procedural memory (Fang et al., 2025b; Hu et al., 2025; Wu et al., 2025b), specifically, reusable skills (Anthropic, 2025b; Wang et al., 2025c) accumulated from past interactions. In real-world streaming settings (Wu et al., 2024), a skill-based self-evolving agent typically follows a closed-loop workflow: for each new task, it selects relevant skills, uses them to guide execution, and updates its skill collection based on the resulting trajectory. This makes skill curation-the extraction of high-quality lessons and their integration into the skill collection-essential for self-evolving agents.
自我演化的关键底座是过程性记忆（Fang 等，2025b；Hu 等，2025；Wu 等，2025b），更具体地说，是可复用技能（Anthropic，2025b；Wang 等，2025c），即从过去交互中积累的技能。在真实流式场景（Wu 等，2024）中，基于技能的自我演化智能体通常遵循闭环工作流：对每个新任务，它会选择相关技能，用这些技能来指导执行，并根据由此产生的轨迹更新其技能集合。这使得技能策划——提取高质量经验教训并集成到技能集合中——对自我演化智能体至关重要。


However, existing skill curation works remain limited. Manually curated skills, such as Anthropic's skills repository (Anthropic, 2025b), demand huge human expertise and cannot scale to the diversity of tasks that agents may encounter. Prompting or heuristic-based methods that dictate memory operations (Qiu et al., 2025; Xu et al., 2025; Zheng et al., 2025) rely on fixed rules and lack downstream performance feedback, preventing them from adapting to the executor's actual needs.
然而，现有的技能策划工作仍有局限。诸如 Anthropic 的技能仓库（Anthropic，2025b）这类人工策划技能，需要巨量的人类专业知识，且无法扩展到智能体可能遇到的多样化任务。采用提示或基于启发式的方法来指定记忆操作（Qiu 等，2025；Xu 等，2025；Zheng 等，2025），依赖固定规则，且缺少对下游表现的反馈，因而无法适应执行器的实际需求。

Recent studies explored reinforcement learning (RL) to optimize skill-based agent systems. However, they either focus on teaching agents to use skills (Tu et al., 2026; Xia et al., 2026) or optimize skill operations within a short task stream (Wang et al., 2025a; Ye et al., 2026). This limits the density of learning signals available for curating highly reusable skills and mastering complex management operations such as skill update and deletion, which are essential for robust and scalable long-term self-evolution.
近期研究探索了强化学习（RL）以优化基于技能的智能体系统。然而，它们要么关注教智能体如何使用技能（Tu 等，2026；Xia 等，2026），要么在短任务流中优化技能操作（Wang 等，2025a；Ye 等，2026）。这会限制用于策划高度可复用技能、并掌握诸如技能更新与删除等复杂管理操作的学习信号密度，而这些对于稳健且可扩展的长期自我演化至关重要。


To tackle this challenge, we propose SkillOS, an experience-driven RL training recipe to learn the capability of skill curation for self-evolving agents. We study skill curation in a modular multi-agent framework in a streaming setting, where a frozen agent executor solves tasks with a skill collection (termed SKILLREPO), while a trainable skill curator updates and manages this collection through function calls (Figure 1(a)). We represent skills as Markdown files (Anthropic, 2025b) (Figure 1(b)) managed via file I/O operations similar to an operating system (OS). Our recipe features two core designs. First, we construct each training instance as a group of related tasks. By mimicking test-time streaming settings, it grounds skill curation in long-term utility: skills induced from earlier experiences are evaluated by their ability to improve later related tasks. Second, we design rewards to better attribute environmental feedback to curation decisions, combining task performance with signals for valid function calls, skill quality, and SKILLREPO's compactness. Together, these designs turn delayed and indirect supervision into learning signals for skill curation.
为应对这一挑战，我们提出 SkillOS：一种由经验驱动的 RL 训练配方，用于学习自我演化智能体的技能策划能力。我们在流式场景下的模块化多智能体框架中研究技能策划：冻结的智能体执行器通过一组技能集合（称为 SKILLREPO）来解任务，而可训练的技能策划器通过函数调用更新并管理该集合（图 1(a)）。我们将技能表示为 Markdown 文件（Anthropic，2025b）（图 1(b)），并通过类似操作系统（OS）的文件 I/O 操作进行管理。我们的配方包含两项核心设计。首先，我们将每个训练实例构建为一组相关任务。通过模拟测试时的流式设置，它使技能策划立足于长期效用：由更早经验诱导得到的技能，会根据其对后续相关任务的改进能力进行评估。其次，我们设计奖励，以便更好地将环境反馈归因到策划决策中：在结合任务表现的同时，还加入有效函数调用、技能质量以及 SKILLREPO 紧凑性的信号。两项设计共同把延迟且间接的监督转化为学习信号，用于技能策划。


We evaluate SKILLOS on both multi-turn agentic tasks and single-turn reasoning tasks. Experiments show that SKILLOS consistently outperforms memory-free and strong memory-based methods in both effectiveness and efficiency, with up to +9.8% relative performance improvement and -6.0% fewer interaction steps compared to the strongest baseline (Table 1). Our trained skill curator generalizes well across executors and tasks, improving performance even with the Gemini-2.5-Pro executor. Notably, our 8B curator also outperforms Gemini-2.5-Pro when used directly as the curator. Beyond performance gains, our analyses further show that the learned skill curator leads to more targeted and effective skill utilization, while the skills in SKILLREPO evolve into more richly structured Markdown files that encode higher-level meta-skills over time. Together, we establish SKILLOS as a practical, modular, and experience-driven RL training recipe for building self-evolving agents.
我们在多轮智能体任务和单轮推理任务上评估 SKILLOS。实验表明，SKILLOS 在有效性与效率上均始终优于无记忆方法和强记忆方法，相对性能提升最高可达 +9.8%，且相比最强基线交互步数减少 -6.0%（表 1）。我们训练得到的技能策划器在不同执行器与任务上泛化良好，即使将 Gemini-2.5-Pro 作为执行器也能提升性能。值得注意的是，当策划器被直接用作策划器时，我们的 8B 策划器也优于 Gemini-2.5-Pro。除性能提升外，我们的分析进一步表明：学习到的技能策划器会带来更有针对性且更有效的技能使用，而 SKILLREPO 中的技能会随着时间演化为结构更丰富的 Markdown 文件，从中编码更高层级的元技能。综上，我们将 SKILLOS 作为一种可落地的、模块化的、由经验驱动的 RL 训练配方，用于构建自我演化智能体。


## 2. Related Work
## 2. 相关工作


Memory for Self-Evolving Agents. Learning from past experiences as procedural memory (Hu et al., 2025; Huang et al., 2026; Shen et al., 2026; Wei et al., 2025; Wu et al., 2025b; Zhang et al., 2024) is a central mechanism for developing self-evolving agents (Fang et al., 2025a; Gao et al., 2025). The central challenge is to encode interaction histories into reusable and retrievable representations.
面向自演化智能体的记忆。从过往经验中学习并将其作为程序性记忆（Hu et al., 2025; Huang et al., 2026; Shen et al., 2026; Wei et al., 2025; Wu et al., 2025b; Zhang et al., 2024）是构建自演化智能体（Fang et al., 2025a; Gao et al., 2025）的核心机制。其核心挑战在于将交互历史编码为可复用、可检索的表示。


Case-based representations are the most concrete form in this research line: they store experiences in minimally processed formats, allowing past histories to be replayed directly or reused as in-context exemplars, such as raw trajectories (Wu et al., 2025a; Zheng et al., 2024; Zhou et al., 2025) and abstracted query-response pairs (Islam et al., 2024; Zhao et al., 2024). Another line of work abstracts experiences into higher-level knowledge that is editable, auditable, and composable, reducing reliance on long trajectory replay and improving both cross-task generalization and efficiency. Such strategy-based memory typically consists of reusable workflows (Tang et al., 2025; Wang et al., 2025d), distilled insights (Ho et al., 2025; Huang et al., 2025; Ouyang et al., 2026), and recurring patterns (Kim et al., 2025; Yang et al., 2024). Recently, skills (Alzubi et al., 2026; Kuroki et al., 2025; Li et al., 2026a; Liang et al., 2026; Ling et al., 2026; Wang et al., 2025c; Yang et al., 2026; Zhang et al., 2026a) have emerged as a new agent-native form of memory and an orchestrable capability layer, owing to their modularity and ease of customization. Anthropic conceptualizes each skill as a folder containing instructions, scripts, and supporting resources (Anthropic, 2025a), which has become the most widely adopted design in the current community. Our work follows this design philosophy, simplifying the setting for research purposes by representing each skill as a single Markdown file.
基于案例的表示是这一研究方向中最具体的形式：它们以最少处理的格式存储经验，使过去的历史能够被直接重放，或作为上下文示例复用，例如原始轨迹（Wu et al., 2025a; Zheng et al., 2024; Zhou et al., 2025）和抽象的问答对（Islam et al., 2024; Zhao et al., 2024）。另一类工作将经验抽象为更高层次、可编辑、可审计且可组合的知识，从而减少对长轨迹重放的依赖，并提升跨任务泛化能力与效率。这类基于策略的记忆通常由可复用工作流（Tang et al., 2025; Wang et al., 2025d）、提炼出的见解（Ho et al., 2025; Huang et al., 2025; Ouyang et al., 2026）以及反复出现的模式（Kim et al., 2025; Yang et al., 2024）构成。近期，技能（Alzubi et al., 2026; Kuroki et al., 2025; Li et al., 2026a; Liang et al., 2026; Ling et al., 2026; Wang et al., 2025c; Yang et al., 2026; Zhang et al., 2026a）因其模块化和易定制性，已成为一种新的原生于智能体的记忆形式，以及可编排的能力层。Anthropic 将每个技能概念化为一个包含说明、脚本和辅助资源的文件夹（Anthropic, 2025a），这已成为当前社区最广泛采用的设计。我们的工作遵循这一设计理念，为研究目的将每个技能简化为单个 Markdown 文件。


Learning Memory and Skill Curation with RL. Training LLM-based agent systems with memory capabilities using RL has become a growing research direction. One research line targets training for long-context management with predefined operations such as compaction (Wang et al., 2025b; Yu et al., 2026; Zhou et al., 2026). Another interesting area focuses more on memory utilization and management by learning additional memory tool-calls (Yan et al., 2025; Zhang et al., 2025a,b) or training policies for different stages, such as memory retrieval (Zhang et al., 2026b). More recently, RL has been applied at various stages of agent skill development. Specifically, SkillRL (Xia et al., 2026) and D2Skill (Tu et al., 2026) teach smaller models to use skills curated from powerful LLMs in an iterative manner. ARISE (Li et al., 2026b) trains a shared policy operating both as skill retriever and worker, with heuristics for skill management. Recent studies have begun to train agents for memory or skill curation (Wang et al., 2025a; Ye et al., 2026), but their supervision is mostly restricted to local adaptation within short task streams. This favors immediately useful operations such as skill insertion, while offering limited signal for complex management operations, such as revising outdated skills and deleting harmful ones. SKILLOS instead formulates skill curation as a long-horizon, executor-grounded learning problem. We group related tasks into training instances and combine downstream task outcomes with intermediate rewards, turning delayed and indirect feedback into learning signals for skill curation.
使用 RL 学习记忆和技能筛选。利用 RL 训练具备记忆能力的 LLM 智能体系统已成为一个不断增长的研究方向。一条研究线致力于通过预定义操作（如压缩）（Wang et al., 2025b; Yu et al., 2026; Zhou et al., 2026）来训练长上下文管理。另一条有趣的方向则更侧重于记忆利用与管理，通过学习额外的记忆工具调用（Yan et al., 2025; Zhang et al., 2025a,b）或为不同阶段训练策略，例如记忆检索（Zhang et al., 2026b）。近年来，RL 也被应用于智能体技能发展的不同阶段。具体而言，SkillRL（Xia et al., 2026）和 D2Skill（Tu et al., 2026）教会较小模型以迭代方式使用由强大 LLM 筛选出的技能。ARISE（Li et al., 2026b）训练一个共享策略，同时作为技能检索器和执行者，并配合技能管理启发式方法。近期研究已开始训练智能体进行记忆或技能筛选（Wang et al., 2025a; Ye et al., 2026），但其监督大多仅限于短任务流中的局部适应。这更偏向于直接有用的操作，例如插入技能，而对复杂的管理操作，如修订过时技能和删除有害技能，提供的信号有限。相较之下，SKILLOS 将技能筛选表述为一个长时程、以执行器为依据的学习问题。我们将相关任务分组为训练实例，并结合下游任务结果与中间奖励，把延迟且间接的反馈转化为用于技能筛选的学习信号。


## 3. Methodology
## 3. 方法论


In this section, we first formalize the problem setting and introduce the multi-agent modular design of SKILLOS. We then detail the RL training recipe designed specifically for training the skill curator.
在本节中，我们首先形式化问题场景，并介绍 SKILLOS 的多智能体模块化设计。随后，我们将详细说明专为训练技能策展器而设计的强化学习训练方案。


### 3.1. Streaming Skill Curation with Multi-Agent Modular Design
### 3.1. 流式场景下的多智能体模块化技能策展


We consider a streaming test-time setting (Wu et al., 2024), where an LLM-based agent is deployed to solve a sequence of tasks $\mathcal{D} = \left\{  {{x}_{1},{x}_{2},\ldots ,{x}_{T}}\right\}$ that arrive over time. At each time stamp $t$ ,the agent must solve the current task ${x}_{t}$ before observing future tasks,producing an execution trajectory ${\xi }_{t} = \left\{  {{o}_{1},{a}_{1},\ldots ,{o}_{n},{a}_{n}}\right\}$ ,where $o$ and $a$ denote observations and actions,respectively. This setting naturally captures the challenge of self-evolving agents, where the system must distill useful experience from the trajectories of past interactions to improve performance on future tasks, and become more capable over time. Figure 1(a) presents an overview of the system.
我们考虑一种流式测试时（Wu et al., 2024）设置：将基于LLM的智能体部署以解决一系列随时间到来的任务 $\mathcal{D} = \left\{  {{x}_{1},{x}_{2},\ldots ,{x}_{T}}\right\}$ 。在每个时间戳 $t$ ，智能体必须在观察未来任务之前先解决当前任务 ${x}_{t}$ ，从而生成执行轨迹 ${\xi }_{t} = \left\{  {{o}_{1},{a}_{1},\ldots ,{o}_{n},{a}_{n}}\right\}$ 。其中 $o$ 与 $a$ 分别表示观测与动作。该设置自然地刻画了自进化智能体的挑战：系统需要从过去交互的轨迹中提炼有用经验，以提升未来任务上的表现，并随着时间推移变得更强。图1(a)展示了系统概览。

Agent Executor. Given a task ${x}_{t}$ ,a frozen agent executor ${\pi }_{\mathcal{L}}$ solves the task conditioning on the current environment observation and relevant skills. Specifically,we retrieve a subset of skills ${\widetilde{\mathcal{S}}}_{t} \subseteq  {\mathcal{S}}_{t}$ using BM25 (Robertson and Zaragoza,2009) for each task ${x}_{t}$ ,and the executor samples actions following $a \sim  {\pi }_{\mathcal{L}}\left( {\cdot  \mid  {x}_{t},{o}_{t},{\widetilde{\mathcal{S}}}_{t}}\right)$ .
智能体执行器。给定任务 ${x}_{t}$ ，一个冻结的智能体执行器 ${\pi }_{\mathcal{L}}$ 会在当前环境观测与相关技能的条件下解决该任务。具体来说，我们对每个任务 ${x}_{t}$ 使用 BM25（Robertson and Zaragoza, 2009）检索一部分技能 ${\widetilde{\mathcal{S}}}_{t} \subseteq  {\mathcal{S}}_{t}$ ，然后执行器按照 $a \sim  {\pi }_{\mathcal{L}}\left( {\cdot  \mid  {x}_{t},{o}_{t},{\widetilde{\mathcal{S}}}_{t}}\right)$ 采样动作。


Skill Curator. After the executor completes task ${x}_{t}$ ,the skill curator ${\pi }_{\mathcal{S}}$ observes the trajectory ${\xi }_{t}$ ,the self-judged correctness of the answers/interactions ${\mathbb{1}}_{{\xi }_{t}}$ ,and a retrieved subset of related skills ${\widetilde{\mathcal{S}}}_{t}$ . It then generates a sequence of structured curation operations ${c}_{t} = \left( {{u}_{t}^{1},\ldots ,{u}_{t}^{{M}_{t}}}\right)  \sim  {\pi }_{\mathcal{S}}\left( \cdot \right) \; \left. {{\xi }_{t},{\mathbb{1}}_{{\xi }_{t}},{\widetilde{\mathcal{S}}}_{t}}\right)$ ,where each operation ${u}_{t}^{m}$ is one of \{insert_skill,update_skill \}. Each operation is implemented as a function call (detailed signature in Figure 8) that manipulates the skill repository ${\mathcal{S}}_{t}$ . Applying these operations transforms the repository from ${\mathcal{S}}_{t}$ to ${\mathcal{S}}_{t + 1}$ as ${\mathcal{S}}_{t + 1} = \operatorname{APPLYOPS}\left( {{\mathcal{S}}_{t},{c}_{t}}\right)$ . The updated repository is then used by the executor on subsequent tasks, forming a closed loop between task execution and experience-driven skill evolution.
技能策展器。当执行器完成任务 ${x}_{t}$ 后，技能策展器 ${\pi }_{\mathcal{S}}$ 会观察轨迹 ${\xi }_{t}$ 、对答案/交互的自我评估正确性 ${\mathbb{1}}_{{\xi }_{t}}$ ，以及检索到的相关技能子集 ${\widetilde{\mathcal{S}}}_{t}$ 。随后，它会生成一串结构化的策展操作 ${c}_{t} = \left( {{u}_{t}^{1},\ldots ,{u}_{t}^{{M}_{t}}}\right)  \sim  {\pi }_{\mathcal{S}}\left( \cdot \right) \; \left. {{\xi }_{t},{\mathbb{1}}_{{\xi }_{t}},{\widetilde{\mathcal{S}}}_{t}}\right)$ ，其中每个操作 ${u}_{t}^{m}$ 都是 \{insert_skill,update_skill \} 之一。每个操作都通过一次函数调用实现（详见图8中的签名），用于操作技能仓库 ${\mathcal{S}}_{t}$ 。应用这些操作后，仓库会从 ${\mathcal{S}}_{t}$ 变为 ${\mathcal{S}}_{t + 1}$ ，如 ${\mathcal{S}}_{t + 1} = \operatorname{APPLYOPS}\left( {{\mathcal{S}}_{t},{c}_{t}}\right)$ 所示。更新后的仓库随后会在后续任务上被执行器使用，从而在任务执行与基于经验的技能演化之间形成闭环。


### 3.2. Learning Skill Curation with RL
### 3.2. 结合 RL 的技能筛选


We optimize the skill curator ${\pi }_{\mathcal{S}}$ with RL and keep the agent executor ${\pi }_{\mathcal{L}}$ frozen. The main challenge is indirect and delayed feedback for curation decisions,which is only revealed through ${\pi }_{\mathcal{L}}$ ’s performance on future relevant tasks. We address this by constructing grouped training instances (§ 3.2.1) and designing a composite reward (§ 3.2.2) that combines future task outcomes with intermediate signals on operation validity, skill quality, and the conciseness of skills. An overview of the training process is shown in Figure 2.
我们用 RL 优化技能筛选器 ${\pi }_{\mathcal{S}}$，并保持智能体执行器 ${\pi }_{\mathcal{L}}$ 冻结不变。主要挑战在于筛选决策只能通过 ${\pi }_{\mathcal{L}}$ 在未来相关任务上的表现间接且延迟地反馈。为此，我们构建了分组训练实例（§ 3.2.1），并设计了一个复合奖励（§ 3.2.2），将未来任务结果与关于操作有效性、技能质量以及技能简洁性的中间信号结合起来。训练过程概览见图 2。


#### 3.2.1. Training Instance Construction
#### 3.2.1. 训练实例构建


To provide downstream learning signals for skill curation, we construct each training instance as a group of related tasks that are solved sequentially. Within each group, SKILLREPO is updated by the curator ${\pi }_{f}$ after each task,allowing skills derived from earlier experiences to be evaluated by whether they help solve related future tasks. This also differs from prior work that focuses on short-horizon transfer (Wang et al., 2025a; Ye et al., 2026), where our grouped formulation exposes the curator to longer skill-evolution trajectories and provides denser feedback for learning complex curation operations.
为向技能策划提供下游学习信号，我们将每个训练实例构造成一组按顺序求解的相关任务。在每个组内，SKILLREPO 会在每个任务后由策划器 ${\pi }_{f}$ 更新，使得由早期经验衍生的技能可通过它们是否有助于解决相关的未来任务来加以评估。这也不同于以短期迁移为重点的先前工作（Wang et al., 2025a; Ye et al., 2026）；在我们的分组形式下，策划器接触到更长的技能演化轨迹，并为学习复杂的策划操作提供更密集的反馈。


Concretely,for each task ${x}_{i}$ in $\mathcal{D} = {\left\{  {x}_{i}\right\}  }_{i = 1}^{N}$ ,we first annotate each instance with a set of skill-relevant attributes. Formally,for each ${x}_{i}$ ,we use Gemini-2.5-Pro (Team,2025a) to produce a set of tags:
具体而言，对于 ${x}_{i}$ 中的每个任务 $\mathcal{D} = {\left\{  {x}_{i}\right\}  }_{i = 1}^{N}$，我们首先为每个实例标注一组与技能相关的属性。形式上，对于每个 ${x}_{i}$，我们使用 Gemini-2.5-Pro (Team,2025a) 生成一组标签：


$$
{Z}_{i} = \left\{  {{z}_{i}^{1},{z}_{i}^{2},\ldots ,{z}_{i}^{\left| {Z}_{i}\right| }}\right\}
$$



where each attribute ${z}_{i}$ captures a salient aspect of the task ${x}_{i}$ ,such as topic and common pitfalls. For example, in mathematical reasoning, attributes may include labels such as "algebra" or "Fourier transformation". These attributes serve as proxies for task-relatedness and potential skill dependency.
其中，每个属性 ${z}_{i}$ 都刻画了任务 ${x}_{i}$ 的一个显著方面，例如主题和常见陷阱。例如，在数学推理中，属性可以包括“代数”或“傅里叶变换”等标签。这些属性可作为任务相关性和潜在技能依赖性的代理。


Based on the annotated attributes,we then partition $\mathcal{D}$ into a collection of $M$ task groups using the similarity of attributes of these data samples:
基于这些已标注的属性，我们再根据这些数据样本属性的相似性，将 $\mathcal{D}$ 划分为若干 $M$ 任务组：


$$
\mathcal{D} = \left\{  {{G}_{1},{G}_{2},\ldots ,{G}_{M}}\right\}  ,\;{G}_{m} = \left\{  {{x}_{m,1},{x}_{m,2},\ldots ,{x}_{m,\left| {G}_{m}\right| }}\right\}  ,
$$



where all instances within the same group ${G}_{m}$ exhibit non-trivial dependency in terms of required skills. Detailed description of data processing and grouping algorithms can be found in Appendix B.2.
其中，同一组 ${G}_{m}$ 内的所有实例在所需技能方面都表现出非平凡的依赖关系。数据处理和分组算法的详细说明见附录 B.2。


#### 3.2.2. Training Loop and Policy Optimization
#### 3.2.2. 训练循环与策略优化


We employ Grouped Reward Policy Optimization (GRPO Shao et al. (2024)) for its training stability and sample efficiency. The training loop shown in Algorithm 1 optimizes the skill curator policy ${\pi }_{\mathcal{S}}$ to maximize a composite reward function over the distribution of generated traces. For a task group $G = \left( {{x}_{1},\ldots ,{x}_{\left| G\right| }}\right)$ ,the curator produces a sequence of curation decisions $c = \left( {{c}_{1},\ldots ,{c}_{\left| G\right| }}\right)$ as the executor proceeds through the group. Each training step, the reward combines four signals:
我们采用分组奖励策略优化（GRPO Shao et al. (2024)），因其训练稳定性和样本效率。算法 1 所示的训练循环优化技能策展策略 ${\pi }_{\mathcal{S}}$，以最大化生成轨迹分布上的复合奖励函数。对于任务组 $G = \left( {{x}_{1},\ldots ,{x}_{\left| G\right| }}\right)$，随着执行器在该组中推进，策展器会生成一系列策展决策 $c = \left( {{c}_{1},\ldots ,{c}_{\left| G\right| }}\right)$。每个训练步中，奖励由四个信号组成：


$$
r = \underset{\text{ task outcome }}{\underbrace{{r}^{\text{ task }}}} + {\lambda }_{\mathrm{f}}\underset{\text{ function call }}{\underbrace{{r}^{\mathrm{{fc}}}}} + {\lambda }_{\mathrm{u}}\underset{\text{ content quality }}{\underbrace{{r}^{\text{ cnt }}}} + {\lambda }_{\mathrm{c}}\underset{\text{ compression }}{\underbrace{{r}^{\text{ comp }}}} \tag{1}
$$



Task outcome reward. The first task uses an empty SKILLREPO, before any curator update occurs. We thus define the task outcome reward as the average success over the remaining tasks as ${r}^{\text{ task }} = \frac{1}{\left| G\right|  - 1}\mathop{\sum }\limits_{{i = 2}}^{\left| G\right| }\mathbb{1}\left( {\xi }_{i}\right)$ ,which provides executor-grounded signal on downstream performance achieved by the evolving SKILLREPO from ${\pi }_{\mathcal{S}}$ .
任务结果奖励。在任何策展器更新发生之前，第一个任务使用空的 SKILLREPO。我们因此将任务结果奖励定义为剩余任务的平均成功率，即 ${r}^{\text{ task }} = \frac{1}{\left| G\right|  - 1}\mathop{\sum }\limits_{{i = 2}}^{\left| G\right| }\mathbb{1}\left( {\xi }_{i}\right)$，这为不断演化的 SKILLREPO 在 ${\pi }_{\mathcal{S}}$ 上取得的下游表现提供了以执行器为基础的信号。


Function call reward. The function call reward measures whether the curator produces valid skill operations. For each curation decision ${c}_{i}$ ,let $\operatorname{Valid}\left( {c}_{i}\right)$ be the fraction of generated function calls that are valid and successfully executed. We define the function call reward as ${r}^{\mathrm{{fc}}} = \frac{1}{\left| G\right| }\mathop{\sum }\limits_{{i = 1}}^{\left| G\right| }\operatorname{Valid}\left( {c}_{i}\right)$ .
函数调用奖励。函数调用奖励衡量策展器是否生成了有效的技能操作。对于每个策展决策 ${c}_{i}$，设 $\operatorname{Valid}\left( {c}_{i}\right)$ 为已生成函数调用中有效且成功执行的比例。我们将函数调用奖励定义为 ${r}^{\mathrm{{fc}}} = \frac{1}{\left| G\right| }\mathop{\sum }\limits_{{i = 1}}^{\left| G\right| }\operatorname{Valid}\left( {c}_{i}\right)$。


Compression reward. To discourage verbatim trajectory copying, we reward concise repository updates. Let ${\mathcal{S}}_{i}$ denote the skill repository after applying ${c}_{i}$ ,and let ${\chi }_{i}$ denote the curator input context at position $i$ . We define ${r}^{\text{ comp }} = \frac{1}{\left| G\right| }\mathop{\sum }\limits_{{i = 1}}^{\left| G\right| }\left( {1 - \frac{\left| {\mathcal{S}}_{i}\right| }{\left| {\chi }_{i}\right| }}\right)$ ,where $\left| {\mathcal{S}}_{i}\right|$ and $\left| {\chi }_{i}\right|$ denote token lengths. This encourages the curator to distill reusable skills rather than store raw trajectories.
压缩奖励。为抑制对轨迹的逐字复制，我们奖励简洁的仓库更新。设 ${\mathcal{S}}_{i}$ 表示应用 ${c}_{i}$ 后的技能仓库，设 ${\chi }_{i}$ 表示位置 $i$ 处的策展器输入上下文。我们定义 ${r}^{\text{ comp }} = \frac{1}{\left| G\right| }\mathop{\sum }\limits_{{i = 1}}^{\left| G\right| }\left( {1 - \frac{\left| {\mathcal{S}}_{i}\right| }{\left| {\chi }_{i}\right| }}\right)$，其中 $\left| {\mathcal{S}}_{i}\right|$ 和 $\left| {\chi }_{i}\right|$ 表示 token 长度。这鼓励策展器提炼可复用的技能，而不是存储原始轨迹。


Content quality reward. The content quality reward evaluates whether the curated skills are semantically meaningful and likely to be useful for future tasks. Let $\operatorname{Judge}\left( {c}_{i}\right)$ denote the scalar score assigned by an external judge (Qwen3-32B) ${c}_{i}$ ,we compute the reward as ${r}^{\text{ cnt }} = \frac{1}{\left| G\right| }\mathop{\sum }\limits_{{i = 1}}^{\left| G\right| }\operatorname{Judge}\left( {c}_{i}\right)$ .
内容质量奖励。内容质量奖励评估所策展的技能在语义上是否有意义，以及是否可能对未来任务有用。设 $\operatorname{Judge}\left( {c}_{i}\right)$ 表示外部评审器（Qwen3-32B）赋予的标量评分 ${c}_{i}$，我们将奖励计算为 ${r}^{\text{ cnt }} = \frac{1}{\left| G\right| }\mathop{\sum }\limits_{{i = 1}}^{\left| G\right| }\operatorname{Judge}\left( {c}_{i}\right)$。


For each task group $G$ ,we sample $N$ independent rollouts of the entire curation sequence from ${\pi }_{\mathcal{S}}$ . Within each rollout,the executor produces trajectory ${\xi }_{i}$ using the skill repository ${\mathcal{S}}^{i}$ resulting from previous curations ${c}_{ < i}$ till task position $i$ with the same training task group,so different rollouts evolve different repository histories. The GRPO advantage is computed as: ${A}^{n} = {r}^{n} - \frac{1}{N}\mathop{\sum }\limits_{{{n}^{\prime } = 1}}^{N}{r}^{{n}^{\prime }}$ ,where ${r}^{n}$ is the composite reward (Eq. 1) for the $n$ -th rollout. We optimize ${\pi }_{\mathcal{S}}$ with a clipped surrogate objective over all curation steps $i = 1,\ldots ,\left| G\right|$ :
对于每个任务组$G$，我们从${\pi }_{\mathcal{S}}$中采样$N$个整个整理序列的独立 rollout。在每个 rollout 中，执行器使用由先前整理${c}_{ < i}$得到的技能仓库${\mathcal{S}}^{i}$，在相同训练任务组下生成至任务位置$i$的轨迹${\xi }_{i}$，因此不同 rollout 会演化出不同的仓库历史。GRPO 优势计算为：${A}^{n} = {r}^{n} - \frac{1}{N}\mathop{\sum }\limits_{{{n}^{\prime } = 1}}^{N}{r}^{{n}^{\prime }}$，其中${r}^{n}$是第$n$个 rollout 的复合奖励（式 1）。我们对${\pi }_{\mathcal{S}}$在所有整理步骤$i = 1,\ldots ,\left| G\right|$上采用裁剪的替代目标进行优化：

For testing, we additionally include Qwen3-32B, Gemini-2.5-Pro (Team, 2025a), and Gemini-3.1-Flash-Lite (Appendix C.1) as executors to evaluate the generalization of SKILLOS under different executor scales and architectures. ### 4.3. SKILLOS 的泛化


SKILLOS is transferable and remains effective for different agent executors. During training, we use Qwen3-8B as the executor. To test whether SKILLOS brings improvement for executors that are not seen in training, we pair the trained skill curator with different executors. As shown in Table 1 and 2, SKILLOS consistently improves a wide range of frozen executors across benchmarks, from open-source models (Qwen3-8B, Qwen3-32B) to frontier models (Gemini-2.5-Pro). Skill Evolution Dynamics. Beyond aggregate performance, we examine how the skill repository evolves during RL training. ## 6. Conclusion
## 6. 结论


We presented SkillOS, an RL training recipe for learning skill curation in self-evolving agents. By decoupling the skill curator from the agent executor, SKILLOS enables modular skill curation without retraining the underlying executor. Through grouped task streams and executor-grounded rewards, SKILLOS optimizes curation decisions by their downstream impact on future tasks. Across diverse benchmarks and LLM backbones, SKILLOS consistently improves both performance and efficiency. Further analyses show that trained skill curation can outperform frontier models' zero-shot curation ability and generalize across settings, highlighting modular, trained skill curation as a practical path toward agents that self-evolve from experience.
我们提出了 SkillOS，一种用于学习自进化智能体中技能筛选的 RL 训练方案。通过将技能筛选器与智能体执行器解耦，SkillOS 实现了无需重新训练底层执行器的模块化技能筛选。借助分组任务流和基于执行器的奖励，SkillOS 根据其对未来任务的下游影响来优化筛选决策。跨越多样化基准和 LLM 骨干，SkillOS 始终同时提升性能与效率。进一步分析表明，经过训练的技能筛选能力可超越前沿模型的零样本筛选能力，并可在不同设置间泛化，凸显模块化、经训练的技能筛选是通向能从经验中自我进化智能体的一条实用路径。

### B.2. Grouping Training Instances
### B.2. 训练实例分组


In this section,we detail the two-stage pipeline used to turn the raw training set $\mathcal{D} = {\left\{  {x}_{i}\right\}  }_{i = 1}^{N}$ into the grouped training set $\mathcal{G} = {\left\{  {G}_{j}\right\}  }_{j = 1}^{M}$ of Section 3.2.1. Stage 1 annotates each instance with a structured set of latent attributes via an LLM annotator (Sec. B.2.1). Stage 2 assembles groups of related tasks by retrieving, filtering, and ranking candidates under a semantic phrase-level similarity (Sec. B.2.2). For training of single-turn reasoning tasks, we instantiate the pipeline on DEEPMATH-103K (He et al., 2026a),which provides both the raw problems ${x}_{i}$ and a scalar difficulty score ${d}_{i} \in  \mathbb{R}$ that is reused as a curriculum signal by Stage 2. For multi-turn agentic tasks, we leverage the default task type annotation for each benchmark (e.g., 6 task types in ALFWorld) as they naturally expose a discrete partition of tasks into families that share the same underlying skills, and we can use this partition directly in place of the annotated attribute set ${Z}_{i}$ .
本节详细介绍将原始训练集 $\mathcal{D} = {\left\{  {x}_{i}\right\}  }_{i = 1}^{N}$ 转换为第 3.2.1 节中的分组训练集 $\mathcal{G} = {\left\{  {G}_{j}\right\}  }_{j = 1}^{M}$ 所采用的两阶段流程。第一阶段通过 LLM 标注器（见 B.2.1 节）为每个实例标注一组结构化的潜在属性。第二阶段在语义短语级相似度下，通过检索、筛选和排序候选项，组装相关任务组（见 B.2.2 节）。对于单轮推理任务的训练，我们将该流程应用于 DEEPMATH-103K（He et al., 2026a），它同时提供原始问题 ${x}_{i}$ 和标量难度分数 ${d}_{i} \in  \mathbb{R}$；该分数在第二阶段被复用于课程信号。对于多轮智能体任务，我们利用每个基准的默认任务类型标注（例如 ALFWorld 中的 6 类任务），因为它们天然将任务划分为共享相同底层技能的家族，我们可以直接用这一划分替代标注的属性集 ${Z}_{i}$ 。

#### B.2.1. Stage 1: Latent Attribute Annotation
#### B.2.1. 阶段 1：潜在属性标注


We implement the attribute set ${Z}_{i}$ of each instance ${x}_{i}$ as a tuple of five phrase-lists,
我们将每个实例 ${x}_{i}$ 的属性集合 ${Z}_{i}$ 实现为一个由五个短语列表组成的元组，


$$
{Z}_{i} = \left( {{T}_{i},{S}_{i},{C}_{i},{R}_{i},{P}_{i}}\right)
$$



where ${T}_{i}$ is the list of high-level topics, ${S}_{i}$ the required skills or capabilities, ${C}_{i}$ the underlying mathematical concepts or theorems, ${R}_{i}$ the applicable heuristic strategies,and ${P}_{i}$ the common pitfalls. Each dimension is populated by a small set of short phrases (at most five words each). The annotator is instructed to: (i) emit standardized terminology rather than free-form rationales, (ii) omit any content specific to the question text or its final answer, and (iii) use as few phrases per dimension as necessary to characterize the task. We enforce the output schema via structured decoding with a fixed JSON response schema, and query Gemini-2.5-Pro with the highest thinking-budget configuration. The exact annotation instruction is reproduced in Figure 16.
其中 ${T}_{i}$ 是高层主题列表，${S}_{i}$ 是所需技能或能力，${C}_{i}$ 是底层的数学概念或定理，${R}_{i}$ 是适用的启发式策略，${P}_{i}$ 是常见的陷阱。每个维度都由一小组短短语构成（每个短语最多五个词）。标注者被要求：（i）输出标准化术语而非自由形式的推理，（ii）省略任何与问题文本或其最终答案相关的特定内容，（iii）为刻画任务所需的每个维度尽可能少地使用短语。我们通过结构化解码来强制输出符合预设的模式，并使用固定的 JSON 响应模式；随后以最高思考预算配置调用 Gemini-2.5-Pro。精确的标注指令在图 16 中复现。


#### B.2.2. Stage 2: Group Construction
#### B.2.2 阶段 2：组构建


Given ${\left\{  \left( {x}_{i},{Z}_{i},{d}_{i}\right) \right\}  }_{i = 1}^{N}$ ,we construct each group ${G}_{j} = \left( {{x}_{j,1},\ldots ,{x}_{j,n}}\right)$ by sampling a seed task and then iteratively appending related tasks. The core primitive is a pair sampler that,given a source ${x}_{s}$ ,returns an admissible successor ${x}_{t}$ ; longer groups are obtained by iterating this primitive with a growing exclusion set so that instances within a group remain distinct.
给定 ${\left\{  \left( {x}_{i},{Z}_{i},{d}_{i}\right) \right\}  }_{i = 1}^{N}$ ，我们通过先采样一个种子任务，再反复追加相关任务来构建每个组 ${G}_{j} = \left( {{x}_{j,1},\ldots ,{x}_{j,n}}\right)$ 。核心原语是一个成对采样器：它在给定源 ${x}_{s}$ 的情况下，返回一个可接受的后继 ${x}_{t}$ ；更长的组则通过反复使用该原语并逐步扩大排除集合来获得，从而保证同一组内的实例保持区分性。

Phrase similarity. Because the annotated phrases come from a large open vocabulary (e.g., "pigeonhole principle" vs. "counting argument"), exact set overlap is unreliable. We therefore score the similarity between any two phrase lists $A$ and $B$ using a soft-Jaccard ${\mathrm{{SJ}}}_{\tau }\left( {A,B}\right)$ that combines exact matches with a greedy one-to-one matching between remaining phrases under a sentence-embedding cosine similarity (computed with all-MiniLM-L6-v2 (Reimers and Gurevych, 2019)) above a threshold $\tau$ . We write ${m}_{\tau }\left( {A,B}\right)$ for the resulting integer matched-pair count,which we use alongside ${\mathrm{{SJ}}}_{\tau }$ in the filters below.
短语相似度。因为标注短语来自一个大型开放词表（例如“抽屉原理”与“计数论证”），精确的集合重叠并不可靠。因此，我们使用一个软 Jaccard ${\mathrm{{SJ}}}_{\tau }\left( {A,B}\right)$ 来度量任意两份短语列表 $A$ 和 $B$ 的相似度：它将精确匹配与基于句向量嵌入的余弦相似度进行的贪婪一对一匹配结合起来（用 all-MiniLM-L6-v2 计算（Reimers 和 Gurevych，2019）），且相似度需高于阈值 $\tau$ 。我们将所得的整数匹配对数量记为 ${m}_{\tau }\left( {A,B}\right)$，并在下方过滤条件中与 ${\mathrm{{SJ}}}_{\tau }$ 一同使用。


Dependency gate. For a source ${x}_{s}$ and candidate ${x}_{t}$ ,we accept the pair only when all of the following hold:
依赖门控。对于源 ${x}_{s}$ 和候选 ${x}_{t}$ ，仅当满足以下全部条件时，我们才接受该对：


1. Shared foundation: ${m}_{\tau }\left( {{C}_{s},{C}_{t}}\right)  \geq  {\kappa }_{C}$ and ${m}_{\tau }\left( {{S}_{s},{S}_{t}}\right)  \geq  {\kappa }_{S}$ ;
1. 共享基础：${m}_{\tau }\left( {{C}_{s},{C}_{t}}\right)  \geq  {\kappa }_{C}$ 和 ${m}_{\tau }\left( {{S}_{s},{S}_{t}}\right)  \geq  {\kappa }_{S}$ ；


2. Shared reasoning: ${m}_{\tau }\left( {{R}_{s},{R}_{t}}\right)  + {m}_{\tau }\left( {{P}_{s},{P}_{t}}\right)  \geq  1$ ;
2. 共享推理：${m}_{\tau }\left( {{R}_{s},{R}_{t}}\right)  + {m}_{\tau }\left( {{P}_{s},{P}_{t}}\right)  \geq  1$ ；


3. Not a near-duplicate: ${\mathrm{{SJ}}}_{\tau }\left( {{T}_{s},{T}_{t}}\right)  \leq  {\theta }_{T}$ and the weighted overall similarity $\Omega \left( {{x}_{s},{x}_{t}}\right)  \leq  {\sigma }_{\max }$ ;
3. 不是近似重复：${\mathrm{{SJ}}}_{\tau }\left( {{T}_{s},{T}_{t}}\right)  \leq  {\theta }_{T}$ 和加权的整体相似度 $\Omega \left( {{x}_{s},{x}_{t}}\right)  \leq  {\sigma }_{\max }$ ；


4. Not too unrelated: $\Omega \left( {{x}_{s},{x}_{t}}\right)  \geq  {\sigma }_{\min }$ ;
4. 不至于过于不相关：$\Omega \left( {{x}_{s},{x}_{t}}\right)  \geq  {\sigma }_{\min }$ ；


5. Progression: ${x}_{t}$ introduces at least one new concept or skill,i.e. $\left| {C}_{t}\right|  > {m}_{\tau }\left( {{C}_{s},{C}_{t}}\right)$ or $\left| {S}_{t}\right|  > {m}_{\tau }\left( {{S}_{s},{S}_{t}}\right)$ ;
5. 递进性：${x}_{t}$ 至少引入一个新概念或技能，即 $\left| {C}_{t}\right|  > {m}_{\tau }\left( {{C}_{s},{C}_{t}}\right)$ 或 $\left| {S}_{t}\right|  > {m}_{\tau }\left( {{S}_{s},{S}_{t}}\right)$ ；


6. Curriculum direction: ${d}_{t} - {d}_{s} \geq  {\delta }_{\min }$ .
6. 课程方向：${d}_{t} - {d}_{s} \geq  {\delta }_{\min }$ 。


Here $\Omega$ is a convex combination of per-dimension soft-Jaccard scores across $\{ C,S,R,P,T\}$ with weights listed in Table 5. Conditions (1)-(2) ensure genuine reuse of foundational knowledge and reasoning machinery; (3)-(4) place the pair in a useful "related but not redundant" band; (5) guarantees that ${x}_{t}$ carries something new for the skill curator to compress into the library; and (6) enforces a forward curriculum.
这里 $\Omega$ 是在 $\{ C,S,R,P,T\}$ 上按表 5 中给出的权重，对各维度软 Jaccard 得分进行的凸组合。（1）—（2）确保真正复用基础知识与推理机制；（3）—（4）将该对放入“相关但不冗余”的有用区间；（5）保证 ${x}_{t}$ 对技能策划者而言确实带来新内容，能够被压缩进库中；（6）则强制采用向前的课程安排。

Candidate retrieval and scoring. Scoring all $N - 1$ alternatives per source is prohibitive,so we precompute an inverted index over the dependency fields $\{ C,R,P\}$ : for each source ${x}_{s}$ ,the candidate pool consists of tasks that share at least one exact dependency phrase with ${x}_{s}$ ,capped at ${K}_{\text{ inv }}$ entries via uniform subsampling. Routing retrieval through dependency fields rather than topics prevents groups from collapsing onto a single narrow subject. Among the candidates that pass the gate, we select the one that maximizes
候选检索与评分。对每个源任务枚举所有$N - 1$个候选进行评分代价高昂，因此我们在依赖字段$\{ C,R,P\}$上预计算倒排索引：对每个源${x}_{s}$，候选池由与${x}_{s}$共享至少一个精确依赖短语的任务组成，并通过均匀子采样将其上限截断为${K}_{\text{ inv }}$条。通过依赖字段而非主题进行检索，可防止组别收缩到单一狭窄主题。在通过门控的候选中，我们选择使


$$
s\left( {{x}_{s},{x}_{t}}\right)  = \mathop{\sum }\limits_{{f \in  \{ C,S,R,P,T\} }}{w}_{f}{\mathrm{{SJ}}}_{\tau }\left( {{f}_{s},{f}_{t}}\right)  + \lambda  \cdot  b\left( {{d}_{s},{d}_{t}}\right) ,
$$



where $b\left( \cdot \right)$ is a bounded difficulty bonus that rewards moderate forward steps. If no inverted-index candidate passes the gate,we fall back to a uniform random pool of size $F$ and re-apply the same gate and scoring; this catches pairs whose phrases agree semantically but not lexically. Extensions sourced from the fallback pool are tagged so downstream training can audit or downweight them. The difficulty gap ${d}_{t} - {d}_{s}$ is additionally modulated by a randomized curriculum mode $\left( {{p}_{ \uparrow  },{p}_{ = },{p}_{ \downarrow  }}\right)$ ; for our main experiments, we use an almost exclusively forward curriculum, which produced a more stable training signal than mixed curricula.
其中$b\left( \cdot \right)$是有界的难度奖励，用于鼓励适度的前向推进。如果没有倒排索引候选通过门控，我们便回退到大小为$F$的均匀随机池，并重新应用相同的门控与评分；这可捕捉那些短语在语义上而非字面上对应的配对。来自回退池的扩展会被标记，以便下游训练进行审计或降低权重。难度差${d}_{t} - {d}_{s}$还会受随机课程模式$\left( {{p}_{ \uparrow  },{p}_{ = },{p}_{ \downarrow  }}\right)$调制；在主实验中，我们使用几乎完全前向的课程，其训练信号比混合课程更稳定。

#### B.3.2. Baselines
#### B.3.2. 基线


We compare SKILLOS against five representative baselines that span memory-free agents, recent memory-augmented methods, and two internal variants of our own framework. All baselines share the same frozen Agent Executor and are evaluated under identical task suites, retrieval budgets, and decoding settings to isolate the contribution of the memory mechanism.
我们将 SKILLOS 与五个具有代表性的基线进行对比，覆盖无记忆智能体、近期的记忆增强方法，以及我们框架的两个内部变体。所有基线都使用相同且固定不变的 Agent Executor，并在相同的任务集合、检索预算和解码设置下评估，以此隔离记忆机制带来的贡献。


(i) No Memory. A memory-free baseline in which the Agent Executor solves each task independently, without access to any external memory or cross-task knowledge transfer. Each episode begins from a blank state, and no information is retained across tasks. This baseline establishes a lower bound and isolates the contribution of any form of accumulated experience.
(i) 无记忆（No Memory）。一种无记忆的基线：Agent Executor 在每个任务上独立求解，无法访问任何外部记忆或跨任务的知识迁移。每个回合都从空白状态开始，任务之间不保留任何信息。该基线用于建立下界，并隔离任何形式的累积经验所带来的影响。


(ii) ReasoningBank (Ouyang et al., 2026). A recent memory-augmented method that distills reusable reasoning insights from past trajectories and stores them as a searchable bank for future tasks. At inference time, relevant insights are retrieved and injected into the executor's context to guide reasoning. ReasoningBank represents the class of experience-distillation approaches, which emphasize the content of stored knowledge but rely on fixed, heuristic policies for deciding what to write or discard.
(ii) ReasoningBank（Ouyang et al., 2026）。一种近期提出的记忆增强方法，它从过去的轨迹中蒸馏可复用的推理见解，并将其存储为可检索的知识库以用于未来任务。推理时，会检索相关见解并注入到执行器的上下文中以指导推理。ReasoningBank 属于经验蒸馏类方法：强调被存储知识的内容，但在决定写入或丢弃什么时依赖固定的启发式策略。


(iii) MemP (Fang et al., 2025b). A procedural-memory method that induces reusable procedures from agent experience and applies advanced memory-management strategies - including consolidation, forgetting, and re-indexing - to maintain the memory store over time. MemP represents the class of rule-based memory management approaches, which feature more sophisticated maintenance policies than ReasoningBank but still prescribe curation decisions through hand-designed heuristics rather than learning them from downstream task feedback.
(iii) MemP（Fang et al., 2025b）。一种过程性记忆方法，它从智能体经验中诱导可复用的过程，并采用先进的记忆管理策略——包括巩固、遗忘和再索引——以便在时间上持续维护记忆库。MemP 属于基于规则的记忆管理方法类别，其维护策略比 ReasoningBank 更复杂，但仍通过人工设计的启发式来规定筛选与维护决策，而不是从下游任务反馈中学习。


(iv) SKILLOS-base. A variant of our framework in which the Skill Curator is instantiated with the same open-source backbone as SKILLOS but without any RL fine-tuning, while all other components remain identical to SKILLOS. This baseline serves two purposes: (a) it provides a lower-bound reference point that reflects the intrinsic prompting-based curation ability of the open-source backbone prior to optimization, and (b) it isolates the contribution of our GRPO-based training, since SKILLOS-base shares exactly the same model architecture, prompting template, and memory interface as SKILLOS but forgoes end-to-end optimization against task performance.
(iv) SKILLOS-base。我们框架的一个变体：Skill Curator 使用与 SKILLOS 相同的开源骨干，但不进行任何 RL 微调；而所有其他组件保持与 SKILLOS 完全一致。该基线用于两点：（a）提供一个下界参考，用于反映在优化之前，开源骨干自身基于提示（prompting）的策划能力；（b）隔离我们基于 GRPO 的训练贡献，因为 SKILLOS-base 与 SKILLOS 共享完全相同的模型架构、提示模板和记忆接口，但放弃了针对任务表现的端到端优化。


(v) SKILLOS-gemini. A variant of our framework in which the Skill Curator is instantiated with Gemini-2.5-Pro instead of a trained open-source model, while all other components remain identical to SKILLOS. This baseline serves two purposes: (a) it provides a strong closed-source reference point for the upper bound of prompting-based curation, and (b) it isolates the effect of our GRPO-based training, since SKILLOS-gemini shares the same prompting template and memory interface as SKILLOS but forgoes RL optimization against task performance.
(v) SKILLOS-gemini。我们框架的一个变体：Skill Curator 使用 Gemini-2.5-Pro 替代训练好的开源模型，而其他所有组件保持与 SKILLOS 完全一致。该基线用于两点：（a）为基于提示的策划上界提供一个强的闭源参考；（b）隔离我们基于 GRPO 的训练效果，因为 SKILLOS-gemini 与 SKILLOS 共享相同的提示模板和记忆接口，但放弃了针对任务表现的 RL 优化。


Together, these baselines cover the main design axes along which memory-augmented agents differ from SKILLOS: whether memory exists at all (i), how stored knowledge is represented (ii vs. iii), and whether curation decisions are prescribed by heuristics or learned from task feedback (ii and iii vs. SKILLOS), as well as whether the curator itself benefits from RL optimization (iv and v vs. Skillos).
总体而言，这些基线覆盖了记忆增强智能体与 SKILLOS 的主要设计差异：是否存在记忆（i）、存储知识的表示方式（ii 与 iii），以及策划决策是由启发式预先规定还是从任务反馈中学习（ii 和 iii vs. SKILLOS）；此外还包括策划器自身是否受 RL 优化收益（iv 和 v vs. Skillos）。


#### B.3.3. Evaluation Metrics
#### B.3.3. 评估指标


We evaluate SKILLOS and all baselines along two complementary axes - task effectiveness and action efficiency - using metrics tailored to each benchmark. Across all benchmarks and methods, every configuration is run with three independent random seeds; we report the mean across seeds, with one standard deviation shown as a subscript (e.g., ${85.7}_{\pm {1.6}}$ ). Within each backbone block of Tables 1 and 2, the best value in each column is highlighted in bold.
我们从两个互补维度评估 SKILLOS 和所有基线——任务有效性与动作效率——并使用针对各基准定制的指标。在所有基准和方法中，每种配置都使用三个独立随机种子运行；我们报告种子的均值，并以标准差作为下标显示（例如，${85.7}_{\pm {1.6}}$）。在表1和表2的每个骨干块中，每列的最佳值均以粗体突出显示。


Success Rate (SR $\uparrow$ ). Our primary effectiveness metric on both ALFWorld and WebShop. On ALFWorld, SR is the fraction of evaluation episodes in which the agent reaches the goal state within the step budget,yielding a binary $\{ 0,1\}$ outcome per episode. We report SR both per task category - Pick, Look, Clean, Heat, Cool, and Pick2 - and as a macro-average (Avg. SR) across the six categories, so that categories with fewer tasks are not dominated by larger ones. On WebShop, following (Yao et al., 2022), SR is the fraction of episodes whose final reward equals exactly 1, i.e., the purchased product fully matches all specified attributes, options, type, and price constraints.
成功率（SR $\uparrow$）。我们在 ALFWorld 和 WebShop 上使用的主要有效性指标。在 ALFWorld 中，SR 是指智能体在步数预算内到达目标状态的评估回合占比，每个回合产生二元 $\{ 0,1\}$ 结果。我们分别按任务类别——Pick、Look、Clean、Heat、Cool 和 Pick2——报告 SR，并给出跨六类的宏平均（Avg. SR），以避免任务较少的类别被任务较多的类别掩盖。在 WebShop 中，按照（Yao et al., 2022），SR 是指最终奖励恰好等于 1 的回合占比，即所购产品完全匹配所有指定属性、选项、类型和价格约束。


WebShop Score (↑). In addition to SR, WebShop provides a dense per-episode reward in [0, 100] that credits partial matches on attributes, options, type, and price even when the purchase is not a perfect match. We report the average score across evaluation episodes as a finer-grained complement to SR: two methods with similar SR may differ substantially in how close their near-misses are to the target product.
WebShop 分数（↑）。除了 SR 之外，WebShop 还提供 [0, 100] 范围内的稠密逐回合奖励，即使购买并非完全匹配，也会对属性、选项、类型和价格的部分匹配给予奖励。我们报告评估回合的平均分，作为对 SR 的更细粒度补充：两个 SR 相近的方法，在接近目标商品的程度上可能差异很大。


Number of Steps (Steps ↓). Our efficiency metric on ALFWorld and WebShop. Steps is the average number of environment actions the agent issues per episode, computed over all evaluation episodes regardless of success. Failed episodes contribute steps up to their termination point (task completion, max-step cutoff, or early stop). This metric captures a dimension that SR and Score alone cannot: two methods may achieve comparable effectiveness while differing substantially in how efficiently they reach the goal, which has direct implications for inference cost and deployment feasibility.
步数（Steps ↓）。我们在 ALFWorld 和 WebShop 上使用的效率指标。Steps 是智能体每回合发出的环境动作平均数量，按所有评估回合计算，不论是否成功。失败回合会计入直到终止点的步数（任务完成、最大步数截断或提前停止）。该指标刻画了 SR 和 Score 单独无法反映的维度：两种方法可能实现相近的有效性，却在到达目标的效率上差异显著，这对推理成本和部署可行性有直接影响。


Accuracy (Acc. ↑) on reasoning benchmarks. For the single-turn reasoning datasets — AIME24, AIME25, and GPQA - we report exact-match accuracy: the fraction of questions whose extracted final answer matches the ground truth. For AIME24 and AIME25, we adopt the evaluation protocol from the HuggingFace math_verify ${}^{1}$ toolkit,which parses the model’s final boxed expression and verifies mathematical equivalence to the reference answer (accounting for equivalent numerical forms, simplifications, and formatting variants). For GPQA, which is a multiple-choice benchmark, we extract the predicted option letter from the model's response and score it as correct if and only if it exactly matches the ground-truth option. We additionally report an average accuracy (Avg. Acc.) across the three datasets to summarize overall reasoning ability.
推理基准上的准确率（Acc. ↑）。对于单轮推理数据集——AIME24、AIME25 和 GPQA——我们报告精确匹配准确率：提取出的最终答案与标准答案一致的题目占比。对于 AIME24 和 AIME25，我们采用 HuggingFace math_verify ${}^{1}$ 工具包中的评估协议，该工具包会解析模型最终的 boxed 表达式，并验证其与参考答案的数学等价性（考虑等价的数值形式、化简和格式变体）。对于 GPQA，这是一个多项选择基准，我们从模型回复中提取预测的选项字母，并且仅当其与真实选项完全一致时判为正确。我们还报告这三个数据集上的平均准确率（Avg. Acc.），以概括整体推理能力。


Evaluation protocol. All methods share the same frozen Agent Executor, retrieval budget (top-k skills retrieved via BM25), maximum step budget, and decoding temperature within each backbone, so that differences in the reported metrics are attributable to the memory mechanism rather than to confounding inference settings. Unless stated otherwise, all numbers in the main paper are computed on the official held-out evaluation splits of each benchmark.
评估协议。所有方法在各自骨干内都共享相同的冻结 Agent Executor、检索预算（通过 BM25 检索 top-k 技能）、最大步数预算和解码温度，因此报告指标上的差异可归因于记忆机制，而非混杂的推理设置。除非另有说明，正文中的所有数值均基于各基准官方保留评测划分计算。

## D. Limitations
## D. 局限性


Retrieval Mechanism. Our current implementation relies on a relatively simple keyword-based retrieval mechanism, such as BM25, to retrieve relevant skills from the skill repository. This design choice allows us to isolate the main focus of this work: studying how skills can be curated, updated, and organized through experience-driven learning. However, more advanced retrieval methods, such as dense retrieval, hybrid retrieval, or learned retrievers, may further improve the relevance of retrieved skills and thus lead to stronger downstream performance. We leave the joint optimization of skill curation and skill retrieval to future work.
检索机制。我们目前的实现依赖相对简单的基于关键词的检索机制（如 BM25）从技能库中检索相关技能。这样的设计选择使我们能够聚焦于本工作的核心：研究如何通过经验驱动的学习来策划、更新与组织技能。然而，更先进的检索方法（如稠密检索、混合检索或学习型检索器）可能进一步提升被检索技能的相关性，从而带来更强的下游性能。我们将技能策划与技能检索的联合优化留待未来工作。

Simplified Skill Representation. Following Anthropic's skill paradigm (Anthropic, 2025b), we instantiate each skill as a single Markdown file that combines a YAML frontmatter and Markdown body. This simplification keeps the curator's action space tractable, but it discards two affordances of the original SKILL.md format: (i) supporting scripts and external resource files that allow skills to encapsulate executable procedures rather than purely declarative knowledge, and (ii) hierarchical organization in which a top-level skill can reference or compose lower-level sub-skills. As a result, behaviors that are most naturally expressed as runnable code or as compositions of finer-grained primitives must currently be flattened into prose. Extending SKILLOS to multi-file, hierarchical, and partially executable skills is a natural next step.
简化的技能表示。遵循 Anthropic 的技能范式（Anthropic, 2025b），我们将每个技能实例化为单个 Markdown 文件，将 YAML frontmatter 与 Markdown 正文合并在一起。该简化使策划者的操作空间保持可控，但同时丢弃了原始 SKILL.md 格式的两项能力：（i）支持脚本和外部资源文件，使技能能够封装可执行流程，而不仅仅是声明式知识；以及（ii）层级化组织——顶层技能可以引用或组合更低层级的子技能。结果是，目前需要把那些最自然地以可运行代码或更细粒度原语的组合来表达的行为，压平成纯文本叙述。将 SKILLOS 扩展为多文件、层级化且部分可执行的技能，是下一步的自然选择。


Frozen Agent Executor. Throughout training,we keep the agent executor ${\pi }_{\mathcal{L}}$ frozen and optimize only the skill curator ${\pi }_{\mathcal{S}}$ . This decoupling is deliberate: it isolates the contribution of skill curation, makes the recipe modular across executors, and avoids confounding our analysis with executor-side adaptation. The downside is that the curator can only shape the system's behavior through what it writes into SKILLREPO; any miscalibration between the curated skills and the executor's idiosyncrasies must be absorbed by the curator alone. Joint or alternating optimization of ${\pi }_{\mathcal{S}}$ and ${\pi }_{\mathcal{L}}$ may yield a better-aligned pair, at the cost of executor specificity and substantially higher training cost.
冻结的智能体执行器。在整个训练过程中，我们将智能体执行器 ${\pi }_{\mathcal{L}}$ 保持冻结，只优化技能策划者 ${\pi }_{\mathcal{S}}$。这种解耦是有意为之：它隔离了技能策划的贡献，使得配方在不同执行器之间具有模块化特点，并避免将执行器侧的适配与我们的分析混杂。缺点是，策划者只能通过写入 SKILLREPO 的内容来塑造系统行为；任何策划技能与执行器各自特性的错配，都只能由策划者单独吸收。尽管会损失执行器的专属性并显著提高训练成本，${\pi }_{\mathcal{S}}$ 与 ${\pi }_{\mathcal{L}}$ 的联合或交替优化可能会得到更匹配的一对。

## E. Future Research Directions
## E. 未来研究方向


Our work opens several promising directions for future research.
我们的工作开启了若干有前景的未来研究方向。


Agentic Search over Experiential Memory. SKILLOS currently retrieves relevant skills from SKILLREPO through a fixed top- $k$ BM25 lookup,treating retrieval as a static,one-shot operation. As the skill repository grows across thousands of tasks and domains, the bottleneck of self-evolving agents shifts from what to store to how to reliably retrieve and inject the right fragments at each decision step. A natural next step is to replace static retrieval with agentic search: letting the Skill Curator (or a dedicated retrieval agent) actively issue multiple queries, reformulate them based on intermediate evidence, and iteratively decide which skills to surface, cite, or compose for the executor. This reframes memory access as a first-class decision in the agent's policy rather than a preprocessing step, and opens the door to scaling SKILLOS to memory stores orders of magnitude larger than those considered here.
基于经验记忆的主体式搜索。当前，SKILLOS 通过固定的 top- $k$ BM25 查找从 SKILLREPO 检索相关技能，并将检索视为静态的、一次性操作。随着技能库在成千上万的任务与领域中不断扩展，自我演化智能体的瓶颈将从“存什么”转向“如何在每个决策步骤中可靠地检索并注入正确的片段”。自然的下一步是用主体式搜索替代静态检索：让技能策展人（Skill Curator，或专门的检索智能体）主动发起多次查询，根据中间证据对查询进行改写，并迭代决定向执行器呈现哪些技能、引用哪些技能或将哪些技能进行组合。这将把记忆访问重定义为智能体策略中的一等决策，而非预处理步骤，并为将 SKILLOS 扩展到比本文所考虑的更大数量级的记忆存储铺平道路。


Hierarchical and Compositional Skills. Our current skills are flat Markdown entries, each describing a single reusable pattern. Real agent competence, however, is hierarchical: high-level procedures invoke lower-level sub-skills, which in turn depend on primitive operations. Extending SKILLREPO to support hierarchical decomposition - where the curator learns not only to insert, update, and delete skills but also to link, compose, and abstract them - could enable the agent to build increasingly expressive procedural libraries over time. This direction connects naturally to program-synthesis and library-learning literature, and would allow SKILLOS to scale to longer-horizon tasks where single-skill retrieval is insufficient.
分层与可组合技能。我们当前的技能是扁平的 Markdown 条目，每个条目都描述一种单一的可复用模式。然而，真正的智能体能力是分层的：高层流程会调用更底层的子技能，而子技能又依赖原始操作。将 SKILLREPO 扩展为支持分层分解——不仅让策展人学习如何插入、更新和删除技能，还能对它们进行链接、组合与抽象——可以使智能体随着时间推移构建越来越富表达力的过程性知识库。这个方向自然衔接程序合成与库学习领域，并能让 SKILLOS 扩展到需要更长时间跨度的任务，而在这些任务中，单个技能检索已不足以应对。


Multi-Agent and Shared Memory. SKILLOS treats memory as a single agent's private artifact. In many realistic deployments, however, multiple agents operate in parallel (e.g., code review, multi-hop research, collaborative robotics) and could benefit from shared experiential memory. Open questions include how to arbitrate conflicting curation decisions from different agents, how to attribute credit when a shared skill contributes to one agent's success but another's failure, and how to preserve specialization while enabling cross-agent transfer. Our GRPO-based curator provides a natural starting point, but extending it to the multi-agent credit-assignment setting is non-trivial and likely to require new algorithmic ideas.
多智能体与共享记忆。SKILLOS 将记忆视为单个智能体的私有产物。然而在许多实际部署中，多智能体会并行工作（例如代码审查、多跳研究、协作机器人），因此可能需要共享的经验记忆。开放问题包括：如何裁决不同智能体之间相互冲突的策展决策；当共享技能促成一个智能体成功、却导致另一个智能体失败时，如何归因奖励（credit）；以及在实现跨智能体迁移的同时如何保持专业化。基于 GRPO 的策展人提供了一个自然起点，但将其扩展到多智能体的归因分配场景并不简单，可能需要新的算法思路。
